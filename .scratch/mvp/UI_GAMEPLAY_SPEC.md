# UI_GAMEPLAY_SPEC_v0.2

상태: Codex / Unity 구현 기준 문서  
기준 문서: UI_LAYOUT_v0.1.md  
범위: 16:9 Unity UI 레이아웃, 화면 상태, 핵심 상호작용, 컴포넌트 동작 규칙  
비범위: 최종 비주얼 스타일, 세부 색상 팔레트, 카드 일러스트, 애니메이션 타이밍

---

## 1. Purpose

이 문서는 Unity MVP 이후 UI를 실제 게임 화면으로 발전시키기 위한 구현 기준서다.

목표는 최종 아트 디자인을 끝내는 것이 아니라, Codex가 다음 UI 구현/리팩터링 작업을 할 때 다음 사항을 명확히 이해하도록 하는 것이다.

- 화면이 어떤 공통 프레임을 유지하는가
- 상태마다 Center Main Board가 어떻게 바뀌는가
- 카드, 칩, Payment Pot, 버튼이 어떤 규칙으로 동작하는가
- 플레이어가 매 턴 어떤 정보를 읽고 어떤 선택을 해야 하는가

---

## 2. Core UI Principle

이 게임의 UI는 메뉴 화면이 아니라 보드게임 테이블이다.

플레이어는 매 턴 다음 질문에 답해야 한다.

1. 지금 어떤 자산 카드를 볼 것인가?
2. 지금 살 것인가, 예약할 것인가, 기다릴 것인가, 자원을 확보할 것인가?
3. 이번 분기 목표 압박과 장기 포트폴리오 가치 사이에서 어떤 선택을 할 것인가?
4. 현재 보유 칩으로 비용을 충족할 수 있는가?
5. 딜 칩을 써서 전문자원 슬롯을 대체하고 현금 비용을 낮출 것인가?

## 3. Global Layout

기준 화면 비율: 16:9  
권장 기준 프레임: 1920 x 1080

```text
+----------------------------------------------------------------+
| Top Status Bar                                                  |
| 연도 / 분기 / 남은 영업일 / 분기 목표 / 현재 수익 / 환매 압력 |
+---------------------------------------------+------------------+
|                                             |                  |
| Center Main Board                           | Right Context    |
|                                             |                  |
| Market / Card Detail / Gain Liquidity       | 예약 카드        |
| Quarter Result                              | 보유 자산 요약   |
|                                             |                  |
+---------------------------------------------+------------------+
| Bottom Chip Tray                                               |
| 현금 지폐 다발 / 리서치 칩 / 신용 칩 / 원자재 칩 / 딜 칩       |
+----------------------------------------------------------------+
```

### Persistent Areas

다음 영역은 기본적으로 모든 화면 상태에서 유지된다.

- Top Status Bar
- Right Context
- Bottom Chip Tray

### Dynamic Area

다음 영역만 화면 상태에 따라 바뀐다.

- Center Main Board

## 4. Top Status Bar

### Role

자원 표시 영역이 아니라, 런의 시간과 압박을 보여주는 영역이다.

### Contains

- 회계년도
- 분기
- 남은 영업일
- 분기 목표 게이지 + 숫자
- 현재 분기 수익
- 환매 압력 게이지 + 숫자

### Does Not Contain

- 현금
- 리서치
- 신용
- 원자재
- 딜

### Rule

상단은 이번 분기의 시간 압박과 실패 압박을 담당한다.  
자원은 플레이어가 직접 조작하는 물건이므로 Bottom Chip Tray에 둔다.

---

## 5. Right Context

### Role

항상 보이는 플레이어 보드 상태판이다.  
선택 카드의 상세 패널이 아니다.

### Structure

```text
+------------------+
| 예약 카드         |
| [slot] [slot]    |
| [slot]           |
+------------------+
| 보유 자산 요약    |
| 보유 수           |
| 총 분기 수익      |
| 총 장기 가치      |
| 보유 미니 카드    |
+------------------+
```

### Reserved Cards

- 예약 슬롯은 3칸이다.
- 비어 있으면 빈 카드 슬롯으로 표시한다.
- 예약된 카드는 미니 카드로 표시한다.
- 예약 카드는 아직 살 수 있는 구체적인 미래 선택지이므로 카드 실물이 작게 보여야 한다.

### Owned Assets Summary

- 보유 카드 수
- 총 분기 운용 수익
- 총 장기 운용 가치
- 보유 미니 카드 목록은 선택 사항이다.
- 보유 영역은 개별 카드보다 포트폴리오 총합이 먼저 읽혀야 한다.

### Purchase Impact

구매 영향 미리보기는 Right Context에 넣지 않는다.  
필요하다면 Card Detail State의 Center Main Board 안, 특히 Actions 또는 Payment Pot 근처에 둔다.

---

## 6. Bottom Chip Tray

### Role

플레이어의 물리적 자원 영역이다.  
홀덤에서 자기 칩이 있는 곳에 가깝다.

### Cash

- 칩이 아니라 지폐 다발로 표현한다.
- 보유 수를 숫자로 표시한다.
- 드래그 불가.
- Buy 버튼의 현금 비용 판정에 사용한다.

### Professional Resources

대상:

- 리서치
- 신용
- 원자재
- 딜

규칙:

- 보유량만큼 개별 칩으로 표시한다.
- 하나의 스택 숫자로만 축약하지 않는다.
- 같은 자원끼리 약간 겹쳐 배치한다.
- 기본은 1줄이다.
- 많아지면 2줄까지 사용한다.
- 줄당 최대 5개 칩을 배치한다.
- 칩은 하단 트레이 밖으로 넘어가면 안 된다.
- 각 자원은 고유 영역을 가진다.
- Payment Pot이 활성화된 상태에서만 드래그 가능하다.

### Deal

- 전문자원 슬롯을 대체할 수 있는 와일드 칩이다.
- Payment Pot에 놓인 딜 1개마다 Buy 버튼의 현금 비용이 1 감소한다.

## 7. Asset Card Layout Rules

참고 방향:

- 아크노바: 카드 중앙과 상단의 큰 이미지
- 스플렌더: 아이콘 + 숫자 중심의 비용 언어
- 하스스톤: 강한 카드 프레임과 명확한 카드 정체성

### Allowed Text On Card

- 카드 이름
- 태그명

### Forbidden Text On Card

- 현금
- 리서치
- 신용
- 원자재
- 분기 운용 수익
- 장기 운용 가치
- 설명문 또는 효과문 줄글

### Card Areas

```text
+----------------------+
| 현금 배지      수익   |
| 전문자원 비용         |
|                      |
| 큰 이미지             |
|                      |
| 희귀도 / 이름 띠      |
| 태그명                |
|                      |
| 장기 가치             |
+----------------------+
```

### Placement Rules

- 카드 중앙~상단: 자산 이미지 크게
- 좌상단: 매수 비용
  - 현금은 지폐 배지
  - 전문자원은 칩 비용 스택
- 우상단: 분기 운용 수익, 아이콘 + 숫자
- 중앙 띠: 희귀도 색상띠와 카드 이름
- 하단 중간: 태그명
- 최하단: 장기 운용 가치, 큰 아이콘 + 숫자

### Readability Rule

카드 일러스트는 카드 정체성을 강화해야 하지만, 비용/수익/가치/태그의 가독성을 침범하면 안 된다.

## 8. Screen State: Market State

### Layout

Center Main Board는 3열 x 3행으로 구성한다.

```text
+-------------------------------------------------------------+
| 매도 임박              현재 시장              예비 시장       |
|                                                             |
| [card]                 [card]                 [small card]   |
| [card]                 [card]                 [small card]   |
| [card]                 [card]                 [small card]   |
+-------------------------------------------------------------+
```

### Columns

#### 매도 임박

- 카드 3장
- 중간 크기
- 클릭 가능
- Buy와 Reserve 가능
- 행동 가능한 밝기
- 곧 사라질 느낌을 주는 페이드/블러/이탈 효과 사용 가능

#### 현재 시장

- 카드 3장
- 중간 크기
- 클릭 가능
- Buy와 Reserve 가능
- 기본 선택 영역으로 가장 안정적으로 읽힌다.

#### 예비 시장

- 카드 3장
- 왼쪽/중앙보다 조금 작다.
- 차분하고 살짝 어둡다.
- 클릭 시 상세 미리보기만 가능하다.
- Buy와 Reserve 불가.

### Market Flow

```text
예비 시장 -> 현재 시장 -> 매도 임박 -> 제거
```

## 9. Screen State: Card Detail State

Card Detail State에서도 공통 프레임을 유지한다.  
Right Context와 Bottom Chip Tray는 계속 보인다.

### Center Main Board Layout

```text
+-------------------------------------------------------------+
| [Large Card Showcase] [Payment Pot / Table] [Actions]       |
+-------------------------------------------------------------+
```

### Left: Large Card Showcase

- 선택한 자산 카드를 크게 보여준다.
- Market State의 카드 문법을 그대로 사용한다.
- 줄글 상세 패널로 대체하지 않는다.
- 하스스톤형 세로 카드처럼 카드 실물이 보여야 한다.

### Center: Payment Pot / Table

- 매수 가능한 카드에서만 활성화된다.
- 전문자원 요구 슬롯만 포함한다.
- 현금 슬롯은 만들지 않는다.
- 전문자원 칩과 딜 칩을 받는 물리적 결제 테이블이다.

### Right: Actions

- Buy
- Reserve
- Back 또는 Close
- Market Peek
- 구매 영향 스트립

---

## 10. Card Detail: Buyable Card

대상:

- 매도 임박 카드
- 현재 시장 카드

### Structure

```text
+-------------------------------------------------------------+
| [Large Card]      [Payment Pot Slots]      [ Buy 현금 비용 ] |
|                   [slot] [slot]            [ Reserve 거래 ]  |
|                   [slot]                   [ 영향 스트립 ]    |
|                                            [ Back ] [Peek]   |
+-------------------------------------------------------------+
```

### Buy Button

- 큰 원형 버튼이다.
- 버튼 내부 또는 하단에 현금 지폐 아이콘 + 최종 현금 비용을 표시한다.
- Payment Pot에 딜 칩이 들어가면 현금 비용이 감소한다.
- 활성 조건:
  - 모든 전문자원 슬롯이 채워짐
  - 최종 현금 비용을 낼 수 있음

### Reserve Button

- Buy와 같은 급의 큰 원형 버튼이다.
- 예약 거래를 아이콘 + 숫자로 표시한다.
  - 환매 압력 +1
  - 딜 +1
- 긴 설명문은 넣지 않는다.

### Purchase Impact Strip

- Actions 근처에 작게 둔다.
- Right Context에 넣지 않는다.
- Buy를 실행했을 때 변하는 핵심 값만 보여준다.
  - 현금 변화
  - 분기 운용 수익 증가
  - 장기 운용 가치 증가
- 텍스트 라벨 없이 아이콘 + 숫자 변화로 표시한다.
- 계산 보조 정보이며 별도 상세 패널이 아니다.

### Back / Close

- 보조 액션이다.
- Card Detail에서 Market State로 돌아간다.

### Market Peek

- 토글이 아니라 press-and-hold 동작이다.
- 누르고 있는 동안 Card Detail을 숨겨 뒤의 시장판을 볼 수 있다.
- 손을 떼면 Card Detail이 돌아온다.
- Market Peek 중에는 뒤의 시장 카드와 상호작용하지 않는다.

## 11. Card Detail: Preview Card

대상:

- 예비 시장 카드

### Rules

- 큰 카드 미리보기만 제공한다.
- Buy 버튼 숨김.
- Reserve 버튼 숨김.
- Payment Pot 숨김.
- Close와 Market Peek만 제공한다.

### Purpose

예비 시장 카드는 미래 정보이지 현재 거래 대상이 아니다.

## 12. Payment Pot Rules

### Role

전문자원 비용을 지불하는 물리적 결제 테이블이다.

### Slot Rules

- 전문자원 비용만 슬롯을 만든다.
- 현금은 슬롯을 만들지 않는다.
- 각 슬롯은 요구 자원이 정해진 고정 슬롯이다.
- 빈 슬롯에는 흐릿한 요구 자원 아이콘이 보인다.
- 해당 전문자원 칩을 넣을 수 있다.
- 딜 칩은 어떤 전문자원 슬롯에도 넣을 수 있다.
- 잘못된 칩을 놓으면 거부되거나 튕겨나간다.
- 채운 칩은 다시 Bottom Chip Tray로 회수할 수 있다.

### Deal Rules

- 딜은 전문자원 슬롯의 와일드 칩이다.
- Payment Pot에 놓인 딜 1개마다 Buy 버튼의 현금 비용이 1 감소한다.
- 감소된 현금 비용은 Buy 버튼에 표시한다.

### Buy Activation

Buy는 다음 조건을 모두 만족할 때만 활성화된다.

1. 모든 전문자원 슬롯이 채워짐.
2. 최종 현금 비용을 낼 수 있음.

## 13. Screen State: Gain Liquidity State

Center Main Board는 카드가 아니라 자원 오브젝트 선택 화면이다.

```text
+-------------------------------------------------------------+
| 남은 선택 횟수: N                                           |
|                                                             |
| [현금 지폐 다발] [리서치 칩] [신용 칩] [원자재 칩]          |
+-------------------------------------------------------------+
```

### Rules

- 허용 횟수만큼 자원을 순차 선택한다.
- 자원 선택은 즉시 확정이며 취소할 수 없다.
- 선택하는 즉시 Bottom Chip Tray에 반영된다.
- 남은 선택 횟수도 즉시 감소한다.
- 상한에 걸린 자원은 비활성 또는 선택 불가 상태로 보인다.
- 딜은 Gain Liquidity 선택지에 없다.
- 자원 선택지에는 자산 카드 프레임을 사용하지 않는다.

### Purpose

이 상태는 카드를 고르는 화면이 아니라, 테이블에서 자원을 가져오는 화면처럼 느껴져야 한다.

## 14. Screen State: Quarter Result State

Quarter Result State도 공통 프레임을 유지한다.

### Layout

```text
+-------------------------------------------------------------+
| Top Status Bar                                               |
+---------------------------------------------+----------------+
| Quarter Result Board                         | Right Context  |
| 목표 게이지 결과                             | 고정 요약      |
| 수익 / 목표                                  |                |
| 환매 압력 변화                               |                |
| [Continue]                                   |                |
+---------------------------------------------+----------------+
| Bottom Chip Tray                                               |
+-------------------------------------------------------------+
```

### Rules

- 분기 결과를 보여준다.
- 분기 수익과 분기 목표를 비교한다.
- 목표 미달 시 환매 압력 증가를 보여준다.
- Continue 액션을 제공한다.
- Right Context는 유지한다.
- Bottom Chip Tray는 유지하되 조작 불가 또는 낮은 강조 상태로 보인다.
- Center Main Board는 분기 결과에 집중한다.

## 15. Core Interactions

### Card Click

- Market 카드 클릭 시 Card Detail State로 진입한다.
- 매도 임박과 현재 시장 카드는 매수 가능한 상세로 열린다.
- 예비 시장 카드는 미리보기 상세로 열린다.

### Chip Drag

- Payment Pot이 활성화된 상태에서 리서치, 신용, 원자재, 딜 칩을 Bottom Chip Tray에서 드래그할 수 있다.
- 현금 지폐 다발은 드래그하지 않는다.

### Payment Pot Snap

- 유효한 칩은 호환 슬롯에 스냅된다.
- 유효하지 않은 칩은 트레이로 돌아간다.

### Chip Recall

- 슬롯에 놓인 칩은 클릭 또는 지정 입력으로 회수할 수 있다.
- 회수된 칩은 Bottom Chip Tray로 돌아간다.

### Buy

- 모든 전문자원 슬롯이 채워지고 현금이 충분할 때만 활성화된다.
- Buy는 최종 현금 비용을 표시한다.

### Reserve

- 매도 임박과 현재 시장 카드에서만 가능하다.
- Reserve는 환매 압력 +1과 딜 +1을 표시한다.

### Back

- Card Detail에서 Market State로 돌아간다.

### Market Peek

- 누르고 있는 동안 Card Detail을 숨겨 시장판을 본다.
- 떼면 Card Detail이 돌아온다.
- Market Peek 중에는 시장 카드와 상호작용하지 않는다.
