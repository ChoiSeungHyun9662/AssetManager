# UI_LAYOUT_v0.1

상태: Figma 전달용 초안
범위: 16:9 Unity 와이어프레임 레이아웃, 화면 상태, 컴포넌트 역할, 핵심 상호작용 규칙

## 1. 목적

이 문서는 구현 전에 레이아웃 구조를 확정하기 위한 기획서다.

목표는 최종 비주얼 디자인을 끝내는 것이 아니다. 목표는 Figma에서 제대로 된 UI 레이아웃과 UI Asset Map을 만들 수 있도록, 게임의 핵심 의사결정이 어떤 화면 구조에서 읽혀야 하는지 정의하는 것이다.

플레이어가 매 턴 읽어야 하는 핵심 질문:

- 지금 어떤 자산 카드를 볼 것인가?
- 지금 살 것인가, 예약할 것인가, 기다릴 것인가, 자원을 확보할 것인가?
- 이번 분기 목표 압박과 장기 포트폴리오 가치 사이에서 어떤 선택을 할 것인가?

## 2. 이 문서에서 정하는 것

- 16:9 화면 구조.
- 화면 상태 구조:
  - Market State
  - Card Detail State
  - Gain Liquidity State
  - Quarter Result State
- 공통 레이아웃 영역:
  - Top Status Bar
  - Center Main Board
  - Right Context
  - Bottom Chip Tray
- 자산 카드의 정보 배치.
- Payment Pot 구조.
- 카드 클릭, 카드 상세 진입, 칩 드래그, Payment Pot 스냅, 칩 회수, Buy 활성화, Reserve, Back, Market Peek 규칙.
- Figma 전달 요구사항.

## 3. 이 문서에서 정하지 않는 것

- 최종 아이콘 모양.
- 최종 카드 프레임 장식.
- 최종 일러스트 스타일.
- 정확한 색상 팔레트.
- 16:9 기준을 넘어선 정확한 픽셀값.
- 세부 애니메이션 타이밍.
- Unity 구현 방식.

구체 비주얼은 Figma에서 먼저 레이아웃을 확정한 뒤 진행한다. Recraft AI 등 이미지 생성 도구는 카드 아트, 아이콘, 질감, 시각 베리에이션 제작 단계에서 사용한다.

## 4. 전체 16:9 프레임

기준 비율: 16:9.

Figma 권장 작업 프레임: 1920 x 1080.

```text
+----------------------------------------------------------------+
| Top Status Bar                                                  |
| 연도 / 분기 / 남은 영업일 / 분기 목표 / 현재 수익 / 환매 압력 |
+---------------------------------------------+------------------+
|                                             |                  |
| Center Main Board                           | Right Context    |
|                                             |                  |
| Market / Card Detail / Gain Liquidity       | 예약 카드        |
| 상태에 따라 바뀌는 핵심 조작 영역           | 보유 자산 요약   |
|                                             |                  |
+---------------------------------------------+------------------+
| Bottom Chip Tray                                               |
| 현금 지폐 다발 / 리서치 칩 / 신용 칩 / 원자재 칩 / 딜 칩       |
+----------------------------------------------------------------+
```

영역 역할:

- Top Status Bar: 시간과 압박.
- Center Main Board: 현재 행동.
- Right Context: 고정된 플레이어 보드 상태.
- Bottom Chip Tray: 현재 보유 자원과 조작 가능한 칩.

Center Main Board만 상태에 따라 바뀐다. Top Status Bar, Right Context, Bottom Chip Tray는 기본적으로 고정된다.

## 5. Top Status Bar

역할: 자원 표시가 아니라 런의 시간과 압박을 보여준다.

포함 정보:

- 회계년도와 분기.
- 남은 영업일.
- 분기 목표 게이지 + 숫자.
- 현재 분기 수익.
- 환매 압력 게이지 + 숫자.

제외 정보:

- 현금.
- 리서치.
- 신용.
- 원자재.
- 딜.

이유:

- 상단은 이번 분기의 시간 압박과 실패 압박을 담당한다.
- 자원은 실제로 조작하는 물건이므로 Bottom Chip Tray에 둔다.

## 6. Right Context

역할: 항상 보이는 플레이어 보드 상태판.

Right Context는 선택 카드 상세 패널이 아니다. Market State, Card Detail State, Gain Liquidity State에서 내용 구조가 고정되어야 한다.

2층 구조:

```text
+------------------+
| 예약 카드         |
| [slot] [slot]    |
| [slot]           |
+------------------+
| 보유 자산 요약    |
| 보유 수           |
| 총 분기 수익      |
| 총 장기 가치      |
| 보유 미니 카드    |
+------------------+
```

1층: 예약 카드

- 예약 슬롯 3칸.
- 비어 있으면 빈 카드 슬롯.
- 예약된 카드는 미니 카드로 표시.
- 미니 카드는 빠르게 훑을 수 있는 압축 정보만 보여준다.
- 예약 영역은 미니 카드 중심이다.
- 예약 카드는 아직 살 수 있는 구체적인 미래 선택지이므로 카드 실물이 작게 보여야 한다.

2층: 보유 자산 요약

- 보유 카드 수.
- 총 분기 운용 수익.
- 총 장기 운용가치.
- 보유 영역은 요약 숫자 중심이다.
- 보유 자산은 개별 카드보다 현재 포트폴리오 총합이 먼저 읽혀야 한다.
- 보유 미니 카드 목록은 선택 사항이며, 넣더라도 요약 숫자보다 약하게 보인다.

충돌 정리:

- 첨부 자료에는 Right Context에 구매 영향 미리보기를 넣는 추천안이 있었다.
- 현재 결정은 Right Context를 고정 정보판으로 유지하는 것이다.
- 선택 카드의 구매 영향이 필요하다면 Card Detail의 Center Main Board 안, 특히 Payment Pot 또는 Actions 주변에 둔다.

## 7. Bottom Chip Tray

역할: 플레이어의 물리적 자원 영역.

현금:

- 칩이 아니라 지폐 다발로 표현한다.
- 보유 수를 숫자로 표시한다.
- 드래그 불가.
- Buy 버튼의 현금 비용 판정에 사용한다.

리서치 / 신용 / 원자재 / 딜:

- 보유량만큼 개별 칩으로 표시한다.
- 하나의 스택 숫자로 축약하지 않는다.
- 같은 자원끼리 약간 겹쳐 배치한다.
- 기본은 1줄이다.
- 많아지면 2줄까지 사용한다.
- 줄당 최대 5개 칩을 배치한다.
- 칩은 하단 트레이 밖으로 넘어가면 안 된다.
- 현금, 리서치, 신용, 원자재, 딜은 각각 고유 영역을 가진다.
- 칩이 많아져도 다른 자원 영역이나 현금 지폐 다발을 가리면 안 된다.
- Payment Pot이 활성화된 상태에서는 드래그 가능하다.
- 딜은 전문자원 슬롯을 대체할 수 있는 와일드 칩이다.

핵심 구분:

- 현금은 구매력이다.
- 전문자원과 딜은 Payment Pot에 놓는 물리 칩이다.

## 8. 자산 카드 레이아웃

참고 방향:

- 아크노바: 카드 중앙과 상단의 큰 이미지.
- 스플렌더: 아이콘 + 숫자 중심의 비용 언어.
- 하스스톤: 강한 카드 프레임과 명확한 카드 정체성.

카드에서 허용되는 텍스트:

- 카드 이름.
- 태그명.

카드에서 금지되는 텍스트:

- 현금.
- 리서치.
- 신용.
- 원자재.
- 분기 운용 수익.
- 장기 운용가치.
- 설명문 또는 효과문 줄글.

카드 영역:

```text
+----------------------+
| 현금 배지      수익   |
| 전문자원 비용         |
|                      |
| 큰 이미지             |
|                      |
| 희귀도 / 이름 띠      |
| 태그명                |
|                      |
| 장기 가치             |
+----------------------+
```

영역 규칙:

- 카드 중앙에서 상단: 자산 이미지 크게.
- 좌상단: 매수 비용.
  - 현금은 별도 지폐 배지로 표시.
  - 전문자원은 별도 칩 비용 스택으로 표시.
  - 현금과 전문자원은 같은 비용 영역에 속하지만 같은 형태로 보이면 안 된다.
- 우상단: 분기 운용 수익, 아이콘 + 숫자만 표시.
- 중앙 띠: 희귀도 색상띠와 카드 이름.
- 하단 중간: 태그명.
- 최하단: 장기 운용가치, 큰 아이콘 + 숫자.

분기 운용 수익과 장기 운용가치는 둘 다 금융 계열 시각 언어를 사용해도 된다. 의미 구분은 주로 위치, 크기, 시각적 무게로 만든다.

## 9. Market State

Center Main Board는 3열 x 3행으로 구성한다.

```text
+-------------------------------------------------------------+
| 매도 임박              현재 시장              예비 시장       |
|                                                             |
| [card]                 [card]                 [small card]   |
| [card]                 [card]                 [small card]   |
| [card]                 [card]                 [small card]   |
+-------------------------------------------------------------+
```

열 규칙:

- 왼쪽 열: 매도 임박.
  - 카드 3장.
  - 중간 크기.
  - 클릭 가능.
  - Buy와 Reserve 가능.
  - 행동 가능한 밝기.
  - 왼쪽 테두리 페이드, 블러, 이탈 효과 등으로 곧 사라질 느낌을 준다.
- 중앙 열: 현재 시장.
  - 카드 3장.
  - 중간 크기.
  - 클릭 가능.
  - Buy와 Reserve 가능.
  - 기본 선택 영역으로 가장 안정적으로 읽힌다.
- 오른쪽 열: 예비 시장.
  - 카드 3장.
  - 왼쪽/중앙보다 조금 작다.
  - 차분하고 살짝 어둡다.
  - 클릭 시 상세 미리보기만 가능하다.
  - Buy와 Reserve 불가.

열 제목 규칙:

- 기본 화면에서 열 제목만 표시한다.
  - 매도 임박
  - 현재 시장
  - 예비 시장
- 기본 화면에서 보조 설명문은 표시하지 않는다.
  - 다음 진행 시 사라짐
  - 매수 / 예약 가능
  - 다음 진행 시 이동
- 각 열의 의미는 텍스트가 아니라 크기, 밝기, 테두리, 페이드 효과로 전달한다.

시장 흐름:

```text
예비 시장 -> 현재 시장 -> 매도 임박 -> 제거
```

## 10. Card Detail State

Card Detail State에서도 공통 프레임을 유지한다. Right Context와 Bottom Chip Tray는 계속 보인다.

Center Main Board는 3분할 구조다.

```text
+-------------------------------------------------------------+
| [Large Card Showcase] [Payment Pot / Table] [Actions]       |
+-------------------------------------------------------------+
```

왼쪽: Large Card Showcase

- 선택한 자산 카드를 크게 보여준다.
- Market State의 카드와 같은 카드 문법을 사용한다.
- 줄글 상세 패널로 대체하지 않는다.

중앙: Payment Pot / Table

- 매수 가능한 카드에서만 활성화된다.
- 전문자원 요구 슬롯만 포함한다.
- 현금 슬롯은 만들지 않는다.

오른쪽: Actions

- Buy.
- Reserve.
- Back 또는 Close.
- Market Peek.
- 구매 영향 스트립.

### 10.1 매수 가능한 카드 상세

대상:

- 매도 임박 카드.
- 현재 시장 카드.

구조:

```text
+-------------------------------------------------------------+
| [Large Card]      [Payment Pot Slots]      [ Buy 현금 비용 ] |
|                   [slot] [slot]            [ Reserve 거래 ]  |
|                   [slot]                   [ 영향 스트립 ]    |
|                                            [ Back ] [Peek]   |
+-------------------------------------------------------------+
```

Buy 버튼:

- 큰 원형 버튼.
- 버튼 내부 또는 하단에 현금 지폐 아이콘 + 최종 현금 비용 표시.
- Payment Pot에 딜 칩이 들어가면 현금 비용이 감소한다.
- 활성 조건:
  - 모든 전문자원 슬롯이 채워짐.
  - 최종 현금 비용을 낼 수 있음.

Reserve 버튼:

- Buy와 같은 급의 큰 원형 버튼.
- 예약 거래를 아이콘 + 숫자로 표시한다.
  - 환매 압력 +1.
  - 딜 +1.
- 긴 설명문은 넣지 않는다.

구매 영향 스트립:

- Card Detail의 Actions 근처에 작게 둔다.
- Right Context에 넣지 않는다.
- Buy를 실행했을 때 변하는 핵심 값만 보여준다.
  - 현금 변화.
  - 분기 운용 수익 증가.
  - 장기 운용가치 증가.
- 텍스트 라벨을 쓰지 않고 아이콘 + 숫자 변화로 표시한다.
- 이 스트립은 계산 보조 정보이지 별도 상세 패널이 아니다.

Back / Close:

- 보조 액션.

Market Peek:

- 토글이 아니라 press-and-hold 동작.
- 누르고 있는 동안 Card Detail을 숨겨 뒤의 시장판을 볼 수 있다.
- 손을 떼면 Card Detail이 돌아온다.
- Market Peek 중에는 뒤의 시장 카드와 상호작용하지 않는다.

### 10.2 예비 시장 카드 상세

대상:

- 예비 시장 카드.

규칙:

- 큰 카드 미리보기만 제공한다.
- Buy 버튼 숨김.
- Reserve 버튼 숨김.
- Payment Pot 숨김.
- Close와 Market Peek만 제공한다.

목적:

- 예비 시장 카드는 미래 정보이지 현재 거래 대상이 아니다.

## 11. Payment Pot

역할: 전문자원 비용을 지불하는 물리적 결제 테이블.

슬롯 규칙:

- 전문자원 비용만 슬롯을 만든다.
- 현금은 슬롯을 만들지 않는다.
- 각 슬롯은 요구 자원이 정해진 고정 슬롯이다.
- 빈 슬롯에는 흐릿한 요구 자원 아이콘이 보인다.
- 해당 전문자원 칩을 넣을 수 있다.
- 딜 칩은 어떤 전문자원 슬롯에도 넣을 수 있다.
- 잘못된 칩을 놓으면 거부되거나 튕겨나간다.
- 채운 칩은 다시 Bottom Chip Tray로 회수할 수 있다.

예시:

```text
요구 비용: 리서치 1 + 신용 1

+-------------------+
| [리서치 슬롯]      |
| [신용 슬롯]        |
+-------------------+
```

딜 규칙:

- 딜은 전문자원 슬롯의 와일드 칩이다.
- Payment Pot에 놓인 딜 1개마다 Buy 버튼의 현금 비용이 1 감소한다.
- 감소된 현금 비용은 Buy 버튼에 표시한다.

## 12. Gain Liquidity State

Center Main Board는 카드가 아니라 자원 오브젝트 선택 화면이다.

```text
+-------------------------------------------------------------+
| 남은 선택 횟수: N                                           |
|                                                             |
| [현금 지폐 다발] [리서치 칩] [신용 칩] [원자재 칩]          |
+-------------------------------------------------------------+
```

규칙:

- 이미 확정된 룰에 따라 허용 횟수만큼 자원을 순차 선택한다.
- 자원 선택은 즉시 확정이며 취소할 수 없다.
- 선택하는 즉시 Bottom Chip Tray에 반영된다.
- 남은 선택 횟수도 즉시 감소한다.
- 상한에 걸린 자원은 비활성 또는 선택 불가 상태로 보인다.
- 딜은 Gain Liquidity 선택지에 없다.
- 자원 선택지에는 자산 카드 프레임을 사용하지 않는다.

목적:

- 이 상태는 카드를 고르는 화면이 아니라, 테이블에서 자원을 가져오는 화면처럼 느껴져야 한다.

## 13. Quarter Result State

Quarter Result State도 공통 프레임을 유지한다. Top Status Bar, Right Context, Bottom Chip Tray는 계속 보이고, Center Main Board만 분기 결과판으로 바뀐다.

확정 요구사항:

- 분기 결과를 보여준다.
- 분기 수익과 분기 목표를 비교한다.
- 목표 미달 시 환매 압력 증가를 보여준다.
- Continue 액션을 제공한다.

레이아웃 규칙:

- Right Context는 유지한다.
- Bottom Chip Tray는 유지하되 조작 불가 또는 낮은 강조 상태로 보인다.
- Center Main Board는 분기 결과에 집중한다.
- 결과판은 분기 목표 게이지 결과, 수익 / 목표, 환매 압력 변화를 가장 크게 보여준다.
- Continue는 결과판의 주 액션이다.

구조:

```text
+-------------------------------------------------------------+
| Top Status Bar                                               |
+---------------------------------------------+----------------+
| Quarter Result Board                         | Right Context  |
| 목표 게이지 결과                             | 고정 요약      |
| 수익 / 목표                                  |                |
| 환매 압력 변화                               |                |
| [Continue]                                   |                |
+---------------------------------------------+----------------+
| Bottom Chip Tray                                               |
+-------------------------------------------------------------+
```

## 14. 핵심 상호작용

카드 클릭:

- Market 카드 클릭 시 Card Detail State로 진입한다.
- 매도 임박과 현재 시장 카드는 매수 가능한 상세로 열린다.
- 예비 시장 카드는 미리보기 상세로 열린다.

칩 드래그:

- Payment Pot이 활성화된 상태에서 리서치, 신용, 원자재, 딜 칩을 Bottom Chip Tray에서 드래그할 수 있다.
- 현금 지폐 다발은 드래그하지 않는다.

Payment Pot 스냅:

- 유효한 칩은 호환 슬롯에 스냅된다.
- 유효하지 않은 칩은 트레이로 돌아간다.

칩 회수:

- 슬롯에 놓인 칩은 회수할 수 있다.
- 회수된 칩은 Bottom Chip Tray로 돌아간다.

Buy 활성화:

- 모든 전문자원 슬롯이 채워지고 현금이 충분할 때만 Buy가 활성화된다.
- Buy는 최종 현금 비용을 표시한다.

Reserve:

- 매도 임박과 현재 시장 카드에서만 가능하다.
- Reserve는 환매 압력 +1과 딜 +1을 표시한다.

Back:

- Card Detail에서 Market State로 돌아간다.

Market Peek:

- 누르고 있는 동안 Card Detail을 숨겨 시장판을 본다.
- 떼면 Card Detail이 돌아온다.
