# 07. 보유 자산 인컴과 운용 수익 기록

Status: ready-for-agent

## Parent

.scratch/mvp/PRD.md

## What to build

매수한 자산을 보유 자산으로 표시하고, 다음 영업일 시작부터 인컴을 발생시키며, 인컴을 운용 수익으로 기록한다. 예약 카드와 시장 카드는 운용가치, 인컴, 분기 정산 대상에서 제외되는 상태 분리가 명확해야 한다.

## Code work

- AssetCard module에 OwnedAssetState와 AcquiredOrder를 연결한다.
- 보유 자산 목록을 관리하고 보유 자산 수와 총 운용가치를 계산한다.
- 보유 자산 인컴 적용 함수를 구현한다.
- 인컴은 다음 영업일 시작 시 발생하고 매수한 당일 즉시 발생하지 않는다.
- 인컴으로 얻은 현금은 ResourceLedger의 운용 수익 경로로 기록한다.
- 예약 카드와 시장 카드는 보유 자산 계산, 운용가치 계산, 인컴 계산에서 제외한다.
- 보유 자산은 새 카드 보충 후보에서 제외되도록 card runtime state를 갱신한다.

## Unity Editor work

- 보유 자산 또는 포트폴리오 요약 영역을 만든다.
- 보유 자산 수, 현재 운용가치, 이번 분기 운용 수익을 표시한다.
- 매수 후 보유 자산 리스트에 카드 요약 UI가 추가되게 연결한다.
- 다음 영업일 시작 시 인컴 발생 결과가 현금과 운용 수익 표시에서 갱신되게 연결한다.

## Verification

- 시장 카드를 매수하면 보유 자산 수가 증가한다.
- 매수 직후 같은 영업일에는 해당 카드 인컴이 발생하지 않는다.
- 다음 영업일 시작 시 해당 카드 인컴이 현금과 분기 운용 수익에 반영된다.
- 보유 자산 운용가치 합계가 표시된다.
- 예약 카드와 시장 카드는 운용가치와 인컴에 포함되지 않는다.

## Acceptance criteria

- [ ] 매수 완료된 카드는 Owned 상태가 된다.
- [ ] 보유 자산 수와 현재 운용가치를 계산해 표시한다.
- [ ] 보유 자산 인컴은 영업일 시작 시 발생한다.
- [ ] 매수 당일에는 새 보유 자산 인컴이 즉시 발생하지 않는다.
- [ ] 인컴 현금은 운용 수익으로 기록된다.
- [ ] 예약 카드와 시장 카드는 운용가치와 인컴 계산에서 제외된다.
- [ ] 보유 자산은 시장에 다시 등장하지 않는다.
