# 06. 시장 카드 매수와 최소 칩 결제

Status: ready-for-agent

## Parent

.scratch/mvp/PRD.md

## What to build

시장 카드 상세보기에서 전문 자원 비용 슬롯과 딜 칩을 사용해 결제 상태를 만들고, 매수 버튼으로 시장 카드를 보유 자산으로 전환한다. 매수 확정 전에는 자원을 소비하지 않고, 검증 실패 시 자원과 카드와 시장 상태를 바꾸지 않아야 한다.

## Code work

- PurchasePayment module을 구현한다.
- 카드의 전문 자원 비용을 PaymentSlotState 목록으로 만든다.
- 전문 자원 칩 클릭 배치와 회수, 딜 칩 클릭 배치와 회수를 지원한다.
- 딜 칩은 모든 전문 자원 비용 슬롯에 배치 가능하게 한다.
- 딜 칩 1개마다 기본 현금 비용을 1 낮추고 최종 현금 비용은 0 미만으로 내려가지 않게 한다.
- 매수 가능 조건은 선택 카드 유효성, 구매 출처 유효성, 모든 전문 슬롯 충족, 최종 현금 비용 지불 가능, 현재 행동 상태를 기준으로 판정한다.
- 매수 확정 시 전문 자원, 딜, 현금을 소비하고 카드를 Owned 상태로 바꾼다.
- 시장 카드 매수 후 구매한 시장 슬롯만 새 카드로 보충하고 시장 테이프를 진행하지 않는다.
- 매수 후 구매 출처를 MarketTape로 기록한다.
- 매수 확정 후 추가 매수권이 없으면 영업일을 종료한다.

## Unity Editor work

- CardDetail Panel에 전문 자원 비용 슬롯 UI를 표시한다.
- 보유 자원 영역의 리서치, 신용, 원자재, 딜 칩을 클릭해 슬롯에 배치할 수 있게 연결한다.
- 슬롯 위 칩을 클릭해 회수할 수 있게 연결한다.
- 매수 버튼의 활성/비활성 상태와 최종 현금 비용 표시를 결제 상태에 연결한다.
- 시장 카드 매수 후 보유 자산 영역에 카드가 추가되고 시장 슬롯이 보충되게 연결한다.
- 자원 부족, 슬롯 미충족, 현금 부족에 대한 짧은 피드백을 연결한다.

## Verification

- Play Mode에서 시장 카드 상세보기로 들어가 칩을 배치하고 회수할 수 있다.
- 칩 배치만으로는 보유 자원이 줄지 않는다.
- 매수 확정 시 보유 자원이 줄고 카드가 보유 자산으로 이동한다.
- 시장 카드 매수 후 산 자리만 새 카드로 보충된다.
- 시장 테이프 진행은 발생하지 않는다.
- 결제 조건이 부족하면 매수 버튼이 비활성화되거나 클릭해도 상태가 바뀌지 않는다.

## Acceptance criteria

- [ ] 전문 자원 비용 슬롯이 카드 비용에서 생성된다.
- [ ] 전문 자원 칩과 딜 칩을 슬롯에 배치하고 회수할 수 있다.
- [ ] 딜 칩은 어떤 전문 자원 슬롯에도 배치 가능하다.
- [ ] 딜 칩 수만큼 기본 현금 비용이 감소한다.
- [ ] 매수 확정 전에는 자원이 소비되지 않는다.
- [ ] 검증 실패 시 자원, 카드, 시장 상태가 변경되지 않는다.
- [ ] 시장 카드 매수는 산 자리만 보충하고 시장 테이프를 진행하지 않는다.
- [ ] 매수 확정은 영업일을 소비한다.
