# 08. 자원 확보 액션

Status: ready-for-agent

## Parent

.scratch/mvp/PRD.md

## What to build

Market 상태에서 중앙 은행을 클릭해 GainLiquidity 상태로 전환하고, 현금, 리서치, 신용, 원자재를 선택해 자원을 확보한다. 첫 자원 획득 전에는 닫을 수 있지만, 첫 자원 획득 후에는 자원 확보가 완료되거나 유효 선택지가 없어 자동 종료될 때까지 다른 행동을 할 수 없어야 한다.

## Code work

- LiquidityAction module을 구현한다.
- 선택 가능한 기본 자원은 현금, 리서치, 신용, 원자재로 제한한다.
- 딜은 자원 확보로 얻을 수 없게 한다.
- 첫 자원 클릭 시 HasGainedAnyResource를 true로 만들고 행동을 확정한다.
- 같은 자원 2개 또는 서로 다른 자원 3개 선택 시 완료 처리한다.
- 현재 선택 상태에서 유효하지 않은 조합이 만들어지지 않도록 다음 선택 가능 여부를 판정한다.
- 전문 자원 합계가 10이면 전문 자원 버튼을 비활성화한다.
- 전문 자원 9/10에서 전문 자원 1개를 얻으면 이후 전문 자원 선택을 막는다.
- 더 이상 유효한 선택지가 없으면 자동 종료하고 필요하면 자원칩 최대 한도 도달 메시지를 표시한다.
- 조달 현금은 ResourceLedger의 조달 현금 경로로 추가한다.

## Unity Editor work

- 시장 영역에 중앙 은행 UI를 배치한다.
- GainLiquidity Panel을 만들고 현금, 리서치, 신용, 원자재 버튼, 선택된 자원 표시, 닫기 버튼, 메시지 영역을 배치한다.
- 중앙 은행 클릭은 Market 상태에서만 GainLiquidity로 전환되게 연결한다.
- 첫 자원 획득 전 닫기 버튼은 Market 복귀로 연결한다.
- 첫 자원 획득 후 닫기 버튼과 다음 영업일 버튼이 비활성화되게 연결한다.
- 자원 버튼의 활성/비활성 상태를 선택 상태와 전문 자원 한도에 연결한다.

## Verification

- Play Mode에서 중앙 은행 클릭으로 자원 확보 화면이 열린다.
- 아무 자원도 얻지 않은 상태에서 닫기를 누르면 영업일 소비 없이 Market으로 돌아간다.
- 현금, 현금을 선택하면 현금 +2 후 영업일이 종료된다.
- 현금, 리서치, 신용을 선택하면 세 자원 획득 후 영업일이 종료된다.
- 전문 자원 10/10 상태에서는 현금만 선택 가능하다.
- 자원 확보로 얻은 현금은 분기 운용 수익을 증가시키지 않는다.

## Acceptance criteria

- [ ] 자원 확보는 중앙 은행 클릭으로 진입한다.
- [ ] GainLiquidity는 시장 영역을 대체한다.
- [ ] 첫 자원 획득 전 닫기는 영업일을 소비하지 않는다.
- [ ] 첫 자원 획득 후 닫기는 불가능하다.
- [ ] 같은 자원 2개 또는 서로 다른 자원 3개로 완료된다.
- [ ] 딜은 선택할 수 없다.
- [ ] 전문 자원 한도 10에 따라 버튼 상태가 갱신된다.
- [ ] 조달 현금은 운용 수익에 포함하지 않는다.
