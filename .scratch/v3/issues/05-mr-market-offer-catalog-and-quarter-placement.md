## What to build

별도 시장 상황 시스템을 만들지 않고, Mr.Market 제안 3개를 매 분기 시장 상황으로 배치한다. 분기 시작 시 시장 테이프를 먼저 갱신한 뒤, 결제 완화형 1개를 1~2번 슬롯에, 성격 변환형 1개를 3~5번 슬롯에, 템포형 1개를 6~8번 슬롯에 배치한다.

이 슬라이스는 제안 카탈로그, 역할군, 분기 배치, 공개 상태, 분기말 폐기를 완성한다. 실제 구매 효과 적용은 뒤 이슈에서 다룬다.

## Acceptance criteria

- [ ] Mr.Market 제안은 별도 시장 상황 시스템이 아니라 현재 분기의 시장 상황으로 관리된다.
- [ ] 분기 시작 시 시장 테이프 갱신이 먼저 실행되고 이후 제안 3개가 생성된다.
- [ ] 1~2번 슬롯에는 결제 완화형 제안 1개가 배치된다.
- [ ] 3~5번 슬롯에는 성격 변환형 제안 1개가 배치된다.
- [ ] 6~8번 슬롯에는 템포형 제안 1개가 배치된다.
- [ ] 한 분기의 세 제안은 서로 다른 역할군에서 나온다.
- [ ] 제안은 슬롯에 묶이며 카드에 직접 붙지 않는다.
- [ ] 제안은 공개 정보로 조회 가능하다.
- [ ] 제안은 태그 조건이나 미션 조건을 요구하지 않는다.
- [ ] 결제 완화형 카탈로그는 현금, 독서, 명상, 인내 비용 -1 네 공식을 가진다.
- [ ] 성격 변환형 카탈로그는 PRD의 20개 초안 공식을 데이터로 가진다.
- [ ] 템포형 카탈로그는 PRD의 20개 초안 공식을 데이터로 가진다.
- [ ] 같은 공식은 다른 분기에 다시 등장할 수 있다.
- [ ] 분기말에 사용되지 않은 제안은 폐기된다.
- [ ] 회계년도가 뒤로 갈수록 강한 제안이 더 높은 확률로 등장할 수 있는 가중치 구조가 있다.
- [ ] 관련 EditMode 테스트가 역할군별 배치, 분기 폐기, 공개 상태, 회계년도 가중치를 검증한다.
- [ ] 최소 UI-facing 검증이 시장 슬롯 위의 제안 배지를 확인한다.
