## What to build

v3의 새 규칙이 플레이어에게 과하게 설명적이지 않으면서도 읽히도록 표시 경로를 통합한다. 카드 표면에는 핵심 숫자, 태그, 제안 배지, 계산된 최종 값이 보여야 한다. 호버 또는 호가 모달에는 원본 값과 최종 값만 보여주고, 왜 바뀌었는지에 대한 사유 목록은 보여주지 않는다.

제안 슬롯의 주식을 사려 할 때는 확인 모달에서 제안 공식과 최종 계산 값을 보여준다. 미션 쪽은 후보/확정 상태, 조건, 정산 공식, 분기말 미션 수익이 일관되게 보여야 한다.

## Acceptance criteria

- [ ] 주식 카드 표면은 핵심 숫자와 태그를 보여준다.
- [ ] 제안 슬롯의 주식 카드 표면은 제안 배지를 보여준다.
- [ ] 주식 카드 표면은 모든 수정이 반영된 최종 가치와 최종 배당을 보여준다.
- [ ] 소모형 자원 카드가 제안 슬롯에 있으면 제안이 비활성임을 보여준다.
- [ ] 호버 또는 호가 모달은 원본 가치와 최종 가치를 보여준다.
- [ ] 호버 또는 호가 모달은 원본 배당과 최종 배당을 보여준다.
- [ ] 호버 또는 호가 모달은 원본 비용과 최종 비용을 보여준다.
- [ ] 호버 또는 호가 모달은 값 변경의 사유별 상세 로그를 보여주지 않는다.
- [ ] 제안 슬롯 주식 구매 확인 모달은 제안 공식과 최종 계산 값을 보여준다.
- [ ] 미션 후보 UI는 이름, 대상 태그, 클리어 조건, 정산 공식, 표시용 난이도를 보여준다.
- [ ] 확정 후 미션 UI는 확정 미션 1장만 보여준다.
- [ ] 분기말 UI는 미션 수익을 별도 라인으로 보여준다.
- [ ] 표시 문자열은 `시장 상황`이라는 별도 시스템이 있는 것처럼 보이지 않고 Mr.Market 제안을 중심으로 표현된다.
- [ ] 관련 PlayMode/UI 테스트가 카드 표면, 호버/모달, 구매 확인, 미션 후보/확정, 분기말 표시를 검증한다.
