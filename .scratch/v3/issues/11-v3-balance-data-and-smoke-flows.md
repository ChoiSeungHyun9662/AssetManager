## What to build

v3 기능이 따로따로 붙은 뒤, 실제 런 흐름에서 함께 작동하는지 검증할 수 있는 밸런스 데이터와 스모크 시나리오를 정리한다. 이 이슈는 최종 수치 밸런싱이 아니라, 기획자가 검토하고 수정할 수 있는 데이터 구조와 초안 값을 준비하는 작업이다.

핵심은 태그, 미션, Mr.Market 제안, 소모형 자원 카드, 예약, 추가 매수권, 분기 평가가 한 화면 흐름에서 충돌하지 않는지 확인하는 것이다. 매도 타이밍 설계는 여전히 이슈 범위 밖이며, 후속 논의로 남긴다.

## Acceptance criteria

- [ ] 초기 미션 풀 25장이 실제 런 후보 생성에 사용된다.
- [ ] 결제 완화형 4종, 성격 변환형 20종, 템포형 20종이 데이터로 검토 가능하다.
- [ ] 제안별 가중치와 회계년도별 강도 가중치를 데이터로 조정할 수 있다.
- [ ] 1/2 회계년도 8영업일, 3 회계년도 10영업일 기준에 맞춘 분기 평가 목표 초안이 있다.
- [ ] 늘어난 영업일 수에 맞춰 배당/현금 흐름 스모크 기대값이 갱신된다.
- [ ] 단일 1x8 시장에서 주식과 소모형 자원 카드가 계속 경쟁한다.
- [ ] 별도 투자 철학 시장 테이프가 생기지 않는다.
- [ ] 빠른 진입 미션을 초반에 확정하고 여러 분기 미션 수익을 누적하는 스모크 플로우가 통과한다.
- [ ] 집중/고가치/호일 미션을 중후반에 확정해 큰 분기 수익을 얻는 스모크 플로우가 통과한다.
- [ ] 성격 변환형 제안으로 산 카드가 미션 정산 고점을 높이는 스모크 플로우가 통과한다.
- [ ] 템포 제안으로 추가 매수권 체인이 발생하는 스모크 플로우가 통과한다.
- [ ] 예약 카드가 다음 분기에 새 제안을 받는 스모크 플로우가 통과한다.
- [ ] PlayMode 또는 end-to-end 테스트가 v3 주요 규칙을 한 런 안에서 함께 검증한다.
- [ ] 매도 타이밍 설계는 구현하지 않고 후속 논의 항목으로만 남긴다.
