## What to build

Mr.Market 제안 슬롯의 기본 구매 규칙을 완성하고, 결제 완화형 제안을 실제 결제 계산에 적용한다. 제안은 주식 카드에만 활성화된다. 제안 슬롯의 주식은 일반 매수로 살 수 없고, 반드시 제안 조건을 포함한 매수로만 살 수 있다.

소모형 자원 카드가 제안 슬롯에 있으면 제안은 비활성으로 보이고 소모되지 않는다. 구매 실패는 비용, 제안, 영업일, 추가 매수권을 바꾸지 않는 완전 no-op이어야 한다.

## Acceptance criteria

- [ ] 제안은 주식 카드에만 활성화된다.
- [ ] 소모형 자원 카드가 제안 슬롯에 있으면 제안은 비활성 상태로 보인다.
- [ ] 소모형 자원 카드 구매는 비활성 제안을 소모하지 않는다.
- [ ] 비활성으로 남아 있던 제안 슬롯에 이후 주식이 들어오면 해당 주식은 제안을 사용할 수 있다.
- [ ] 제안 슬롯의 주식은 일반 매수 경로로 구매할 수 없다.
- [ ] 제안 슬롯의 주식은 제안 조건이 적용된 매수 경로로만 구매된다.
- [ ] 결제 완화형 제안은 결제 순간에만 적용되고 영구 흔적을 남기지 않는다.
- [ ] 현금 비용 -1 제안은 최종 현금 비용을 0 미만으로 내리지 않는다.
- [ ] 독서 비용 -1 제안은 최종 독서 비용을 0 미만으로 내리지 않는다.
- [ ] 명상 비용 -1 제안은 최종 명상 비용을 0 미만으로 내리지 않는다.
- [ ] 인내 비용 -1 제안은 최종 인내 비용을 0 미만으로 내리지 않는다.
- [ ] 성공한 제안 매수는 해당 제안을 소모한다.
- [ ] 구매 실패는 비용, 제안 소모, 영업일, 추가 매수권 상태를 바꾸지 않는다.
- [ ] 관련 EditMode 테스트가 주식 전용 활성화, 소모형 자원 비활성화, 제안 강제 매수, 할인 계산, 실패 no-op을 검증한다.
- [ ] PlayMode 또는 UI-facing 테스트가 제안 슬롯 주식의 구매 확인 경로와 소모형 자원 비활성 표시를 검증한다.
