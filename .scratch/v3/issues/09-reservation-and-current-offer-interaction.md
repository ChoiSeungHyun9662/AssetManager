## What to build

v3 예약 규칙을 Mr.Market 제안과 맞춘다. 예약은 카드를 시장 슬롯에 고정하는 무료 행동이며, 제안 자체를 예약하지 않는다. 동시에 예약 가능한 카드는 1장뿐이다.

예약 카드가 제안 슬롯에 있으면 그 분기 동안 해당 슬롯의 현재 제안을 받는다. 다음 분기로 넘어가면 카드는 같은 슬롯에 남지만 기존 제안은 폐기되고 새 분기 제안 배치를 따른다.

## Acceptance criteria

- [ ] 예약은 무료 행동으로 처리된다.
- [ ] 동시에 예약 가능한 카드는 1장뿐이다.
- [ ] 예약은 카드만 보존하고 제안을 보존하지 않는다.
- [ ] 예약 주식은 기존 시장 슬롯에 남는다.
- [ ] 예약 주식은 시장 테이프 진행 중 이동하지 않는다.
- [ ] 예약 주식이 제안 슬롯에 있으면 현재 분기의 해당 슬롯 제안으로 구매된다.
- [ ] 예약 주식이 제안 슬롯을 차지하는 동안 다른 카드는 그 제안을 사용할 수 없다.
- [ ] 다음 분기 시작 시 예약 카드는 같은 슬롯에 유지된다.
- [ ] 다음 분기 시작 시 이전 분기의 제안은 폐기된다.
- [ ] 다음 분기 제안이 예약 카드의 슬롯에 배치되면 예약 카드는 새 제안을 받는다.
- [ ] 추가 매수권 상태에서 예약해도 추가 매수권은 유지된다.
- [ ] 추가 매수권으로 예약 주식을 구매할 수 있다.
- [ ] 기존 v2의 다중 예약, 예약 영업일 소비, 예약 Deal 지급이 v3 규칙과 충돌하지 않도록 정리된다.
- [ ] 관련 EditMode 테스트가 1장 제한, 무료 예약, 카드/제안 분리, 분기 전환, 추가 매수권 보존을 검증한다.
- [ ] PlayMode 또는 UI-facing 테스트가 예약 카드의 제안 표시와 구매 경로를 검증한다.
