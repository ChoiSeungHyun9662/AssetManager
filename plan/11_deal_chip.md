# 11. 딜 시스템

## 1. 목적

이 문서는 딜의 역할, 획득 방식, 보유 한도, 주식 매수에서의 사용 규칙을 정의한다.

딜은 예약 보상으로 얻는 특수 자원이다.

```text
딜 1개
= 투자 철학 비용 슬롯 1칸 대체
+ 기본 현금 비용 1 감소
```

---

## 2. 딜 획득

딜은 주식 예약으로 획득한다.

```text
주식 예약
→ 딜 +1
```

딜은 소모형 자원 카드로 획득하지 않는다.

---

## 3. 딜 보유 한도

딜은 최대 3개까지 보유한다.

```text
딜 최대 보유량: 3
```

딜이 3개여도 예약은 가능하다.
추가 딜만 폐기한다.

```text
딜 3 / 3
→ 예약 실행
→ 딜 +1 시도
→ 추가 딜 폐기
→ "딜 최대 보유: 추가 딜 폐기"
```

---

## 4. 딜의 결제 역할

딜은 주식 매수에만 사용할 수 있다.

딜은 모든 투자 철학 비용 슬롯에 배치할 수 있다.

```text
딜 → 독서 슬롯 가능
딜 → 명상 슬롯 가능
딜 → 인내 슬롯 가능
```

딜이 배치된 슬롯은 결제 완료 상태로 본다.

딜 1개는 기본 현금 비용을 1 낮춘다.

```text
사용한 딜 수 = N
기본 현금 비용 -= N
```

최소 현금 비용은 0이다.

---

## 5. 인플레이션과 딜

딜 할인은 인플레이션 적용 전에 반영한다.

```text
기본 현금 비용
→ 딜 할인
→ 인플레이션 적용
→ 최종 현금 비용
```

예시:

```text
기본 현금 비용 5
딜 2개 사용
인플레이션 +1
→ 최종 현금 비용 4
```

---

## 6. 소비 타이밍

딜은 슬롯에 배치하는 순간 소비되지 않는다.

```text
딜 배치
→ 아직 소비 아님
```

딜은 매수 확정 시점에 소비된다.

```text
매수 확정
→ 배치된 딜 소비
```

매수 확정 전 취소하면 딜은 회수된다.

---

## 7. 소모형 자원 카드와 딜

딜은 소모형 자원 카드 결제에 사용할 수 없다.

```text
소모형 자원 카드
→ 현금 비용만 요구
→ 딜 사용 불가
```

딜은 주식 매수의 투자 철학 비용을 뚫는 자원으로 유지한다.

## 9. 구현 시 주의사항

```text
- 딜은 주식 예약으로 얻는다.
- 딜 최대 보유량은 3개이다.
- 딜이 3개여도 예약은 가능하다.
- 딜 한도 초과 시 추가 딜만 폐기한다.
- 딜은 모든 투자 철학 슬롯에 배치 가능하다.
- 딜 1개는 기본 현금 비용을 1 낮춘다.
- 딜 할인은 인플레이션 적용 전에 반영한다.
- 딜은 배치 시 소비되지 않고 매수 확정 시 소비된다.
- 딜은 소모형 자원 카드 결제에 사용할 수 없다.
```
