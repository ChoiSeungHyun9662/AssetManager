# 11. 딜 칩 시스템

## 1. 목적

이 문서는 딜 칩의 역할, 획득 방식, 보유 한도, 매수 결제에서의 사용 규칙, 초과 처리, UI 피드백을 정의한다.

딜 칩은 본 게임의 특수 자원이다.  
전문 자원 비용을 대체할 수 있으며, 동시에 자산 매수의 현금 비용을 낮춘다.

```text
딜 칩 1개
= 전문 자원 비용 슬롯 1칸 대체
+ 기본 현금 비용 1 감소
```

딜 칩은 강력한 자원이므로 최대 보유량을 3개로 제한한다.

```text
딜 최대 보유량: 3개
```

---

## 2. 딜 칩의 핵심 역할

딜 칩은 다음 두 기능을 동시에 가진다.

```text
1. 전문 자원 슬롯 대체
2. 기본 현금 비용 감소
```

즉, 딜 칩은 단순한 조커 자원이 아니라 매수 비용을 크게 낮추는 전략 자원이다.

예시:

```text
카드 비용:
현금 5
리서치 1
신용 1

플레이어가 딜 칩 2개 사용:
→ 리서치 슬롯 1칸 대체
→ 신용 슬롯 1칸 대체
→ 기본 현금 비용 5에서 2 감소
→ 할인 후 현금 비용 3
```

이후 인플레이션이 있다면 할인 후 비용에 적용한다.

---

## 3. 딜 칩 획득

딜 칩은 시장 카드 예약을 통해 획득한다.

```text
시장 카드 예약
→ 딜 +1
```

예약 액션 전체 처리:

```text
시장 카드 예약
→ 예약 슬롯으로 이동
→ 예약한 시장 슬롯 새 카드로 보충
→ 시장 테이프 진행
→ 딜 +1
→ 환매 압력 +1
→ 영업일 종료
```

딜은 유동성 확보로 얻을 수 없다.

```text
유동성 확보
→ 딜 선택 불가
```

---

## 4. 딜 칩 보유 한도

딜 칩은 최대 3개까지 보유할 수 있다.

```text
딜 최대 보유량: 3개
```

보유 상태 예시:

```text
딜 0 / 3
딜 1 / 3
딜 2 / 3
딜 3 / 3
```

딜은 전문 자원 한도에 포함되지 않는다.

```text
전문 자원 한도:
리서치 + 신용 + 원자재 <= 10

딜은 이 합계에 포함하지 않음
```

---

## 5. 딜 칩 한도 초과 처리

딜 칩이 이미 3개인 상태에서도 예약 액션은 가능하다.

```text
딜 3 / 3
→ 예약 가능
```

다만 예약으로 얻는 추가 딜은 한도를 초과하므로 폐기한다.

```text
딜 3 / 3
→ 예약 실행
→ 딜 +1 시도
→ 추가 딜 폐기
```

이때 안내 메시지를 표시한다.

```text
딜 칩 최대 보유: 추가 딜 폐기
```

중요한 점:

```text
딜 한도 초과는 예약 자체를 막지 않는다.
딜 보상만 폐기한다.
```

예약은 카드 보관 효과도 갖기 때문이다.

---

## 6. 딜 칩과 예약 버튼

딜 보유량은 예약 버튼 활성 조건에 영향을 주지 않는다.

```text
딜 3 / 3
→ 예약 버튼 활성 가능
```

예약 버튼 비활성화 조건:

```text
- 예약 슬롯이 가득 찬 경우
```

예약 버튼 비활성화 조건이 아닌 것:

```text
- 딜이 3개인 경우
```

예시:

```text
예약 슬롯 2 / 3
딜 3 / 3
→ 예약 버튼 활성

예약 실행
→ 카드 예약
→ 딜 +1은 폐기
→ 환매 압력 +1
```

---

## 7. 딜 칩의 매수 슬롯 배치

딜 칩은 모든 전문 자원 비용 슬롯에 배치할 수 있다.

```text
딜 칩
→ 리서치 슬롯 배치 가능
→ 신용 슬롯 배치 가능
→ 원자재 슬롯 배치 가능
```

딜 칩이 배치된 슬롯은 결제 완료 상태로 본다.

```text
딜 칩이 올라간 슬롯
= 해당 전문 자원 비용 결제 완료
```

딜 칩이 올라간 슬롯은 더 이상 빈 슬롯이 아니다.

```text
딜 칩 배치 슬롯
→ 자동 배치 대상 아님
→ 다른 칩 배치 불가
```

---

## 8. 딜 칩 현금 할인

딜 칩 1개를 매수 결제에 사용하면 기본 현금 비용을 1 낮춘다.

```text
사용한 딜 칩 수 = N
기본 현금 비용 -= N
```

최소 현금 비용은 0이다.

```text
기본 현금 비용 1
딜 칩 3개 사용
→ 할인 후 현금 비용 0
```

딜 할인은 인플레이션 적용 전에 반영한다.

```text
기본 현금 비용
→ 딜 할인
→ 인플레이션 적용
→ 최종 현금 비용
```

예시:

```text
기본 현금 비용 5
딜 칩 2개 사용
→ 할인 후 3

인플레이션 +1
→ 최종 현금 비용 4
```

---

## 9. 매수 버튼과 딜 칩

딜 칩 사용에 따라 매수 버튼의 비용 표시와 활성 상태가 달라진다.

갱신 항목:

```text
- 사용한 딜 칩 수
- 딜 할인 후 기본 현금 비용
- 인플레이션 적용 후 최종 현금 비용
- 현재 현금으로 지불 가능한지
- 모든 전문 자원 슬롯이 채워졌는지
```

딜 칩을 배치하거나 회수할 때마다 최종 현금 비용을 재계산한다.

```text
딜 칩 배치
→ 현금 비용 감소
→ 매수 버튼 상태 갱신

딜 칩 회수
→ 현금 비용 증가
→ 매수 버튼 상태 갱신
```

---

## 10. 딜 칩 소비 타이밍

딜 칩은 슬롯에 배치하는 순간 소비되지 않는다.

```text
딜 칩 배치
→ 아직 소비 아님
```

딜 칩은 매수 확정 시점에 소비된다.

```text
매수 확정
→ 배치된 딜 칩 소비
```

매수 확정 전 닫기나 취소를 하면 딜 칩은 회수된다.

```text
닫기 / 취소
→ 딜 칩 회수
→ 딜 보유량 변화 없음
```

예약 버튼을 누른 경우에도 매수는 확정되지 않았으므로 배치된 딜 칩을 회수한다.

```text
딜 칩 배치 상태
→ 예약 버튼 클릭
→ 딜 칩 자동 회수
→ 예약 실행
```

---

## 11. 딜 칩과 전문 자원 대체

딜 칩은 전문 자원 슬롯을 대체할 수 있지만, 해당 전문 자원을 실제로 획득하거나 변환하는 것은 아니다.

예시:

```text
리서치 슬롯에 딜 칩 배치
→ 리서치 비용을 낸 것으로 처리
→ 리서치 자원이 증가하거나 감소하지 않음
→ 딜 칩만 매수 확정 시 소비
```

딜 칩으로 대체한 슬롯은 어떤 자원을 대체했는지 결제 컨텍스트에 기록할 수 있다.

```text
딜 칩 1
→ 리서치 슬롯 대체

딜 칩 2
→ 신용 슬롯 대체
```

이 정보는 효과 로그, UI, 툴팁 등에 활용할 수 있다.

---

## 12. 딜 칩과 자동 배치

자동 배치 버튼은 전문 자원 칩을 우선 사용하고, 부족한 슬롯에 딜 칩을 사용할 수 있다.

기본 우선순위:

```text
1. 해당 전문 자원 칩으로 채울 수 있는 슬롯 우선
2. 부족한 슬롯은 딜 칩으로 보완
3. 딜 할인 적용 후 현금 비용 재계산
```

자동 배치가 딜 칩을 사용하더라도 매수는 자동 확정하지 않는다.

```text
자동 배치
→ 딜 칩 배치 가능
→ 매수 버튼 활성화 가능
→ 매수는 별도 클릭 필요
```

---

## 13. 딜 칩과 클릭 배치

딜 칩을 클릭하면 딜 칩이 들어갈 수 있는 빈 전문 자원 슬롯 중 가장 빠른 슬롯에 배치한다.

```text
딜 칩 클릭
→ 빈 전문 자원 슬롯 중 가장 빠른 칸에 배치
```

딜 칩이 들어갈 수 있는 슬롯:

```text
- 비어 있는 리서치 슬롯
- 비어 있는 신용 슬롯
- 비어 있는 원자재 슬롯
```

모든 슬롯이 채워져 있으면 버튼 반응은 있지만 배치는 하지 않는다.

```text
모든 슬롯 채워짐
→ 딜 칩 클릭
→ 버튼 반응만 있음
→ 변화 없음
```

---

## 14. 딜 칩과 드래그 배치

딜 칩을 드래그하면 모든 빈 전문 자원 슬롯을 하이라이트한다.

```text
딜 칩 드래그 시작
→ 빈 리서치 슬롯 하이라이트
→ 빈 신용 슬롯 하이라이트
→ 빈 원자재 슬롯 하이라이트
```

유효 슬롯에 드롭하면 배치된다.

```text
딜 칩 드롭
→ 유효 슬롯이면 배치
```

유효하지 않은 위치에 드롭하면 원위치로 돌아간다.

```text
무효 드롭
→ 딜 칩 원위치 복귀
→ 결제 상태 변화 없음
```

---

## 15. 딜 칩 회수

슬롯에 배치된 딜 칩은 매수 확정 전까지 회수할 수 있다.

회수 방식:

```text
- 슬롯 위 딜 칩 클릭
- 슬롯 밖으로 드래그
- 카드 상세보기 닫기
- 예약 버튼 클릭 시 자동 회수
```

딜 칩 회수 시 다음을 처리한다.

```text
- 슬롯 비우기
- 사용 딜 칩 수 감소
- 딜 할인 제거
- 최종 현금 비용 재계산
- 매수 버튼 상태 갱신
```

---

## 16. 딜 칩 UI 표시

딜 칩은 보유 자원 영역에 칩으로 표시한다.

표시 예시:

```text
딜 ●●○
```

또는:

```text
딜 2 / 3
```

MVP에서는 숫자와 칩 표현 중 구현이 쉬운 방식을 선택할 수 있다.  
다만 딜은 실제 결제 슬롯에 배치되는 자원이므로, 카드 상세보기에서는 칩 오브젝트로 조작 가능해야 한다.

---

## 17. 딜 칩 관련 메시지

딜 칩 관련 대표 메시지는 다음과 같다.

```text
딜 칩 최대 보유: 추가 딜 폐기
```

딜 칩을 배치할 수 없는 경우:

```text
놓을 수 있는 슬롯이 없습니다.
```

현금 할인 툴팁 예시:

```text
기본 현금 비용 5
딜 할인 -2
인플레이션 +1
최종 현금 비용 4
```

툴팁 안에 별도의 “자세히 보기” 버튼은 넣지 않는다.

## 18. 딜 칩 처리 순서

### 18.1 예약으로 딜 획득

```text
예약 버튼 클릭
→ 카드 예약 처리
→ 시장 테이프 진행
→ 딜 +1 시도
→ 딜 한도 검사
→ 한도 초과 시 추가 딜 폐기 메시지
→ 환매 압력 +1
→ 영업일 종료
```

딜 획득과 환매 압력 증가의 실제 순서는 구현상 조정 가능하지만, 결과는 동일해야 한다.

단, 환매 압력 증가 후 한도 도달 시 런 실패가 발생할 수 있으므로, 환매 압력 처리 이후에는 실패 여부를 검사해야 한다.

---

### 18.2 매수에서 딜 사용

```text
카드 상세보기
→ 딜 칩을 전문 자원 슬롯에 배치
→ 최종 현금 비용 재계산
→ 매수 버튼 클릭
→ 결제 검증
→ 딜 칩 소비
→ 현금 차감
→ 자산 획득
```

## 20. 함수 예시

### 20.1 딜 추가

```csharp
void AddDealFromReservation()
{
    if (CurrentDeal >= MaxDeal)
    {
        ShowShortFeedback("딜 칩 최대 보유: 추가 딜 폐기");
        return;
    }

    CurrentDeal += 1;
}
```

---

### 20.2 딜 사용 수 계산

```csharp
int GetUsedDealCount(PurchasePaymentState state)
{
    return state.PaymentSlots.Count(slot => slot.IsFilledByDeal);
}
```

---

### 20.3 딜 할인 적용

```csharp
int ApplyDealDiscount(int baseCashCost, int usedDealCount)
{
    return Math.Max(0, baseCashCost - usedDealCount);
}
```

---

### 20.4 최종 현금 비용 계산

```csharp
int CalculateFinalCashCost(PurchasePaymentState state)
{
    int costAfterDealDiscount =
        ApplyDealDiscount(state.BaseCashCost, GetUsedDealCount(state));

    return ApplyInflationToCashCost(costAfterDealDiscount);
}
```

---

### 20.5 딜 칩 배치 가능 판정

```csharp
bool CanPlaceDealChipInSlot(PaymentSlotState slot)
{
    if (slot.IsFilled)
        return false;

    return IsProfessionalResource(slot.RequiredResourceType);
}
```

## 21. 구현 시 주의사항

```text
- 딜은 유동성 확보로 얻을 수 없다.
- 딜은 예약 액션으로 얻는다.
- 딜 최대 보유량은 3개이다.
- 딜이 3개여도 예약은 가능하다.
- 딜 한도 초과 시 추가 딜만 폐기한다.
- 딜 한도 초과 시 메시지를 표시한다.
- 딜 칩은 모든 전문 자원 슬롯에 배치 가능하다.
- 딜 칩이 배치된 슬롯은 결제 완료 상태이다.
- 딜 칩 1개는 기본 현금 비용을 1 낮춘다.
- 딜 할인은 인플레이션 적용 전에 반영한다.
- 딜 칩은 배치 시 소비되지 않는다.
- 딜 칩은 매수 확정 시 소비된다.
- 딜 칩 회수 시 최종 현금 비용을 반드시 재계산한다.
- 자동 배치가 딜을 사용해도 매수는 자동 확정하지 않는다.
```
