# 21. MVP 범위

## 1. 목적

이 문서는 자산 운용 로그라이트 게임의 MVP 구현 범위를 정의한다.

MVP의 목적은 모든 시스템을 화려하게 완성하는 것이 아니다.
핵심 목표는 게임이 처음부터 끝까지 규칙적으로 진행되고, 플레이어가 한 런을 완료하거나 실패할 수 있는 상태를 만드는 것이다.

MVP에서 가장 중요한 것은 다음이다.

```text
- 영업일이 진행된다.
- 분기가 진행된다.
- 회계년도가 진행된다.
- 시장 카드가 표시된다.
- 자산을 매수할 수 있다.
- 카드를 예약할 수 있다.
- 자원을 확보할 수 있다.
- 분기 목표를 판정할 수 있다.
- 환매 압력으로 실패할 수 있다.
- 최종 정산이 표시된다.
```

---

## 2. MVP 핵심 기준

MVP 포함 여부는 다음 기준으로 판단한다.

```text
1. 게임의 기본 루프가 작동하는 데 필요한가?
2. 자산 매수와 성장 판단에 필요한가?
3. 분기 목표와 환매 압력 구조에 필요한가?
4. 런 실패 또는 최종 정산에 필요한가?
5. Codex가 구현 가능한 독립 시스템으로 분리할 수 있는가?
```

위 기준을 만족하는 기능은 MVP에 포함한다.

반대로 다음 조건에 해당하는 기능은 후순위로 둘 수 있다.

```text
- 없어도 한 런을 끝까지 진행할 수 있음
- 핵심 판단보다 연출 완성도에 가까움
- 밸런스 튜닝 후 결정해도 됨
- 구현 복잡도 대비 MVP 검증 가치가 낮음
```

## 3. MVP 포함 시스템 요약

MVP에 포함해야 하는 시스템은 다음과 같다.

```text
1. 시간 구조
2. 영업일 흐름
3. 시장 영역 3상태
4. 시장 테이프
5. 자원 구조
6. 자산 카드
7. 카드 상세보기
8. 자산 매수
9. 인플레이션
10. 칩 조작 기본 UX
11. 딜
12. 예약 시스템
13. 자원 확보
14. 추가 매수권
15. 분기 마감
16. 환매 압력
17. 4Q 휴가
18. 최종 정산
19. 데이터 테이블
```

## 4. 시간 구조

MVP에 포함한다.

필수 구현:

```text
- 영업일
- 분기
- 회계년도
- 1·2회계년도 4Q 휴가
- 3회계년도 4Q 플레이
- 총 10개 플레이 분기
- 총 44영업일
```

확정 구조:

```text
1회계년도: 1Q~3Q 플레이, 4Q 휴가
2회계년도: 1Q~3Q 플레이, 4Q 휴가
3회계년도: 1Q~4Q 플레이
```

분기당 영업일:

```text
1회계년도: 분기당 4영업일
2회계년도: 분기당 4영업일
3회계년도: 분기당 5영업일
```

---

## 5. 영업일 흐름

MVP에 포함한다.

필수 구현:

```text
- 영업일 시작
- 보유 자산의 영업일 시작 현금 발생
- 시장 상태 진입
- 행동 확정
- 영업일 종료
- 다음 영업일 시작
- 마지막 영업일 후 분기 마감 진입
```

중요 규칙:

```text
- 카드 보기와 닫기는 영업일을 소비하지 않는다.
- 매수 확정은 영업일을 소비한다.
- 예약 버튼 클릭은 영업일을 소비한다.
- 자원 확보 첫 자원 획득 후 완료 시 영업일을 소비한다.
- 다음 영업일 클릭은 영업일을 소비한다.
```

## 6. 시장 영역 3상태

MVP에 포함한다.

시장 영역 상태:

```text
1. 시장
2. 카드 상세보기
3. 자원 확보
```

필수 구현:

```text
- Market 상태
- CardDetail 상태
- GainLiquidity 상태
- 상태 전환
- 다음 영업일 버튼 활성 조건
```

중요 규칙:

```text
- 카드 상세보기와 자원 확보는 시장 영역을 대체한다.
- 다음 영업일은 Market 상태에서만 활성화된다.
- CardDetail / GainLiquidity 상태에서는 다음 영업일을 비활성화한다.
```

## 7. 시장 테이프

MVP에 포함한다.

필수 구현:

```text
- 매도 임박 카드
- 현재 시장 카드
- 예비 시장 카드
- 시장 테이프 진행
- 시장 테이프 갱신
- 슬롯 보충
```

처리 규칙:

```text
시장 카드 매수
→ 산 자리만 보충
→ 시장 테이프 진행 없음

시장 카드 예약
→ 예약한 자리 보충
→ 시장 테이프 진행

분기 시작
→ 같은 회계년도 내 다음 분기면 시장 테이프 진행

회계년도 시작
→ 시장 테이프 갱신
```

MVP에서 시장 테이프 슬롯 수는 데이터로 조정 가능하게 둔다.

## 8. 자원 구조

MVP에 포함한다.

필수 자원:

```text
- 현금
- 리서치
- 신용
- 원자재
- 딜
```

필수 규칙:

```text
리서치 + 신용 + 원자재 <= 10
딜 <= 3
```

전문 자원 한도 초과 규칙:

```text
기존 보유 자원은 건드리지 않는다.
신규 획득분 중 초과분만 폐기한다.
```

현금 구분:

```text
자원 확보로 얻은 현금 = 조달 현금
보유 자산의 영업일 시작 현금 / 분기 마감 정산 수익 = 운용 수익
```

---

## 9. 자산 카드

MVP에 포함한다.

필수 카드 필드:

```text
- 카드 ID
- 카드 이름
- 카드 이미지 또는 플레이스홀더
- 현금 비용
- 전문 자원 비용
- 운용가치
- 운용 수익
- 태그
- 희귀도
```

필수 규칙:

```text
- 운용가치는 보유 자산 카드의 고유 점수이다.
- AUM이라는 용어는 사용하지 않는다.
- 예약 카드는 운용가치에 포함하지 않는다.
- 시장 카드는 운용가치에 포함하지 않는다.
- 보유 자산만 최종 운용가치에 포함한다.
```

## 10. 카드 상세보기

MVP에 포함한다.

필수 구현:

```text
- 시장 카드 클릭 시 상세보기 진입
- 예약 카드 클릭 시 상세보기 진입
- 닫기
- 매수 버튼
- 예약 버튼
- 비용 표시
- 비용 슬롯 표시
```

필수 규칙:

```text
- 닫기는 항상 시장 상태로 복귀한다.
- 닫기는 영업일을 소비하지 않는다.
- 예약 버튼은 시장 카드 상세보기에서만 표시한다.
- 예약 카드 상세보기에서는 예약 버튼을 숨긴다.
- 예약 구역이 가득 차면 예약 버튼을 비활성화한다.
```

## 11. 자산 매수

MVP에 포함한다.

필수 구현:

```text
- 매수 조건 검증
- 전문 자원 슬롯 검증
- 딜 적용
- 최종 현금 비용 계산
- 인플레이션 적용
- 자원 소비
- 현금 차감
- 자산 획득
- 시장 카드 매수 후 산 자리 보충
- 예약 카드 매수 후 예약 구역 비움
```

필수 규칙:

```text
- 매수 확정 전에는 자원을 소비하지 않는다.
- 매수 확정 시 자원을 소비한다.
- 예약 카드 매수도 일반 자산 매수로 판정한다.
- 매수 출처는 MarketTape / Reserved로 기록한다.
- 최종 현금 비용은 비용 수정 효과, 딜 할인, 인플레이션 순서로 계산한다.
- 인플레이션은 전문 자원 비용이 아니라 현금 비용에만 적용한다.
```

---

### 11.1 인플레이션

MVP에 포함한다.
단, 시장 뉴스나 복잡한 경제 이벤트가 아니라 매수 현금 비용에 적용되는 최소 규칙으로 둔다.

필수 구현:

```text
- 현재 인플레이션 값
- 카드 상세보기 또는 비용 표시 영역의 인플레이션 반영 비용
- 최종 현금 비용 계산 시 인플레이션 적용
- 인플레이션 기본값과 분기별 값을 데이터로 조정 가능하게 구성
```

필수 규칙:

```text
- MVP 인플레이션은 0 이상의 정수 가산값이다.
- 기본값은 0이며, 0일 때는 기존 매수 비용과 동일하다.
- 현재 인플레이션이 +N이면 모든 자산 매수의 현금 비용에 +N을 더한다.
- 인플레이션은 비용 수정 효과와 딜 할인이 끝난 뒤 적용한다.
- 예약 카드는 예약 시점이 아니라 실제 매수 시점의 인플레이션을 적용한다.
- 인플레이션 적용 후 최종 현금 비용은 0보다 작아질 수 없다.
- MVP에서는 분기 시작 시 현재 분기의 인플레이션 값을 정하고, 해당 분기 중에는 변하지 않는다.
```

예시:

```text
기본 현금 비용 5
딜 2개 사용
인플레이션 +1
→ 최종 현금 비용 4
```

## 12. 칩 조작 UX

MVP에 포함한다.
다만 고급 연출은 후순위로 둘 수 있다.

MVP 필수:

```text
- 칩 클릭 배치
- 칩 회수
- 비용 슬롯 표시
- 딜 배치
- 매수 버튼 상태 갱신
```

MVP 가능하면 포함:

```text
- 드래그 배치
- 하이라이트
- 자동 배치 버튼
```

후순위 가능:

```text
- 고급 칩 애니메이션
- 세밀한 물리감
- 화려한 사운드
- 복잡한 딤드 연출
```

중요 규칙:

```text
- 칩을 올려도 자원은 아직 소비되지 않는다.
- 자원 소비는 매수 확정 시점에만 발생한다.
- 딜은 모든 전문 자원 슬롯에 배치 가능하다.
- 딜은 기본 현금 비용을 1 낮춘다.
```

## 13. 딜

MVP에 포함한다.

필수 구현:

```text
- 예약 시 딜 +1
- 딜 최대 3개
- 딜로 전문 자원 슬롯 대체
- 딜 사용 시 현금 비용 감소
- 딜 한도 초과 시 추가 딜 폐기
```

필수 메시지:

```text
딜 최대 보유: 추가 딜 폐기
```

중요 규칙:

```text
- 딜이 3개여도 예약은 가능하다.
- 초과 딜만 폐기한다.
- 딜은 자원 확보로 얻을 수 없다.
```

---

## 14. 예약 시스템

MVP에 포함한다.

필수 구현:

```text
- 예약 구역 3개
- 시장 카드 예약
- 예약 카드 유지
- 예약 카드 매수
- 예약 구역 가득 참 처리
- 예약 시 딜 +1
- 예약 시 환매 압력 +1
- 예약 시 시장 테이프 진행
```

중요 규칙:

```text
- 예약 확인 상태는 없다.
- 예약 버튼 클릭은 즉시 예약 액션이다.
- 예약 카드는 분기와 회계년도 전환 후에도 유지된다.
- 예약 카드는 보유 자산이 아니다.
```

## 15. 자원 확보

MVP에 포함한다.

필수 구현:

```text
- 중앙 은행 클릭
- 자원 확보 화면
- 현금 선택
- 리서치 선택
- 신용 선택
- 원자재 선택
- 첫 자원 획득 전 닫기
- 첫 자원 획득 후 닫기 불가
- 같은 자원 2개 획득
- 서로 다른 자원 3개 획득
- 전문 자원 한도에 따른 비활성화
- 선택지 소멸 시 자동 종료
```

필수 메시지:

```text
자원칩 최대 한도 도달
```

중요 규칙:

```text
- 조달 현금은 운용 수익에 포함하지 않는다.
- 딜은 자원 확보로 얻을 수 없다.
```

## 16. 추가 매수권

MVP에 포함한다.
단, 추가 매수권을 주는 카드가 MVP 카드 풀에 없으면 구현 우선순위를 낮출 수 있다.

필수 규칙:

```text
- 자산 매수 전용
- 시장 카드 매수 가능
- 예약 카드 매수 가능
- 자원 확보 불가
- 예약 불가
- 다음 영업일로 이월 없음
- 중첩 없음
- 추가 매수 후 영업일 종료
```

중요 규칙:

```text
추가 매수로도 매수 규칙은 발동한다.
단, GrantExtraBuyAction만 무시한다.
```

---

## 17. 분기 마감

MVP에 포함한다.

필수 구현:

```text
- 마지막 영업일 종료 후 분기 마감 진입
- 분기 마감 정산 수익 계산
- 분기 운용 수익 확정
- 분기 목표 달성률 계산
- 환매 압력 증가량 계산
- 분기 정산 UI 표시
```

정산 애니메이션은 MVP에서 단순화 가능하다.

MVP 최소 형태:

```text
태그별 정산 결과를 순서대로 텍스트/숫자로 표시
```

후순위:

```text
화려한 태그별 애니메이션
카드별 기여도 연출
고급 스킵 연출
```

## 18. 환매 압력

MVP에 포함한다.

필수 구현:

```text
- 예약 시 환매 압력 +1
- 분기 목표 미달 시 달성률별 환매 압력 증가
- 환매 압력 한도 10
- 10 이상 도달 즉시 런 실패
```

분기 목표 미달 규칙:

```text
목표의 50% 미만
→ 환매 압력 +3

목표의 75% 미만
→ 환매 압력 +2

목표의 100% 미만
→ 환매 압력 +1
```

중요 규칙:

```text
환매 압력 증가 후 즉시 한도 검사
```

## 19. 4Q 휴가

MVP에 포함한다.

필수 구현:

```text
- 1회계년도 3Q 후 4Q 휴가 화면

- 다음 회계년도 시작 버튼

- 보유 자산 수

```

- 4Q 휴가에서는 패널티 없음
- 4Q 휴가에서는 시장 테이프 처리 없음
- 다음 회계년도 시작 시 시장 테이프 갱신

## 25. 1차 개발 완료 판정

1차 개발 완료 기준은 다음과 같다.

```text
- 새 런을 시작할 수 있다.
- 1회계년도 1Q부터 진행할 수 있다.
- 영업일이 정상적으로 감소한다.
- 자산을 매수할 수 있다.
- 인플레이션이 매수 현금 비용에 반영된다.
- 보유 자산의 영업일 시작 현금이 발생한다.
- 자원 확보로 자원을 얻을 수 있다.
- 분기 마감이 발생한다.
- 분기 목표를 판정한다.
- 환매 압력이 증가한다.
- 환매 압력 10 이상이면 실패한다.
- 3회계년도 4Q 후 최종 정산이 표시된다.
```

예약과 딜을 포함한 1차 완료 기준:

```text
- 시장 카드를 예약할 수 있다.
- 예약 카드가 유지된다.
- 예약 카드 매수가 가능하다.
- 예약 시 딜이 증가한다.
- 딜이 매수 비용을 대체/할인하고, 할인 후 인플레이션이 적용된다.
- 예약 시 시장 테이프가 진행된다.
```

## 26. MVP 테스트 시나리오

MVP 검증을 위한 기본 테스트 시나리오는 다음과 같다.

### 26.1 기본 진행 테스트

```text
새 런 시작
→ 1회계년도 1Q 시작
→ 다음 영업일 버튼으로 4일 진행
→ 분기 마감 진입
```

---

### 26.2 자산 매수 테스트

```text
시장 카드 클릭
→ 카드 상세보기
→ 칩 배치
→ 매수 확정
→ 자산 보유
→ 다음 영업일 시작
→ 운용 수익 발생

기본 현금 비용 5
→ 딜 2개 배치
→ 인플레이션 +1 적용
→ 최종 현금 비용 4
```

---

### 26.3 자원 확보 테스트

```text
중앙 은행 클릭
→ 자원 확보 화면
→ 현금 선택
→ 현금 선택
→ 현금 +2
→ 영업일 종료
```

---

### 26.4 예약 테스트

```text
시장 카드 클릭
→ 카드 상세보기
→ 예약 버튼 클릭
→ 예약 구역에 카드 추가
→ 딜 +1
→ 환매 압력 +1
→ 시장 테이프 진행
→ 영업일 종료
```

---

### 26.5 환매 압력 실패 테스트

```text
환매 압력 9 상태
→ 시장 카드 예약
→ 환매 압력 +1
→ 환매 압력 10
→ 런 실패
```

---

### 26.6 최종 정산 테스트

```text
3회계년도 4Q까지 완료
→ 환매 압력 10 미만
→ 최종 정산 표시
→ 최종 운용가치 / 평가 / 총 운용 수익 표시
```

## 27. MVP 구현 시 주의사항

```text
- MVP에서는 화려함보다 규칙 정확성을 우선한다.
- UI 위치와 아트 완성도는 후순위다.
- 데이터와 상태 전환이 먼저 안정되어야 한다.
- 자원 소비는 매수 확정 시점에만 발생해야 한다.
- 인플레이션은 딜 할인 후 현금 비용에만 적용해야 한다.
- 조달 현금은 운용 수익에 포함하면 안 된다.
- 환매 압력은 증가 즉시 한도 검사를 해야 한다.
- 예약 카드와 보유 자산을 절대 혼동하면 안 된다.
- 시장 테이프 진행과 갱신을 구분해야 한다.
- AUM이라는 용어와 필드명을 사용하지 않는다.
```
