# 07. 시장 테이프

## 1. 목적

이 문서는 시장 테이프의 구조와 처리 규칙을 정의한다.

시장 테이프는 자산 카드가 시간에 따라 흘러가는 시장 표시 구조이다.  
플레이어는 시장 테이프에서 카드를 확인하고, 매수하거나 예약할 수 있다.

시장 테이프는 단순한 카드 목록이 아니라, 다음 세 구역으로 구성된 흐름 구조이다.

```text
매도 임박 카드
현재 시장 카드
예비 시장 카드
```

각 구역은 시장에서 카드가 얼마나 임박한 상태인지, 또는 앞으로 시장에 나올 예정인지를 나타낸다.

---

## 2. 시장 테이프 구역

시장 테이프는 다음 3개 구역을 가진다.

```text
1. 매도 임박
2. 현재 시장
3. 예비 시장
```

---

### 2.1 매도 임박

매도 임박 구역은 시장에서 곧 사라질 카드들을 표시한다.

시장 테이프 진행 시 매도 임박 카드는 제거된다.

```text
시장 테이프 진행
→ 매도 임박 카드 제거
```

매도 임박 카드는 플레이어가 매수할 수 있는 카드이다.  
단, 다음 시장 테이프 진행이 발생하면 사라진다.

---

### 2.2 현재 시장

현재 시장 구역은 기본적으로 플레이어가 가장 직접적으로 매수/예약을 고려하는 카드 영역이다.

현재 시장 카드는 다음 행동의 대상이 될 수 있다.

```text
- 카드 상세보기
- 매수
- 예약
```

시장 카드 매수 시에는 산 자리만 새 카드로 보충한다.

```text
현재 시장 카드 매수
→ 해당 슬롯만 새 카드로 보충
→ 시장 테이프 진행 없음
```

시장 카드 예약 시에는 예약한 자리를 먼저 보충한 뒤 시장 테이프가 진행된다.

---

### 2.3 예비 시장

예비 시장 구역은 다음 시장 흐름에서 현재 시장으로 이동할 카드들을 보여준다.

시장 테이프 진행 시 예비 시장 카드는 현재 시장으로 이동한다.

```text
예비 시장 카드
→ 현재 시장으로 이동
```

예비 시장이 비면 즉시 새 카드로 보충한다.

```text
예비 시장 빈 슬롯
→ 새 카드로 즉시 보충
```

---

## 3. 시장 카드 슬롯 수

시장 테이프의 각 구역은 동일한 수의 슬롯을 가진다.

```text
매도 임박: N장
현재 시장: N장
예비 시장: N장
```

N의 정확한 값은 UI와 밸런스 단계에서 조정 가능하다.  
MVP에서는 테이블 또는 상수로 관리한다.

```csharp
public int MarketSlotCountPerZone;
```

중요한 점은 시장 테이프 진행 후에도 세 구역이 항상 채워진 상태를 유지한다는 것이다.

```text
시장 테이프는 항상 꽉 찬 상태를 유지한다.
```

---

## 4. 시장 테이프 용어

시장 테이프 관련 용어는 정확히 구분한다.

| 용어 | 의미 |
|---|---|
| 시장 테이프 진행 | 카드들이 한 단계씩 밀려나는 처리 |
| 시장 테이프 갱신 | 기존 시장을 전부 제거하고 새 카드로 재구성하는 처리 |
| 슬롯 보충 | 특정 빈 슬롯에 새 카드 1장을 뽑아 채우는 처리 |

이전 표현인 `시장 테이프 전체 갱신`은 사용하지 않는다.  
전체를 새로 갈아엎는 처리는 `시장 테이프 갱신`이라고 부른다.  
카드가 한 단계씩 이동하는 처리는 `시장 테이프 진행`이라고 부른다.

---

## 5. 시장 테이프 진행

시장 테이프 진행은 시장 카드가 한 단계씩 시간에 따라 밀려나는 처리이다.

처리 순서:

```text
1. 매도 임박 카드들을 제거한다.
2. 현재 시장 카드들을 매도 임박 카드로 이동한다.
3. 예비 시장 카드들을 현재 시장 카드로 이동한다.
4. 비어 있는 예비 시장 슬롯을 새 카드로 전부 보충한다.
```

구조 예시:

```text
진행 전:
매도 임박: [A] [B] [C]
현재 시장: [D] [E] [F]
예비 시장: [G] [H] [I]

시장 테이프 진행 후:
매도 임박: [D] [E] [F]
현재 시장: [G] [H] [I]
예비 시장: [J] [K] [L]
```

---

## 6. 시장 테이프 갱신

시장 테이프 갱신은 시장 전체를 새로 구성하는 처리이다.

처리 순서:

```text
1. 기존 시장 테이프 카드들을 전부 제거한다.
2. 매도 임박 슬롯을 새 카드로 채운다.
3. 현재 시장 슬롯을 새 카드로 채운다.
4. 예비 시장 슬롯을 새 카드로 채운다.
```

시장 테이프 갱신은 회계년도 시작 시 발생한다.

```text
회계년도 시작
→ 시장 테이프 갱신
```

예약 카드는 시장 테이프 갱신의 영향을 받지 않는다.

```text
시장 테이프 갱신
→ 시장 카드만 교체
→ 예약 슬롯은 유지
```

---

## 7. 슬롯 보충

슬롯 보충은 특정 빈 시장 슬롯에 새 카드 1장을 채우는 처리이다.

발생 상황:

```text
- 시장 카드 매수 후 산 자리 보충
- 시장 카드 예약 후 예약한 자리 보충
- 시장 테이프 진행 후 예비 시장 빈 슬롯 보충
```

슬롯 보충은 시장 테이프 진행과 다르다.  
슬롯 보충은 빈 자리 하나를 채우는 국소 처리이다.

---

## 8. 시장 카드 매수 시 처리

시장 카드 매수 시에는 산 자리만 새 카드로 보충한다.

```text
시장 카드 매수
→ 결제 검증
→ 자원 소비
→ 자산 획득
→ 산 시장 슬롯을 새 카드로 보충
→ 시장 테이프 진행 없음
```

예시:

```text
현재 시장:
[A] [B] [C]

B 매수
→ B가 보유 자산으로 이동
→ B 자리 새 카드 D 보충

현재 시장:
[A] [D] [C]
```

시장 카드 매수는 시장 전체를 흐르게 하지 않는다.  
매수는 현재 시장에서 하나를 산 뒤 그 자리만 채우는 행동이다.

---

## 9. 예약 카드 매수 시 처리

예약 카드 매수 시에는 예약 슬롯만 비운다.

```text
예약 카드 매수
→ 결제 검증
→ 자원 소비
→ 자산 획득
→ 해당 예약 슬롯 비움
→ 시장 테이프 변화 없음
```

예약 카드 매수는 시장 테이프에 영향을 주지 않는다.

```text
예약 카드 매수
→ 시장 슬롯 보충 없음
→ 시장 테이프 진행 없음
```

예약 카드는 이미 시장에서 빠져나온 카드이기 때문이다.

---

## 10. 시장 카드 예약 시 처리

시장 카드 예약 시에는 다음 순서로 처리한다.

```text
시장 카드 예약
→ 선택한 시장 카드가 예약 슬롯으로 이동
→ 예약으로 비워진 시장 슬롯에 새 카드 1장 보충
→ 시장 테이프 진행
→ 딜 +1
→ 환매 압력 +1
→ 영업일 종료
```

이 순서에서 중요한 점은 다음이다.

```text
예약한 자리 보충
→ 그다음 시장 테이프 진행
```

즉, 예약으로 빈 자리를 먼저 새 카드로 채운 뒤, 시장 전체가 한 단계 진행된다.

---

## 11. 시장 카드 예약 예시

예약 전 시장 상태:

```text
매도 임박: [A] [B] [C]
현재 시장: [D] [E] [F]
예비 시장: [G] [H] [I]
```

플레이어가 현재 시장의 `E`를 예약한다.

1단계: `E`가 예약 슬롯으로 이동한다.

```text
예약 슬롯: [E]
```

2단계: `E`가 빠진 자리에 새 카드 `J`를 보충한다.

```text
매도 임박: [A] [B] [C]
현재 시장: [D] [J] [F]
예비 시장: [G] [H] [I]
```

3단계: 시장 테이프가 진행된다.

```text
매도 임박 [A] [B] [C] 제거
현재 시장 [D] [J] [F] → 매도 임박으로 이동
예비 시장 [G] [H] [I] → 현재 시장으로 이동
예비 시장 빈 슬롯 → 새 카드 보충
```

최종 상태:

```text
매도 임박: [D] [J] [F]
현재 시장: [G] [H] [I]
예비 시장: [K] [L] [M]
예약 슬롯: [E]
```

---

## 12. 분기 시작 시 시장 처리

분기 시작 시 시장 처리 방식은 현재 분기가 어떤 분기인지에 따라 다르다.

### 12.1 회계년도 시작 1Q

회계년도 시작 시에는 시장 테이프를 갱신한다.

```text
회계년도 시작
→ 시장 테이프 갱신
→ 1Q 시작
```

적용 시점:

```text
1회계년도 시작
2회계년도 시작
3회계년도 시작
```

시장 테이프 갱신은 시장 전체를 새로 구성한다.

---

### 12.2 같은 회계년도 내 다음 분기

같은 회계년도 안에서 다음 분기로 넘어갈 때는 시장 테이프를 진행한다.

```text
1Q 마감
→ 2Q 시작
→ 시장 테이프 진행
```

```text
2Q 마감
→ 3Q 시작
→ 시장 테이프 진행
```

3회계년도에서는 3Q 이후 4Q 시작 시에도 시장 테이프를 진행한다.

```text
3회계년도 3Q 마감
→ 4Q 시작
→ 시장 테이프 진행
```

---

## 13. 회계년도 전환 시 시장 처리

1·2회계년도는 3Q 후 4Q 휴가 화면을 표시한다.  
휴가 이후 다음 회계년도 시작 시 시장 테이프를 갱신한다.

```text
1회계년도 3Q 마감
→ 1회계년도 4Q 휴가
→ 2회계년도 시작
→ 시장 테이프 갱신
```

```text
2회계년도 3Q 마감
→ 2회계년도 4Q 휴가
→ 3회계년도 시작
→ 시장 테이프 갱신
```

휴가 분기 자체에서는 별도 시장 처리를 하지 않는다.

```text
4Q 휴가
→ 시장 테이프 진행 없음
→ 시장 테이프 갱신 없음
```

---

## 14. 예약 카드와 시장 테이프

예약 카드는 시장 테이프와 별도로 유지된다.

```text
예약 카드
→ 분기 전환 후에도 유지
→ 회계년도 전환 후에도 유지
```

시장 테이프 진행이나 갱신은 예약 슬롯에 영향을 주지 않는다.

```text
시장 테이프 진행
→ 예약 슬롯 유지

시장 테이프 갱신
→ 예약 슬롯 유지
```

예약 카드는 플레이어가 매수할 때까지 남아 있다.

---

## 15. 카드 제거 처리

시장 테이프 진행 시 매도 임박 카드는 제거된다.

```text
시장 테이프 진행
→ 매도 임박 카드 제거
```

제거된 카드는 더 이상 시장에 표시되지 않는다.

구현상 제거된 카드는 다음 상태로 바꿀 수 있다.

```csharp
AssetCardRuntimeState.Removed
```

또는 덱 재순환 구조가 있다면 별도 discard pile로 보낼 수 있다.  
MVP에서는 `Removed` 또는 `Discarded` 상태로 관리하면 충분하다.

---

## 16. 카드 중복 출현 방지

보유 중인 자산은 시장에 다시 등장하지 않는다.

```text
보유 자산
→ 시장 재등장 불가
```

예약 중인 카드도 시장에 다시 등장하지 않는다.

```text
예약 카드
→ 시장 재등장 불가
```

시장 테이프에 이미 존재하는 카드도 중복으로 뽑지 않는다.

```text
시장 테이프에 있는 카드
→ 새 카드 보충 후보에서 제외
```

새 카드를 뽑을 때 제외 대상:

```text
- 보유 자산
- 예약 카드
- 현재 시장 테이프에 있는 카드
- 제거된 카드
```

덱 순환을 허용할지 여부는 카드 풀 설계 단계에서 따로 정할 수 있다.  
MVP에서는 중복 출현을 막는 방향으로 구현한다.

---

## 17. 시장 테이프와 영업일

시장 테이프는 매 영업일마다 자동으로 진행되지 않는다.

시장 테이프가 진행되는 상황은 제한적이다.

```text
시장 테이프 진행 발생:
- 예약 액션 실행 후
- 같은 회계년도 내 다음 분기 시작 시
```

시장 테이프 갱신 발생:

```text
- 회계년도 시작 시
```

시장 카드 매수는 시장 테이프를 진행시키지 않는다.

```text
시장 카드 매수
→ 산 자리만 보충
→ 시장 테이프 진행 없음
```

다음 영업일 버튼도 시장 테이프를 진행시키지 않는다.

```text
다음 영업일
→ 영업일만 종료
→ 시장 테이프 변화 없음
```

---

## 18. 시장 테이프와 카드 상세보기

시장 카드 또는 예약 카드를 클릭하면 카드 상세보기 상태로 전환한다.

```text
시장 카드 클릭
→ 카드 상세보기

예약 카드 클릭
→ 카드 상세보기
```

카드 상세보기는 시장 영역을 대체한다.

카드 상세보기에서 닫기를 누르면 시장 상태로 돌아간다.

```text
카드 상세보기
→ 닫기
→ 시장 상태
```

카드 상세보기 진입과 닫기는 시장 테이프에 영향을 주지 않는다.

```text
카드 보기
→ 시장 테이프 변화 없음

카드 닫기
→ 시장 테이프 변화 없음
```

## 19. 시장 테이프와 추가 매수권

추가 매수권이 있는 상태에서도 시장 테이프 규칙은 동일하다.

첫 번째 매수가 시장 카드 매수라면 산 자리를 즉시 보충한다.

```text
첫 번째 시장 카드 매수
→ 산 자리 보충
→ 추가 매수권 발생 가능
```

보충된 새 카드도 추가 매수 대상으로 선택할 수 있다.

```text
산 자리 새 카드 보충
→ 추가 매수권 상태
→ 새 카드도 매수 가능
```

추가 매수로 시장 카드를 매수한 경우에도 산 자리만 보충한다.

```text
추가 매수로 시장 카드 매수
→ 산 자리 보충
→ 영업일 종료
```

추가 매수권 상태에서는 예약을 할 수 없다.

```text
추가 매수권 보유 중
→ 자산 매수 가능
→ 다음 영업일 가능
→ 예약 불가
→ 유동성 확보 불가
```

## 22. 새 카드 뽑기

새 카드 보충은 시장에 나올 수 있는 카드 풀에서 1장을 선택한다.

후보에서 제외할 카드:

```text
- 이미 보유한 카드
- 예약 중인 카드
- 시장 테이프에 이미 있는 카드
- 제거된 카드
```

기본 함수 예시:

```csharp
AssetCardRuntimeData DrawNewMarketCard()
{
    List<AssetCardRuntimeData> candidates =
        GetAvailableMarketCardCandidates();

    if (candidates.Count == 0)
    {
        return null;
    }

    return SelectCardFromCandidates(candidates);
}
```

카드가 부족한 경우의 처리 방식은 카드 풀 설계에서 별도 보강한다.  
MVP에서는 충분한 카드 풀을 전제로 한다.

## 23. 처리 타이밍 요약

| 상황 | 시장 처리 |
|---|---|
| 회계년도 시작 | 시장 테이프 갱신 |
| 같은 회계년도 내 다음 분기 시작 | 시장 테이프 진행 |
| 시장 카드 매수 | 산 자리만 보충 |
| 예약 카드 매수 | 예약 슬롯만 비움 |
| 시장 카드 예약 | 예약한 자리 보충 후 시장 테이프 진행 |
| 다음 영업일 | 시장 변화 없음 |
| 카드 상세보기 닫기 | 시장 변화 없음 |
| 유동성 확보 | 시장 변화 없음 |
| 1·2회계년도 4Q 휴가 | 시장 변화 없음 |

---

## 24. 구현 시 주의사항

```text
- 시장 카드 매수는 시장 테이프를 진행시키지 않는다.
- 예약 카드 매수는 시장 테이프에 영향을 주지 않는다.
- 시장 카드 예약은 예약한 자리를 먼저 보충한 뒤 시장 테이프를 진행한다.
- 시장 테이프 진행 후 예비 시장은 즉시 새 카드로 전부 보충한다.
- 회계년도 시작 시에는 시장 테이프를 갱신한다.
- 같은 회계년도 내 다음 분기 시작 시에는 시장 테이프를 진행한다.
- 4Q 휴가에서는 시장 테이프 처리를 하지 않는다.
- 예약 카드는 시장 테이프 진행/갱신의 영향을 받지 않는다.
- 보유 자산과 예약 카드는 새 카드 보충 후보에서 제외한다.
- 시장 테이프에 이미 있는 카드도 중복으로 뽑지 않는다.
```
