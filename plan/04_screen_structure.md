# 04. 화면 구조

## 1. 목적

이 문서는 MVP 기준 화면 구조와 주요 UI 상태를 정의한다.

본 게임은 별도의 액션 선택창을 사용하지 않는다.
플레이어는 보드 위 요소를 직접 조작하며, 해당 조작이 행동으로 확정되면 영업일을 소비한다.

핵심 화면 구조는 `시장 영역`을 중심으로 설계한다.

시장 영역은 다음 3가지 상태를 가진다.

```text
1. 시장
2. 카드 상세보기
3. 자원 확보
```

카드 상세보기와 자원 확보는 별도 팝업이 아니라, 기존 시장을 표시하던 영역을 대체하는 화면이다.

---

## 2. 전체 화면 구성 개요

MVP 기준 전체 화면은 다음 주요 영역으로 나눈다.

```text
- 상단 상태 영역
- 시장 영역
- 보유 자원 영역
- 예약 카드 영역
- 보유 자산 / 포트폴리오 요약 영역
- 주요 입력 영역
```

구체적인 배치, 좌표, 크기, 색상은 Unity 구현 단계에서 조정한다.
본 문서에서는 각 영역의 역할과 상태 전환만 정의한다.

---

## 3. 상단 상태 영역

상단 상태 영역에는 런 진행 상태와 핵심 리스크 정보를 표시한다.

표시 정보 예시:

```text
- 현재 회계년도
- 현재 분기
- 남은 영업일
- 분기 목표 운용 수익
- 현재 분기 운용 수익
- 환매 압력
```

예시 표기:

```text
2회계년도 · 1Q
남은 영업일 3일
분기 운용 수익 8 / 목표 15
환매 압력 4 / 10
```

상단 상태 영역은 시장 영역 상태와 무관하게 항상 표시한다.

---

## 4. 시장 영역

시장 영역은 게임의 중심 화면이다.

시장 영역은 다음 3가지 상태 중 하나를 가진다.

```csharp
public enum MarketAreaState
{
    Market,
    CardDetail,
    GainLiquidity
}
```

각 상태의 의미는 다음과 같다.

| 상태 | 의미 |
|---|---|
| Market | 시장 테이프와 예약 카드 접근이 가능한 기본 상태 |
| CardDetail | 선택한 카드의 상세 정보와 매수/예약 UI를 표시하는 상태 |
| GainLiquidity | 자원 확보 선택 UI를 표시하는 상태 |

---

## 5. 시장 영역 상태 1: 시장

`Market`은 시장 영역의 기본 상태이다.

이 상태에서는 시장 테이프를 표시하고, 플레이어가 직접 카드 또는 중앙 은행을 선택할 수 있다.

표시 요소:

```text
- 매도 임박 카드
- 현재 시장 카드
- 예비 시장 카드
- 중앙 은행
- 예약 카드 접근 영역
```

가능한 입력:

```text
- 시장 카드 클릭
- 예약 카드 클릭
- 중앙 은행 클릭
- 다음 영업일 클릭
```

시장 상태에서는 `다음 영업일` 입력이 활성화된다.

```text
시장 상태
→ 다음 영업일 활성화
```

---

## 6. 시장 영역 상태 2: 카드 상세보기

시장 카드 또는 예약 카드를 클릭하면 `CardDetail` 상태로 전환한다.

```text
시장 카드 클릭
→ 카드 상세보기 상태

예약 카드 클릭
→ 카드 상세보기 상태
```

카드 상세보기는 시장 영역을 대체한다.
즉, 시장 테이프가 보이던 영역에 선택 카드의 상세 정보와 관련 조작 UI를 표시한다.

표시 정보:

```text
- 카드 이미지
- 카드 이름
- 카드 제반 정보
- 운용가치
- 운용 수익
- 비용
- 태그
- 희귀도
- 닫기 버튼
- 매수 관련 UI
- 예약 버튼
```

단, 예약 버튼은 시장 카드 상세보기에서만 표시한다.

```text
시장 카드 상세보기
→ 예약 버튼 표시

예약 카드 상세보기
→ 예약 버튼 숨김
```

---

## 7. 카드 상세보기 닫기

카드 상세보기 상태에서 닫기 버튼을 누르면 항상 시장 상태로 복귀한다.

```text
카드 상세보기
→ 닫기
→ 시장 상태
```

닫기는 영업일을 소비하지 않는다.

```text
카드 보기 = 무료
카드 닫기 = 무료
매수 확정 = 영업일 소비
예약 버튼 클릭 = 영업일 소비
```

닫기 처리 시 다음 상태를 초기화한다.

```text
- 선택 카드
- 매수 결제 대기 상태
- 비용 슬롯 표시
- 자동 배치 상태
- 매수 버튼 상태
```

단, 이미 비용 슬롯에 칩을 올려둔 상태에서 닫기를 누르는 경우에는 매수 대기 취소와 동일하게 처리한다.

```text
닫기
→ 올려둔 칩 전부 회수
→ 선택 카드 해제
→ 시장 상태 복귀
→ 영업일 소비 없음
```

---

## 8. 시장 영역 상태 3: 자원 확보

시장 상태에서 중앙 은행을 클릭하면 `GainLiquidity` 상태로 전환한다.

```text
중앙 은행 클릭
→ 자원 확보 상태
```

자원 확보는 별도 팝업이 아니라 시장 영역을 대체하는 화면이다.

표시 요소:

```text
- 자원 확보 제목
- 현금 선택 버튼
- 리서치 선택 버튼
- 신용 선택 버튼
- 원자재 선택 버튼
- 선택된 자원 표시
- 닫기 버튼
- 안내 메시지 영역
```

자원 확보에서 선택 가능한 기본 자원은 다음 4개이다.

```text
- 현금
- 리서치
- 신용
- 원자재
```

딜은 자원 확보로 획득할 수 없다.

---

## 9. 자원 확보 닫기

자원 확보 상태에서 닫기 버튼은 첫 자원 획득 전까지만 가능하다.

```text
중앙 은행 클릭
→ 자원 확보 상태 진입
→ 아직 자원 획득 없음
→ 닫기 가능
```

닫기 클릭 시 시장 상태로 복귀한다.

```text
자원 확보
→ 닫기
→ 시장 상태
→ 영업일 소비 없음
```

자원을 하나라도 획득하면 닫기 버튼은 비활성화된다.

```text
첫 자원 클릭
→ 자원 즉시 획득
→ 자원 확보 행동 확정
→ 닫기 비활성화
```

이후에는 자원 확보가 완료될 때까지 진행한다.

---

## 10. 다음 영업일 버튼

`다음 영업일`은 현재 영업일을 종료하는 입력이다.

```text
다음 영업일 클릭
→ 현재 영업일 종료
→ 남은 영업일 1일 소비
```

다음 영업일 버튼은 시장 영역이 `Market` 상태일 때만 활성화된다.

| 시장 영역 상태 | 다음 영업일 |
|---|---:|
| Market | 활성화 |
| CardDetail | 비활성화 |
| GainLiquidity | 비활성화 |

이 규칙은 실수로 영업일을 소비하는 상황을 방지하기 위한 것이다.

```text
카드 상세보기 중
→ 먼저 닫기 필요
→ 이후 다음 영업일 가능

자원 확보 중
→ 첫 자원 획득 전이면 닫기 가능
→ 이후 다음 영업일 불가
```

---

## 11. 보유 자원 영역

보유 자원 영역에는 현재 보유 중인 결제 자원을 표시한다.

표시 대상:

```text
- 현금
- 리서치
- 신용
- 원자재
- 딜
```

전문 자원은 합계 10개 제한을 가진다.

```text
리서치 + 신용 + 원자재 <= 10
```

딜은 별도 한도를 가진다.

```text
딜 <= 3
```

현금은 전문 자원 한도에 포함되지 않는다.

표시 예시:

```text
현금 8
리서치 3
신용 2
원자재 4
딜 1
```

딜과 전문 자원은 칩 물성이 중요하므로, 가능한 경우 칩 리소스를 활용해 표시한다.

---

## 12. 예약 카드 영역

예약 카드 영역에는 플레이어가 예약한 카드를 표시한다.

예약 구역 수는 3개이다.

```text
예약 카드 최대 3장
```

예약 카드 영역은 분기와 회계년도 전환 후에도 유지된다.

```text
분기 전환
→ 예약 카드 유지

회계년도 전환
→ 예약 카드 유지
```

예약 카드를 클릭하면 카드 상세보기 상태로 진입한다.

```text
예약 카드 클릭
→ 카드 상세보기
```

예약 카드 상세보기에서는 예약 버튼을 표시하지 않는다.
예약 카드는 매수만 가능하다.

---

## 13. 보유 자산 / 포트폴리오 요약 영역

보유 자산 영역은 플레이어가 매수 완료한 자산을 표시한다.

표시 정보:

```text
- 보유 자산 수
- 현재 운용가치 합계
- 자산군 태그별 보유량
- 태그별 보유량
- 보유 자산 카드 목록 또는 요약
```

보유 자산 정렬 규칙은 별도 자산/포트폴리오 문서에서 정의한다.

기본 방향은 다음과 같다.

```text
자산군 태그별로 묶음
→ 내부에서는 희귀도 순 정렬
→ 희귀도가 같으면 먼저 산 자산을 위로 정렬
```

## 14. 화면 상태 전환 요약

시장 영역 상태 전환은 다음과 같다.

```text
Market

Market
→ 예약 카드 클릭
→ CardDetail

Market
→ 중앙 은행 클릭

CardDetail
→ 닫기
→ Market

CardDetail
→ 매수 확정
→ 영업일 소비
→ 이후 Market 또는 다음 영업일/분기 처리

CardDetail
→ 예약 버튼 클릭
→ 영업일 소비
→ 이후 Market 또는 다음 영업일/분기 처리

GainLiquidity
→ 닫기
→ Market
단, 첫 자원 획득 전까지만 가능

→ 영업일 소비
→ 이후 Market 또는 다음 영업일/분기 처리
```

## 15. 상태별 주요 입력 가능 여부

| 입력 | Market | CardDetail | GainLiquidity |
|---|---:|---:|---:|
| 시장 카드 클릭 | O | X | X |
| 예약 카드 클릭 | O | X | X |
| 중앙 은행 클릭 | O | X | X |
| 다음 영업일 | O | X | X |
| 닫기 | X | O | 조건부 |
| 매수 확정 | X | O | X |

| 자원 선택 | X | X | O |

자원 확보의 닫기는 조건부이다.

```text
자원 획득 전
→ 닫기 가능

자원 1개 이상 획득 후
→ 닫기 불가
```

## 16. 카드 상세보기와 결제 대기

카드 상세보기 진입 시 매수 확인 / 결제 대기 상태로 들어간다.

카드가 전문 자원 비용을 가진 경우:

```text
카드 상세보기
→ 전문 자원 비용 슬롯 표시
→ 매수 버튼 표시
→ 자동 배치 버튼 표시
→ 칩 배치 가능
```

순수 현금 자산인 경우:

```text
카드 상세보기
→ 전문 자원 비용 슬롯 없음
→ 매수 버튼 표시
```

카드 상세보기 자체는 영업일을 소비하지 않는다.
매수 버튼을 눌러 매수 확정이 발생할 때 영업일을 소비한다.

---

## 17. 예약 버튼과 결제 대기

시장 카드 상세보기에는 예약 버튼이 표시된다.

예약 버튼 클릭은 즉시 예약 액션이다.

```text
예약 버튼 클릭
→ 예약 액션 실행
→ 영업일 소비
```

예약 확인 상태는 존재하지 않는다.

비용 슬롯에 칩이 올라간 상태에서 예약 버튼을 누르면 다음 순서로 처리한다.

```text
예약 버튼 클릭
→ 올려둔 칩 전부 자동 회수
→ 매수 결제 상태 초기화
→ 예약 액션 실행
```

예약 구역이 가득 찬 경우 예약 버튼은 비활성화한다.

```text
예약 구역 3 / 3
→ 예약 버튼 비활성화
→ "예약 구역이 가득 찼습니다"
```

---

## 18. 자원 확보 화면의 자원 버튼 상태

자원 확보 화면에서 자원 버튼은 현재 상태에 따라 활성/비활성화된다.

전문 자원 합계가 10이면 전문 자원 버튼은 비활성화한다.

```text
리서치 + 신용 + 원자재 = 10 / 10
→ 리서치 비활성
→ 신용 비활성
→ 원자재 비활성
→ 현금 활성
```

전문 자원 합계가 9인 상태에서 전문 자원 1개를 획득하면 합계가 10이 된다.

```text
전문 자원 9 / 10
→ 리서치 클릭
→ 리서치 +1
→ 전문 자원 10 / 10
→ 리서치 / 신용 / 원자재 비활성
→ 현금만 활성
```

현재 선택 상태에서 더 이상 유효한 자원 선택지가 없으면 자원 확보를 자동 종료한다.

## 19. 메시지 영역

화면에는 짧은 피드백 메시지를 표시할 수 있는 영역이 필요하다.

대표 메시지:

```text
- 자원칩 최대 한도 도달
- 자원칩 최대 보유: 리서치 +1 폐기
- 딜 최대 보유: 추가 딜 폐기
- 예약 구역이 가득 찼습니다
- 놓을 수 있는 슬롯이 없습니다
- 배치할 수 있는 칩이 없습니다
- 매수 조건이 부족합니다
```

메시지는 시스템 상태를 설명하기 위한 짧은 피드백이다.
긴 설명은 툴팁이나 상세 UI에서 제공한다.

## 20. UI 구현 원칙

### 20.1 위치와 레이아웃은 Unity에서 확정

본 문서는 버튼의 정확한 좌표, 크기, 간격, 색상을 강제하지 않는다.

기획서에서 정의하는 것은 다음이다.

```text
- 어떤 UI가 존재하는가
- 어떤 상태에서 표시되는가
- 어떤 상태에서 활성/비활성화되는가
- 클릭 시 어떤 처리가 일어나는가
```

---

### 20.2 시장 영역 상태는 명확히 하나만

시장 영역은 동시에 여러 상태를 표시하지 않는다.

```text
Market / CardDetail / GainLiquidity 중 하나만 활성
```

예를 들어 카드 상세보기와 자원 확보가 동시에 표시되면 안 된다.

---

### 20.3 행동 확정 전에는 복귀 가능

카드 상세보기와 자원 확보 진입은 검토 상태이다.

```text
카드 상세보기 닫기
→ 시장 복귀
→ 영업일 소비 없음

자원 확보 첫 자원 획득 전 닫기
→ 시장 복귀
→ 영업일 소비 없음
```

단, 행동이 확정된 뒤에는 취소할 수 없다.

```text
매수 확정
→ 영업일 소비

예약 버튼 클릭
→ 영업일 소비

자원 확보 첫 자원 획득
→ 자원 확보 행동 확정
```

## 21. 구현 메모

### 21.1 MarketAreaState

```csharp
public enum MarketAreaState
{
    Market,
    CardDetail,
    GainLiquidity
```

---

### 21.2 상태 전환 함수 예시

```csharp
void SetMarketAreaState(MarketAreaState state)
{
    CurrentMarketAreaState = state;

    MarketView.SetActive(state == MarketAreaState.Market);
    CardDetailView.SetActive(state == MarketAreaState.CardDetail);
    LiquidityView.SetActive(state == MarketAreaState.GainLiquidity);

    UpdateNextBusinessDayButton();
}
```

---

### 21.3 다음 영업일 버튼 활성화

```csharp
void UpdateNextBusinessDayButton()
{
    bool canGoNextBusinessDay =
        CurrentMarketAreaState == MarketAreaState.Market
        && CurrentBusinessDayPhase == BusinessDayPhase.WaitingForPlayerAction;

    NextBusinessDayButton.SetInteractable(canGoNextBusinessDay);
}
```

### 21.4 카드 상세보기 진입

```csharp
void OpenCardDetail(AssetCard card)
```

```csharp
void CloseCardDetail()
```

```csharp
void CloseLiquidityView()
```

## 22. 구현 시 주의사항

```text
- 카드 상세보기와 자원 확보는 팝업이 아니라 시장 영역 대체 상태이다.
- 시장 영역은 한 번에 하나의 상태만 가진다.
- 다음 영업일 버튼은 시장 상태에서만 활성화한다.
- 카드 상세보기 닫기는 영업일을 소비하지 않는다.
- 자원 확보 닫기는 첫 자원 획득 전까지만 가능하다.
- 예약 버튼은 시장 카드 상세보기에서만 표시한다.
- 예약 구역이 가득 차면 예약 버튼은 비활성화한다.
- 예약 버튼 클릭은 즉시 예약 액션이다.
- 자원 확보에서 첫 자원 클릭은 행동 확정으로 본다.
```
