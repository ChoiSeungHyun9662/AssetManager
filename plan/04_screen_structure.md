# 04. 화면 구조

## 1. 목적

이 문서는 MVP 기준 화면 구조와 주요 UI 입력을 정의한다.

새 규칙에서는 시장 영역 상태를 단일 `Market` 상태로 유지한다.
기존 카드 상세보기 화면과 중앙 은행 자원 확보 화면은 제거한다.

---

## 2. 전체 화면 구성

MVP 기준 화면은 다음 영역으로 나눈다.

```text
- 상단 상태 영역
- 시장 영역
- 보유 자원 영역
- 포트폴리오 영역
- 메시지 영역
- 주요 입력 영역
```

별도 액션 선택창은 사용하지 않는다.
플레이어는 보드 위 요소를 직접 조작한다.

---

## 3. 상단 상태 영역

표시 정보:

```text
- 현재 회계년도
- 현재 분기
- 남은 영업일
- 분기 목표 수익
- 현재 분기 수익
- 월세 밀림
```

예시:

```text
2회계년도 · 1Q
남은 영업일 3일
분기 수익 8 / 목표 15
월세 밀림 4 / 10
```

---

## 4. 시장 영역

시장 영역은 1x8 시장 테이프를 표시한다.

```text
[1][2][3][4][5][6][7][8]
```

시장 슬롯에는 다음 카드가 표시된다.

```text
- 주식 카드
- 소모형 자원 카드
```

예약된 주식은 같은 슬롯에 남아 있으며 예약 상태가 시각적으로 표시된다.

가능한 입력:

```text
- 카드 호버
- 주식 매수
- 소모형 자원 카드 매수
- 주식 예약
- 다음 영업일
```

소모형 자원 카드는 예약할 수 없다.

---

## 5. 카드 호버 확대

카드 상세보기 화면은 제거한다.

카드에 마우스를 호버하면 같은 정보를 더 큰 스케일로 표시하는 확대 카드가 뜬다.

```text
시장 카드
→ 호버
→ 확대 카드 표시
```

시장 카드와 확대 카드가 제공하는 정보는 동일하다.
스케일만 다르다.

주식 카드 정보:

```text
- 이미지
- 이름
- 등급
- 가치
- 코스트
- 배당금
```

소모형 자원 카드 정보:

```text
- 이미지
- 등급
- 제공 자원
- 코스트
```

호버는 영업일을 소비하지 않는다.

---

## 6. 매수와 예약 입력

카드 상세보기 상태가 없으므로, 매수/예약 입력은 시장 카드 자체 또는 카드 주변 UI에서 제공한다.

주식 카드에서 가능한 입력:

```text
- 매수
- 예약
```

소모형 자원 카드에서 가능한 입력:

```text
- 매수
```

예약은 주식 카드에만 표시한다.
예약 가능 수가 3/3이면 예약 입력은 비활성화한다.

---

## 7. 보유 자원 영역

표시 대상:

```text
- 현금
- 독서
- 명상
- 인내
- 딜
```

투자 철학과 딜은 칩 형태를 유지한다.

투자 철학 한도:

```text
독서 + 명상 + 인내 <= 10
각 투자 철학 <= 5
```

딜 한도:

```text
딜 <= 3
```

---

## 8. 포트폴리오 영역

포트폴리오 영역은 최대 8칸이다.

```text
포트폴리오: [1][2][3][4][5][6][7][8]
```

표시 정보:

```text
- 보유 주식
- 호일 상태
- 현재 가치 합계
- 포트폴리오 점유 수
```

주식 매도 입력은 포트폴리오의 보유 주식에서 제공한다.
보유 주식 Card Button에 마우스를 올리면 해당 Sell Button을 표시한다.
Card Button과 Sell Button 사이로 커서를 이동하는 동안에는 Sell Button을 유지하고, 둘 다 벗어나면 숨긴다.
Card Button 클릭은 Sell Button 표시 조건이 아니다.
매도는 영업일을 소비하지 않는다.

---

## 9. 다음 영업일 버튼

다음 영업일 버튼은 현재 영업일을 종료하는 입력이다.

```text
다음 영업일
→ 영업일 종료
```

다음 영업일은 시장 상태에서 항상 접근 가능해야 한다.
다만 매수/예약/자동 배치 같은 처리 중에는 임시로 비활성화할 수 있다.

---

## 10. 메시지 영역

짧은 피드백 메시지를 표시한다.

대표 메시지:

```text
- 주식 매도가 필요합니다
- 예약이 가득 찼습니다
- 딜 최대 보유: 추가 딜 폐기
- 투자 철학 최대 보유: 독서 +1 폐기
- 매수 조건이 부족합니다
- 월세가 밀려 파산했습니다
```

메시지는 상태 설명을 위한 짧은 피드백이다.
긴 설명은 카드 호버나 툴팁으로 제공한다.

---

## 11. 구현 데이터 제안

시장 영역 상태는 하나만 둔다.

```csharp
public enum MarketAreaState
{
    Market
}
```

호버 상태는 시장 영역 전환이 아니라 UI 오버레이 상태로 관리한다.

```csharp
public class CardHoverPreviewState
{
    public MarketCardRuntimeData HoveredCard;
    public bool IsVisible;
}
```

---

## 12. 구현 시 주의사항

```text
- 카드 상세보기 화면을 사용하지 않는다.
- 자원 확보 화면을 사용하지 않는다.
- 중앙 은행 UI를 사용하지 않는다.
- 시장 영역 상태는 Market 하나로 유지한다.
- 카드 정보 확인은 호버 확대 카드로 처리한다.
- 시장 카드와 확대 카드의 정보는 동일하고 스케일만 다르다.
- 소모형 자원 카드는 이름을 표시하지 않는다.
- 주식 매도는 포트폴리오 영역에서 제공한다.
- 주식 매도 버튼은 보유 주식 카드 호버 중에만 표시한다.
```
