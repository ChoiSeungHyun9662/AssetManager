# 10. 칩 조작 UX

## 1. 목적

이 문서는 자산 매수 과정에서 사용하는 칩 조작 UX를 정의한다.

본 게임에서 칩의 물성은 중요한 핵심 감성이다.
특히 전문 자원과 딜은 단순 숫자가 아니라, 실제로 잡고 올려놓는 칩처럼 느껴져야 한다.

칩 조작 UX는 다음 목표를 가진다.

```text
- 결제 과정의 손맛 제공
- 배치 가능한 슬롯을 명확히 안내
- 실수 배치를 부드럽게 복구
- 드래그와 클릭을 모두 지원
- 자동 배치로 반복 조작 피로 감소
- 매수 확정 전에는 언제든 칩 회수 가능
```

---

## 2. 칩으로 표현하는 대상

칩으로 표현하는 대상은 다음이다.

```text
- 리서치
- 신용
- 원자재
- 딜
```

현금은 칩으로 배치하지 않는다.
현금은 매수 버튼을 누를 때 자동 차감한다.

보유 자산군이나 태그 요약도 칩으로 표시하지 않는다.

```text
보유 자산군:
주식 5
채권 3

태그:
성장 4
방어 2
```

보유 자산군과 태그는 숫자 텍스트로 표시한다.

---

## 3. 칩 종류

### 3.1 전문 자원 칩

전문 자원 칩은 다음 3종이다.

```text
- 리서치 칩
- 신용 칩
- 원자재 칩
```

각 전문 자원 칩은 해당 자원 슬롯에만 배치할 수 있다.

```text
리서치 칩
→ 리서치 슬롯에 배치 가능

신용 칩
→ 신용 슬롯에 배치 가능

원자재 칩
→ 원자재 슬롯에 배치 가능
```

---

### 3.2 딜

딜은 모든 전문 자원 슬롯에 배치할 수 있다.

```text
딜
→ 리서치 슬롯 배치 가능
→ 신용 슬롯 배치 가능
→ 원자재 슬롯 배치 가능
```

딜은 전문 자원 슬롯 1칸을 대체한다.

또한 투여한 딜 개수만큼 기본 현금 비용을 낮춘다.

```text
딜 1개
= 전문 자원 슬롯 1칸 대체
+ 기본 현금 비용 1 감소
```

딜의 현금 할인은 인플레이션 적용 전에 반영한다.

---

## 4. 칩 배치 대상

칩 배치는 카드 상세보기의 전문 자원 비용 슬롯에서만 발생한다.

```text
카드 상세보기
→ 비용 슬롯 표시
→ 칩 배치 가능
```

비용 슬롯은 카드의 전문 자원 비용에 따라 생성된다.

예시:

```text
카드 비용:
리서치 2
신용 1
원자재 0

생성 슬롯:
리서치 슬롯 2개
신용 슬롯 1개
```

현금 비용에는 칩을 올리지 않는다.

```text
현금 비용
→ 매수 확정 시 자동 차감
```

---

## 5. 비용 슬롯 상태

각 비용 슬롯은 다음 상태 중 하나를 가진다.

```text
- 비어 있음
- 해당 전문 자원 칩 배치됨
- 딜 배치됨
```

슬롯에 칩이 올라가면 해당 슬롯은 결제 완료 상태로 본다.

```text
칩이 올라간 슬롯
→ 채워진 슬롯
```

딜이 올라간 슬롯도 결제 완료 상태이다.

```text
딜 배치
→ 해당 전문 자원 슬롯 결제 완료
```

---

## 6. 배치 방식

칩 배치는 두 가지 방식을 지원한다.

```text
1. 드래그 배치
2. 클릭 배치
```

둘 중 어느 방식이든 최종적으로는 빈 비용 슬롯에 칩을 올리는 처리이다.

---

## 7. 드래그 배치

플레이어는 보유 자원 영역의 칩을 드래그해 비용 슬롯 위에 놓을 수 있다.

```text
칩 드래그 시작
→ 배치 가능한 슬롯 하이라이트
→ 슬롯 위에 드롭
→ 유효하면 배치
→ 유효하지 않으면 원위치 복귀
```

드래그 중에는 배치 가능한 슬롯만 강조한다.

하이라이트 대상:

```text
- 해당 칩이 들어갈 수 있는 빈 슬롯
```

하이라이트하지 않는 대상:

```text
- 이미 칩이 올라간 슬롯
- 해당 칩을 넣을 수 없는 슬롯
- 현금 비용 영역
- 매수 버튼
```

---

## 8. 클릭 배치

플레이어는 보유 자원 영역의 칩을 클릭해 자동으로 배치할 수 있다.

```text
칩 클릭
→ 해당 칩이 들어갈 수 있는 하이라이트 슬롯 중 가장 빠른 칸에 배치
```

여기서 “가장 빠른 칸”은 UI 순서상 먼저 표시되는 빈 슬롯을 의미한다.

예시:

```text
리서치 슬롯 2개가 비어 있음
→ 리서치 칩 클릭
→ 첫 번째 리서치 슬롯에 배치
```

딜 클릭 시:

```text
딜 클릭
→ 딜이 들어갈 수 있는 빈 슬롯 중 가장 빠른 칸에 배치
```

배치 가능한 슬롯이 없으면 버튼 반응은 있지만 별도 로직은 실행하지 않는다.

```text
모든 배치 가능 슬롯이 채워짐
→ 칩 클릭
→ 버튼 반응만 있음
→ 배치 없음
```

---

## 9. 하이라이트 규칙

칩 조작 중 하이라이트는 다음 대상에만 준다.

```text
- 빈 슬롯
- 딜
```

정확히는 다음과 같다.

### 9.1 전문 자원 칩을 조작할 때

전문 자원 칩을 드래그하거나 선택하면, 해당 전문 자원과 일치하는 빈 슬롯만 하이라이트한다.

```text
리서치 칩 선택
→ 비어 있는 리서치 슬롯 하이라이트

신용 칩 선택
→ 비어 있는 신용 슬롯 하이라이트

원자재 칩 선택
→ 비어 있는 원자재 슬롯 하이라이트
```

이미 칩이 올라간 슬롯은 하이라이트하지 않는다.

---

### 9.2 딜을 조작할 때

딜을 드래그하거나 선택하면, 모든 종류의 빈 전문 자원 슬롯을 하이라이트한다.

```text
딜 선택
→ 비어 있는 리서치 슬롯 하이라이트
→ 비어 있는 신용 슬롯 하이라이트
→ 비어 있는 원자재 슬롯 하이라이트
```

딜은 모든 전문 자원 슬롯을 대체할 수 있기 때문이다.

---

## 10. 유효 배치

칩 배치가 유효하려면 다음 조건을 만족해야 한다.

```text
- 현재 카드 상세보기 상태이다.
- 선택 카드가 유효하다.
- 대상 슬롯이 비어 있다.
- 대상 슬롯이 해당 칩을 받을 수 있다.
- 플레이어가 해당 칩을 보유하고 있다.
```

유효 배치 예시:

```text
리서치 칩
→ 빈 리서치 슬롯
→ 배치 가능
```

```text
딜
→ 빈 원자재 슬롯
→ 배치 가능
```

---

## 11. 무효 배치

다음 경우는 무효 배치이다.

```text
- 리서치 칩을 신용 슬롯에 놓음
- 신용 칩을 원자재 슬롯에 놓음
- 원자재 칩을 리서치 슬롯에 놓음
- 이미 칩이 있는 슬롯에 놓음
- 현금 비용 영역에 놓음
- 매수 버튼 위에 놓음
- 카드 상세보기 상태가 아님
```

무효 배치 시에는 칩을 원래 위치로 되돌린다.

```text
무효 드롭
→ 칩 원위치 복귀
→ 짧은 실패 피드백
→ 결제 상태 변화 없음
```

---

## 12. 칩 배치 시 자원 소비 타이밍

칩을 슬롯에 올려도 자원은 아직 소비되지 않는다.

```text
칩 배치
→ 자원 소비 아님
```

자원은 매수 확정 시점에만 소비한다.

```text
매수 확정
→ 배치된 전문 자원 칩 소비
→ 배치된 딜 소비
→ 현금 차감
```

따라서 매수 확정 전에는 칩을 회수할 수 있다.

---

## 13. 칩 회수

플레이어는 비용 슬롯에 올려둔 칩을 회수할 수 있다.

지원 방식:

```text
- 슬롯에 있는 칩 클릭
- 슬롯 밖으로 드래그
- 카드 상세보기 닫기
- 예약 버튼 클릭 시 자동 회수
```

칩 회수 시 해당 슬롯은 다시 빈 슬롯이 된다.

```text
칩 회수
→ 슬롯 비움
→ 매수 버튼 상태 갱신
→ 현금 비용 재계산
```

딜을 회수하면 딜 할인도 제거된다.

```text
딜 회수
→ 딜 사용 수 감소
→ 기본 현금 비용 할인 감소
→ 최종 현금 비용 재계산
```

---

## 14. 모든 칩 회수와 취소

카드 상세보기 닫기 또는 매수 취소 시 올려둔 모든 칩을 자동 회수한다.

```text
닫기
→ 올려둔 칩 전부 회수
→ 결제 상태 초기화
→ 시장 상태 복귀
→ 영업일 소비 없음
```

예약 버튼 클릭 시에도 올려둔 칩을 자동 회수한 뒤 예약을 실행한다.

```text
비용 슬롯에 칩 배치
→ 예약 버튼 클릭
→ 칩 전부 회수
→ 매수 결제 상태 초기화
→ 예약 실행
```

이 경우 매수는 확정되지 않았으므로 자원은 소비되지 않는다.

---

## 15. 딤드 처리

카드 상세보기에서 칩을 올려놓는 과정에 관여하지 않는 영역에는 딤드 시각 표현을 줄 수 있다.

딤드는 다음 목적을 가진다.

```text
- 현재 결제 조작 중임을 알린다.
- 관련 없는 영역을 시각적으로 낮춘다.
- 실수 입력을 줄인다.
```

딤드된 영역을 누르면 다음 처리를 한다.

```text
딤드 영역 클릭
→ 딤드 해제
→ 대기 중인 칩 자동 회수
→ 해당 클릭의 원래 행동은 실행하지 않음
```

즉, 딤드 영역 클릭은 “칩 자동 회수 명령”으로 처리한다.

```text
딤드 클릭
= 이번 클릭은 취소/회수 입력
= 원래 클릭 대상 행동 실행 안 함
```

---

## 16. 자동 배치 버튼

카드 상세보기에는 자동 배치 버튼을 둘 수 있다.

자동 배치 버튼은 현재 보유 자원과 비용 슬롯을 기준으로 가능한 칩을 자동으로 배치한다.

```text
자동 배치 클릭
→ 배치 가능한 칩을 비용 슬롯에 자동 배치
```

자동 배치는 전문 자원 칩을 우선 사용하고, 필요한 경우 딜을 사용할 수 있다.

정확한 우선순위는 별도 밸런스/UX 조정이 가능하지만, 기본 원칙은 다음이다.

```text
1. 해당 전문 자원 칩으로 채울 수 있는 슬롯 우선
2. 부족한 슬롯은 딜으로 보완
3. 현금 비용을 낼 수 있는지 재계산
```

자동 배치 후에도 매수는 자동으로 확정하지 않는다.

```text
자동 배치
→ 슬롯 채움
→ 매수 버튼 활성화 가능
→ 매수는 별도 클릭 필요
```

---

## 17. 자동 배치 중 입력 처리

자동 배치 애니메이션 중 플레이어가 매수 버튼을 누를 수 있다.

이 경우 매수 입력은 큐에 저장한다.

```text
자동 배치 애니메이션 중 매수 버튼 클릭
→ 매수 입력 큐에 저장
→ 자동 배치 완료 후 매수 가능하면 즉시 매수 실행
```

자동 배치 완료 후 매수 조건이 충족되지 않으면 매수 입력 큐는 폐기하고 실패 피드백을 표시한다.

```text
자동 배치 완료
→ 매수 조건 불충족
→ 큐 제거
→ 매수 실패 피드백
```

카드 상세보기 닫기 또는 취소 시 매수 큐는 제거한다.

```text
닫기 / 취소
→ 매수 큐 제거
```

---

## 18. 자동 배치 취소

자동 배치 애니메이션 도중 카드 상세보기를 닫거나 취소하면, 이동 중인 칩을 즉시 회수한다.

```text
자동 배치 중 닫기
→ 자동 배치 애니메이션 중단
→ 이동 중인 칩 즉시 트레이 복귀
→ 이미 배치된 칩 회수
→ 매수 큐 제거
→ 시장 상태 복귀
```

영업일은 소비하지 않는다.

---

## 19. 배치 불가 피드백

칩을 배치할 수 없는 경우 짧은 피드백을 준다.

상황 예시:

```text
- 배치 가능한 슬롯 없음
- 해당 자원 칩 부족
- 딜 없음
- 잘못된 슬롯에 드롭
- 이미 슬롯이 모두 채워짐
```

피드백 예시:

```text
- 칩이 짧게 흔들림
- 슬롯이 짧게 점멸
- 짧은 효과음
- 메시지 표시
```

메시지는 필요한 경우에만 표시한다.

예시 문구:

```text
놓을 수 있는 슬롯이 없습니다.
```

---

## 20. 매수 버튼 상태 갱신

칩이 배치되거나 회수될 때마다 매수 버튼 상태를 갱신한다.

갱신 항목:

```text
- 모든 전문 자원 슬롯이 채워졌는가
- 사용한 딜 수
- 딜 할인 후 기본 현금 비용
- 인플레이션 적용 후 최종 현금 비용
- 현재 현금으로 지불 가능한가
- 매수 버튼 활성/비활성
```

딜 배치 또는 회수 시 최종 현금 비용이 바뀔 수 있으므로 반드시 재계산한다.

---

## 21. 비용 툴팁

매수 버튼 또는 비용 영역에는 최종 현금 비용 계산식을 툴팁으로 제공할 수 있다.

단, 툴팁 안에 별도의 “자세히 보기” 버튼은 넣지 않는다.
조작 뎁스가 늘어나기 때문이다.

툴팁 예시:

```text
기본 현금 비용 5
딜 할인 -2
인플레이션 +1
최종 현금 비용 4
```

툴팁은 정보를 보여주는 용도이며, 추가 입력 단계를 만들지 않는다.

## 22. 클릭 배치와 드래그 배치의 우선순위

드래그와 클릭은 모두 동일한 결제 상태를 변경한다.

```text
드래그 배치
→ PaymentSlotState 갱신

클릭 배치
→ PaymentSlotState 갱신
```

구현상 둘은 같은 내부 함수로 연결하는 것이 좋다.

```csharp
bool TryPlaceChipInSlot(ChipData chip, PaymentSlotState slot)
```

클릭 배치는 가능한 슬롯 중 가장 빠른 슬롯을 찾아 `TryPlaceChipInSlot`을 호출한다.

## 25. 배치 가능 판정

```csharp
bool CanPlaceChipInSlot(ChipData chip, PaymentSlotState slot)
{
    if (slot.IsFilled)
        return false;

    if (chip.ResourceType == ResourceType.Deal)
        return IsProfessionalResource(slot.RequiredResourceType);

    return chip.ResourceType == slot.RequiredResourceType;
}
```

## 26. 클릭 배치 예시

```csharp
void OnChipClicked(ChipData chip)
{
    PaymentSlotState targetSlot =
        FindFirstValidEmptySlotForChip(chip);

    if (targetSlot == null)
    {
        PlayChipNoTargetFeedback(chip);
        return;
    }

    TryPlaceChipInSlot(chip, targetSlot);
}
```

```csharp
PaymentSlotState FindFirstValidEmptySlotForChip(ChipData chip)
{
    return CurrentPaymentState.PaymentSlots
        .Where(slot => CanPlaceChipInSlot(chip, slot))
        .OrderBy(slot => slot.SlotIndex)
        .FirstOrDefault();
}
```

void OnPlacedChipClicked(PaymentSlotState slot)

    ReturnChipToTray(slot.PlacedResourceType.Value);

## 28. 딜 할인 계산 예시

```csharp
int GetUsedDealCount(PurchasePaymentState state)
{
    return state.PaymentSlots.Count(slot => slot.IsFilledByDeal);
}
```

```csharp
int CalculateCashCostBeforeInflation(
    int baseCashCost,
    int usedDealCount
)
{
    return Math.Max(0, baseCashCost - usedDealCount);
}
```

## 29. 결제 상태 재계산

```csharp
void RecalculatePaymentState()
{
    int usedDealCount = GetUsedDealCount(CurrentPaymentState);

    int cashCostBeforeInflation =
        CalculateCashCostBeforeInflation(
            CurrentPaymentState.BaseCashCost,
            usedDealCount
        );

    CurrentPaymentState.FinalCashCost =
        ApplyInflationToCashCost(cashCostBeforeInflation);

    CurrentPaymentState.IsValid =
        AreAllPaymentSlotsFilled(CurrentPaymentState)
        && CurrentCash >= CurrentPaymentState.FinalCashCost;

    UpdateBuyButton();
    UpdateCostTooltip();
}
```

## 30. 구현 시 주의사항

```text
- 칩 조작은 전문 자원과 딜에만 적용한다.
- 현금은 칩으로 배치하지 않는다.
- 칩을 슬롯에 올려도 자원은 아직 소비되지 않는다.
- 자원 소비는 매수 확정 시점에만 발생한다.
- 딜은 모든 전문 자원 슬롯에 배치 가능하다.
- 딜은 슬롯 대체와 현금 비용 감소를 동시에 수행한다.
- 딜 할인은 인플레이션 적용 전에 반영한다.
- 클릭 배치는 가능한 슬롯 중 가장 빠른 칸에 배치한다.
- 모든 슬롯이 채워진 경우 칩을 클릭해도 버튼 반응만 있고 배치는 하지 않는다.
- 칩 회수 시 매수 버튼과 최종 현금 비용을 즉시 갱신한다.
- 카드 상세보기 닫기 시 모든 칩을 회수한다.
- 예약 버튼 클릭 시에도 올려둔 칩을 전부 회수한 뒤 예약한다.
- 딤드 영역 클릭은 칩 자동 회수 입력이며, 클릭한 원래 행동은 실행하지 않는다.
- 자동 배치는 매수를 자동 확정하지 않는다.
```
