# 03. 시간 구조

## 1. 목적

이 문서는 게임의 시간 단위와 진행 구조를 정의한다.

시간 계층:

```text
영업일 < 분기 < 회계년도
```

---

## 2. 시간 단위

### 2.1 영업일

영업일은 플레이어가 주요 행동 1회를 확정할 수 있는 최소 시간 단위이다.

영업일을 소비하는 행동:

```text
- 주식 매수
- 소모형 자원 카드 매수
- 주식 예약
- 다음 영업일
```

영업일을 소비하지 않는 행동:

```text
- 카드 호버
- 칩 배치/회수
- 주식 매도
```

### 2.2 분기

분기는 여러 영업일로 구성된 중간 목표 단위이다.

각 분기는 다음 요소를 가진다.

```text
- 회계년도 번호
- 분기 번호
- 분기당 영업일 수
- 분기 목표 수익
- 분기 수익 기록
```

### 2.3 회계년도

본 게임은 총 3개의 회계년도로 구성된다.

```text
1회계년도
2회계년도
3회계년도
```

1·2회계년도는 4Q가 휴가로 처리된다.
3회계년도는 4Q까지 모두 플레이한다.

---

## 3. 회계년도 구성

```text
1회계년도
- 1Q: 플레이
- 2Q: 플레이
- 3Q: 플레이
- 4Q: 휴가

2회계년도
- 1Q: 플레이
- 2Q: 플레이
- 3Q: 플레이
- 4Q: 휴가

3회계년도
- 1Q: 플레이
- 2Q: 플레이
- 3Q: 플레이
- 4Q: 플레이
```

총 플레이 분기는 10개이다.

분기당 영업일:

```text
1회계년도: 4영업일
2회계년도: 4영업일
3회계년도: 5영업일
```

총 플레이 영업일은 44일이다.

---

## 4. 분기 시작 처리

분기 시작 시:

```text
1. 현재 회계년도/분기 설정
2. 분기 목표 수익 설정
3. 남은 영업일 설정
4. 현재 분기 수익 0으로 초기화
5. 시장 테이프 갱신
6. 첫 영업일 시작
```

분기 첫 영업일에는 시장 테이프 갱신만 하고 진행은 생략한다.

```text
분기 첫 영업일
→ 시장 테이프 갱신
→ 시장 테이프 진행 없음
```

---

## 5. 일반 영업일 시작 처리

일반 영업일 시작:

```text
1. 시장 테이프 진행
2. 보유 주식 배당금 지급
3. 시장 상태 진입
4. 플레이어 입력 대기
```

분기 첫 영업일 시작:

```text
1. 시장 테이프 갱신
2. 시장 테이프 진행 없음
3. 보유 주식 배당금 지급
4. 시장 상태 진입
5. 플레이어 입력 대기
```

---

## 6. 영업일 종료 처리

영업일 종료:

```text
1. 현재 영업일 행동 상태 정리
2. 남은 영업일 1 감소
3. 남은 영업일 확인
```

남은 영업일이 있으면 다음 영업일을 시작한다.

```text
남은 영업일 > 0
→ 다음 영업일 시작
```

남은 영업일이 없으면 분기 마감으로 이동한다.

```text
남은 영업일 == 0
→ 분기 마감
```

마지막 영업일 종료 후에는 다음 영업일 배당금이 발생하지 않는다.

---

## 7. 분기 마감 처리

분기 마감:

```text
1. 분기 마감 정산 수익 반영
2. 분기 수익 확정
3. 분기 목표 달성률 계산
4. 월세 밀림 증가량 계산
5. 월세 밀림 증가 적용
6. 월세 밀림 한도 검사
7. 파산하지 않았다면 분기 정산 UI 표시
8. 다음 일정 진행
```

분기 목표 달성률은 분기 수익 기준이다.

```text
분기 수익
= 배당금
+ 주식 매도 수익
+ 분기 마감 정산 수익
```

---

## 8. 4Q 휴가

1·2회계년도의 4Q는 플레이하지 않는다.

4Q 휴가에서는 다음 처리를 하지 않는다.

```text
- 영업일 진행
- 주식 매수
- 소모형 자원 카드 매수
- 주식 예약
- 주식 매도
- 시장 테이프 진행
- 시장 테이프 갱신
- 보상 지급
- 패널티 적용
```

4Q 휴가 화면은 해당 회계년도 요약을 표시하고 다음 회계년도로 넘어간다.

---

## 9. 휴가 이후 회계년도 전환

휴가 화면에서 계속을 누르면 다음 회계년도가 시작된다.

```text
1회계년도 4Q 휴가
→ 2회계년도 시작

2회계년도 4Q 휴가
→ 3회계년도 시작
```

다음 회계년도 1Q 시작도 분기 시작이므로 시장 테이프 갱신을 수행한다.

---

## 10. 최종 정산 진입

3회계년도 4Q 마감 후 파산하지 않았다면 최종 정산으로 이동한다.

```text
3회계년도 4Q 마감
→ 월세 밀림 10 미만
→ 최종 정산
```

월세 밀림이 10 이상이면 파산으로 이동하고 최종 정산은 표시하지 않는다.

## 12. 구현 시 주의사항

```text
- 분기 첫 영업일에는 시장 테이프 갱신만 하고 진행은 생략한다.
- 일반 영업일 시작에는 시장 테이프 진행 후 배당금을 지급한다.
- 마지막 영업일 종료 후에는 배당금 없이 분기 마감으로 이동한다.
- 1·2회계년도 4Q는 휴가이며 플레이하지 않는다.
- 3회계년도 4Q는 플레이한다.
- 월세 밀림 10 도달 시 이후 진행보다 파산이 우선한다.
```
