# 12. 예약 시스템

## 1. 목적

이 문서는 새 예약 시스템의 규칙, 대상, 보상, 월세 밀림 처리, 시장 테이프와의 관계를 정의한다.

예약은 더 이상 별도 예약 구역으로 카드를 옮기는 행동이 아니다.
예약은 시장 8칸 안에 있는 주식 1장을 그 자리에서 잠그는 행동이다.

```text
예약
= 시장 슬롯 안의 주식을 잠금
= 시장 테이프 진행/당김/갱신의 영향을 받지 않게 함
```

---

## 2. 핵심 규칙

예약의 핵심 효과:

```text
시장 주식 1장 예약 상태로 잠금
→ 딜 +1
→ 월세 밀림 +1
→ 영업일 종료
```

예약은 주식 매수가 아니다.

```text
예약된 주식
→ 포트폴리오에 없음
→ 가치 합산 대상 아님
→ 배당금 발생 대상 아님
→ 분기 마감 정산 대상 아님
```

예약된 주식을 실제로 매수해야 보유 주식이 된다.

---

## 3. 예약 가능 대상

예약 가능:

```text
주식 카드
```

예약 불가:

```text
소모형 자원 카드
이미 예약된 주식
포트폴리오에 있는 주식
```

소모형 자원 카드는 즉시 효과를 발동하고 사라지는 카드이므로 예약할 수 없다.

---

## 4. 예약 수 제한

동시에 예약 가능한 주식은 최대 3장이다.

```text
예약 주식 최대 수: 3
```

예약 주식 수가 3장인 경우 새 예약은 불가능하다.

```text
예약 3 / 3
→ 새 예약 불가
→ "예약이 가득 찼습니다" 알림
→ 영업일 소비 없음
```

딜이 3개여도 예약은 가능하다.
딜 초과분만 폐기한다.

---

## 5. 예약과 시장 테이프

예약된 주식은 시장 8칸 안에 그대로 남는다.

```text
예약된 주식
→ 시장 슬롯 유지
→ 시장 테이프 진행 영향 없음
→ 시장 테이프 당김 영향 없음
→ 시장 테이프 갱신 영향 없음
```

예약 액션 직후에는 시장 테이프가 진행되지 않는다.
시장은 다음 영업일 시작 시 진행된다.

```text
예약 액션
→ 시장 테이프 진행 없음
→ 영업일 종료
→ 다음 영업일 시작 시 시장 테이프 진행
```

---

## 6. 예약 처리 순서

예약 액션 처리 순서:

```text
1. 예약 가능 여부 검증
2. 대상 주식을 예약 상태로 표시
3. 딜 +1 처리
4. 월세 밀림 +1 처리
5. 월세 밀림 한도 검사
6. 파산하지 않았다면 영업일 종료
```

예약 실패 시:

```text
비용 소비 없음
딜 획득 없음
월세 밀림 증가 없음
영업일 소비 없음
```

---

## 7. 딜 보상

예약은 딜을 획득하는 주요 수단이다.

```text
주식 예약
→ 딜 +1
```

딜 최대 보유량은 3개이다.

```text
딜 3 / 3 상태에서 예약
→ 예약은 정상 실행
→ 추가 딜만 폐기
→ "딜 최대 보유: 추가 딜 폐기" 표시
```

딜 한도는 예약 가능 여부를 막지 않는다.

---

## 8. 월세 밀림

예약은 월세 밀림을 1 증가시킨다.

```text
주식 예약
→ 월세 밀림 +1
```

월세 밀림 한도는 10이다.

```text
월세 밀림 >= 10
→ 파산
→ 게임 오버
```

예약으로 월세 밀림이 10 이상이 되면 영업일 종료보다 파산이 우선한다.

---

## 9. 예약된 주식 매수

예약된 주식도 시장 슬롯에 있으므로 시장 카드처럼 매수할 수 있다.

처리:

```text
예약된 주식 매수
→ 결제 검증
→ 자원 소비
→ 포트폴리오 추가 또는 호일 처리
→ 예약 상태 해제
→ 해당 시장 슬롯 비움
→ 시장 테이프 당김
→ 영업일 종료
```

예약된 주식을 매수하면 예약 수가 감소한다.

---

## 10. 호일 완성과 예약

동일 종목 3장을 보유해 호일이 완성되면, 같은 종목의 예약 주식도 제거한다.

```text
호일 완성
→ 덱의 같은 종목 제거
→ 시장의 같은 종목 제거
→ 예약된 같은 종목 제거
→ 예약 수 감소
→ 생긴 시장 빈칸은 시장 테이프 당김
```

이미 호일이 완성된 종목은 이번 게임에서 다시 등장하지 않는다.

---

## 11. 추가 매수권과 예약

추가 매수권 상태에서는 예약할 수 없다.

```text
추가 매수권 상태에서 가능:
- 주식 매수
- 소모형 자원 카드 매수 여부는 카드 효과 설계에 따라 제한 가능
- 다음 영업일

추가 매수권 상태에서 불가:
- 주식 예약
```

추가 매수권은 매수 전용 권리이므로 예약 버튼이나 예약 입력은 숨기거나 비활성화한다.

---

## 12. 구현 데이터 제안

예약은 별도 예약 구역이 아니라 시장 슬롯 상태로 관리한다.
별도 `Reservation Panel`과 예약 카드 버튼 목록은 만들지 않는다.
예약 상태와 예약 주식 매수는 시장 슬롯 카드 표면에서 처리한다.

```csharp
public class MarketSlot
{
    public int SlotIndex;
    public MarketCardRuntimeData Card;
    public bool IsReserved;
}
```

예약 수:

```csharp
int GetReservedStockCount()
{
    return MarketSlots.Count(slot => slot.IsReserved);
}
```

상수:

```csharp
const int MaxReservedStockCount = 3;
```

---

## 13. 구현 시 주의사항

```text
- 예약 구역은 제거한다.
- Reservation Panel은 제거한다.
- 예약은 시장 슬롯 안의 주식을 잠근다.
- 주식 카드만 예약 가능하다.
- 소모형 자원 카드는 예약 불가다.
- 동시 예약은 최대 3장이다.
- 딜이 3개여도 예약은 가능하다.
- 딜 초과분은 폐기한다.
- 예약은 월세 밀림 +1을 발생시킨다.
- 월세 밀림 증가 후 즉시 한도 검사를 한다.
- 예약 액션 직후 시장 테이프는 진행하지 않는다.
- 예약된 주식은 시장 테이프 진행/당김/갱신의 영향을 받지 않는다.
- 예약된 주식을 매수하면 예약 상태를 해제하고 시장 테이프 당김을 적용한다.
- 호일 완성 시 같은 종목의 예약 주식도 제거한다.
```
