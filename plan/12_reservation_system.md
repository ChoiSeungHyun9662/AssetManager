# 12. 예약 시스템

## 1. 목적

이 문서는 v3 기준 예약 규칙과 Mr.Market 제안과의 관계를 정의한다.

예약은 시장 8칸 안에 있는 주식 1장을 그 자리에서 잠그는 무료 행동이다.
예약은 카드를 보존하지만, 슬롯에 붙은 Mr.Market 제안 자체를 보존하지 않는다.

```text
예약
= 시장 슬롯 안의 주식을 잠금
= 시장 테이프 진행/당김/갱신의 영향을 받지 않게 함
= 영업일, 딜, 월세 밀림을 바꾸지 않음
```

---

## 2. 핵심 규칙

예약의 핵심 효과:

```text
시장 주식 1장 예약 상태로 잠금
→ 영업일 소비 없음
→ 딜 획득 없음
→ 월세 밀림 증가 없음
```

예약은 주식 매수가 아니다.

```text
예약된 주식
→ 포트폴리오에 없음
→ 가치 합산 대상 아님
→ 배당금 발생 대상 아님
→ 미션 정산 대상 아님
```

예약된 주식을 실제로 매수해야 보유 주식이 된다.

---

## 3. 예약 가능 대상

예약 가능:

```text
주식 카드
```

예약 불가:

```text
소모형 자원 카드
이미 예약된 주식
포트폴리오에 있는 주식
```

소모형 자원 카드는 즉시 효과를 발동하고 사라지는 카드이므로 예약할 수 없다.

---

## 4. 예약 수 제한

동시에 예약 가능한 주식은 최대 1장이다.

```text
예약 주식 최대 수: 1
```

이미 예약된 주식이 있으면 새 예약은 불가능하다.

```text
예약 1 / 1
→ 새 예약 불가
→ "이미 예약된 주식이 있습니다" 알림
→ 영업일 소비 없음
```

---

## 5. 예약과 시장 테이프

예약된 주식은 시장 8칸 안에 그대로 남는다.

```text
예약된 주식
→ 시장 슬롯 유지
→ 시장 테이프 진행 영향 없음
→ 시장 테이프 당김 영향 없음
→ 시장 테이프 갱신 영향 없음
```

예약 액션 직후에는 시장 테이프가 진행되지 않는다.
예약은 영업일을 소비하지 않으므로 플레이어는 같은 영업일에 다른 행동을 계속할 수 있다.

---

## 6. 예약과 Mr.Market 제안

Mr.Market 제안은 카드가 아니라 슬롯에 붙는다.
예약은 카드만 고정하고 제안은 고정하지 않는다.

```text
예약 주식이 제안 슬롯 위에 있음
→ 그 분기 동안 해당 슬롯의 현재 제안 조건으로 매수 가능

다음 분기 시작
→ 예약 주식은 같은 슬롯에 남음
→ 기존 제안은 폐기됨
→ 새 분기 제안이 해당 슬롯에 배치될 수 있음
```

예약된 주식이 제안 슬롯 위에 있으면 그 주식이 해당 제안을 점유한다.
다른 카드는 그 제안을 사용할 수 없다.

---

## 7. 예약 처리 순서

예약 액션 처리 순서:

```text
1. 예약 가능 여부 검증
2. 대상 주식을 예약 상태로 표시
3. 시장 슬롯 위치 유지
4. 입력 상태 갱신
```

예약 실패 시:

```text
비용 소비 없음
딜 획득 없음
월세 밀림 증가 없음
영업일 소비 없음
```

---

## 8. 예약된 주식 매수

예약된 주식도 시장 슬롯에 있으므로 시장 카드처럼 매수할 수 있다.

예약된 주식이 Mr.Market 제안 슬롯 위에 있으면 일반 매수가 아니라 현재 제안 조건으로만 매수한다.

처리:

```text
예약된 주식 매수
→ 결제 검증
→ 현재 슬롯의 Mr.Market 제안이 있으면 제안 조건 적용
→ 자원 소비
→ 포트폴리오 추가 또는 호일 처리
→ 예약 상태 해제
→ 해당 시장 슬롯 비움
→ 사용한 제안 소진
→ 시장 테이프 당김
→ 추가 매수권 처리 또는 영업일 종료
```

---

## 9. 호일 완성과 예약

동일 종목 3장을 보유해 호일이 완성되면, 같은 종목의 예약 주식도 제거한다.

```text
호일 완성
→ 덱의 같은 종목 제거
→ 시장의 같은 종목 제거
→ 예약된 같은 종목 제거
→ 예약 수 감소
→ 생긴 시장 빈칸은 시장 테이프 당김
```

이미 호일이 완성된 종목은 이번 게임에서 다시 등장하지 않는다.

---

## 10. 추가 매수권과 예약

추가 매수권 상태에서도 예약할 수 있다.
예약은 추가 매수권을 소비하지 않고 그대로 유지한다.

```text
추가 매수권 보유
→ 예약
→ 추가 매수권 유지
```

추가 매수권은 다음 영업일로 이월되지 않는다.
다음 영업일을 누르면 사용하지 않은 추가 매수권은 폐기된다.

---

## 11. 구현 데이터 제안

예약은 별도 예약 구역이 아니라 시장 슬롯 상태로 관리한다.
별도 `Reservation Panel`과 예약 카드 버튼 목록은 만들지 않는다.
예약 상태와 예약 주식 매수는 시장 슬롯 카드 표면에서 처리한다.

```csharp
public class MarketSlot
{
    public int SlotIndex;
    public MarketCardRuntimeData Card;
    public bool IsReserved;
    public string CurrentMarketOfferId;
}
```

예약 수:

```csharp
int GetReservedStockCount()
{
    return MarketSlots.Count(slot => slot.IsReserved);
}
```

상수:

```csharp
const int MaxReservedStockCount = 1;
```

---

## 12. 구현 시 주의사항

```text
- 예약 구역은 제거한다.
- Reservation Panel은 제거한다.
- 예약은 시장 슬롯 안의 주식을 잠근다.
- 주식 카드만 예약 가능하다.
- 소모형 자원 카드는 예약 불가다.
- 동시 예약은 최대 1장이다.
- 예약은 영업일을 소비하지 않는다.
- 예약은 딜을 지급하지 않는다.
- 예약은 월세 밀림을 증가시키지 않는다.
- 예약 액션 직후 시장 테이프는 진행하지 않는다.
- 예약된 주식은 시장 테이프 진행/당김/갱신의 영향을 받지 않는다.
- 예약은 카드만 보존하고 Mr.Market 제안은 보존하지 않는다.
- 예약된 주식을 매수하면 예약 상태를 해제하고 시장 테이프 당김을 적용한다.
- 호일 완성 시 같은 종목의 예약 주식도 제거한다.
- 추가 매수권 상태의 예약은 추가 매수권을 보존한다.
```
