# 00. 문서 개요

## 1. 문서 목적

이 문서는 주식 포트폴리오 로그라이트 게임의 MVP 시스템 기획서 세트 개요이다.

본 문서의 목적은 다음과 같다.

```text
1. 게임의 핵심 시스템 구조를 명확히 정의한다.
2. Unity 구현 시 필요한 화면 상태, 데이터 구조, 처리 순서, 예외 규칙을 정리한다.
3. Codex 또는 개발자가 기능 단위로 구현할 수 있도록 각 시스템의 책임 범위를 분리한다.
4. 이미 확정된 기획 결정을 기준으로 불필요한 재논의를 줄인다.
```

본 문서는 밸런스 최종 확정 문서가 아니다.
카드별 비용, 가치, 배당금, 분기 목표 수익, 평가 기준값 등은 데이터 테이블에서 조정 가능하게 둔다.

---

## 2. 현재 기준 문서

주식 규칙 대개편의 기준 문서는 다음이다.

```text
22_stock_rules_overhaul.md
```

기존 세부 문서에 남아 있는 옛 규칙과 22번 문서가 충돌하는 경우, 22번 문서를 우선한다.

특히 다음 항목은 22번 문서의 규칙을 따른다.

```text
- 자산 → 주식
- 전문 자원 → 투자 철학
- 리서치/신용/원자재 → 독서/명상/인내
- 운용가치 → 가치
- 운용 수익 → 배당금/수익으로 분리
- 환매 압력 → 월세 밀림
- 런 실패 → 파산
- 시장 3구역 → 1x8 시장
- 예약 구역 → 시장 슬롯 예약
- 중앙 은행 → 소모형 자원 카드
- 카드 상세보기 화면 → 카드 호버 확대
```

## 3. 대상 독자

이 문서는 다음 독자를 대상으로 한다.


## 4. MVP 범위

MVP는 게임의 핵심 판단 구조와 한 런의 진행이 끝까지 작동하는 것을 목표로 한다.

MVP에서 반드시 포함해야 하는 요소:

```text
- 영업일 / 분기 / 회계년도 진행
- 시장 1x8 표시
- 시장 테이프 진행/갱신/당김
- 주식 덱과 소모형 자원 덱
- 주식 매수
- 소모형 자원 카드 매수
- 포트폴리오 8칸 제한
- 동일 주식 3장 호일 합성
- 주식 예약
- 딜 획득 및 사용
- 주식 매도
- 배당금과 수익 기록
- 분기 목표 수익 판정
- 월세 밀림 증가
- 월세 밀림 10 도달 시 파산
- 4Q 휴가
- 최종 정산
```

MVP에서 후순위로 둘 수 있는 요소:

```text
- 고급 연출
- 고급 칩 물리감
- 세밀한 사운드
- 장문의 튜토리얼
- 카드별 상세 기여도 리포트
- 회계년도별 상세 리포트
- 휴가 이벤트
- 복잡한 시장 뉴스
```

## 5. 핵심 용어

| 용어 | 의미 |
|---|---|
| 영업일 | 플레이어가 주요 행동 1회를 수행할 수 있는 최소 시간 단위 |
| 분기 | 여러 영업일로 구성된 중간 목표 단위 |
| 회계년도 | 여러 분기로 구성된 상위 진행 단위 |
| 주식 | 포트폴리오에 들어가는 카드 |
| 소모형 자원 카드 | 매수 시 자원 획득 효과를 발동하고 사라지는 시장 카드 |
| 가치 | 주식이 가진 최종 평가용 점수 |
| 배당금 | 보유 주식이 영업일 시작에 제공하는 현금 |
| 수익 | 분기 목표 판정에 쓰는 누적값. 배당금, 주식 매도 수익, 분기 정산 수익 포함 |
| 조달 현금 | 소모형 현금 카드로 얻은 현금. 수익에는 포함하지 않음 |
| 투자 철학 | 독서, 명상, 인내 칩의 묶음 |
| 딜 | 투자 철학 슬롯을 대체하고 현금 비용을 낮추는 예약 보상 자원 |
| 월세 밀림 | 누적 리스크 수치. 10 이상이면 파산 |
| 시장 테이프 진행 | 새 영업일 시작 시 시장 카드가 왼쪽으로 이동하는 처리 |
| 시장 테이프 갱신 | 분기 시작 시 예약되지 않은 시장 슬롯을 새 카드로 교체하는 처리 |
| 시장 테이프 당김 | 빈칸 발생 시 오른쪽 비예약 카드를 왼쪽으로 압축하는 처리 |
| 예약된 주식 | 시장 슬롯에 고정되어 시장 테이프의 영향을 받지 않는 주식 |
| 호일 주식 | 동일 주식 3장이 합쳐진 무지개 호일 상태의 주식 |

## 6. 용어 사용 원칙

### 6.1 AUM 용어 사용 금지

`AUM`이라는 용어를 사용하지 않는다.

| 사용 금지 | 사용 |
|---|---|
| AUM | 가치 |
| 최종 AUM | 최종 가치 |
| 카드 AUM | 카드 가치 |
| AUM 기준 평가 | 가치 기준 평가 |

### 6.2 옛 자산 용어 사용 금지

포트폴리오에 들어가는 카드는 주식으로 부른다.

| 사용 금지 | 사용 |
|---|---|
| 자산 | 주식 |
| 보유 자산 | 보유 주식 |
| 자산 매수 | 주식 매수 |
| 자산 카드 | 주식 카드 |

### 6.3 옛 리스크 용어 사용 금지

| 사용 금지 | 사용 |
|---|---|
| 환매 압력 | 월세 밀림 |
| 대규모 환매 | 파산 |
| 런 실패 | 파산 / 게임 오버 |

### 6.4 턴과 스테이지 용어 사용 금지

유저-facing 텍스트에서는 `턴`, `스테이지`를 사용하지 않는다.

| 사용 금지 | 사용 |
|---|---|
| 턴 | 영업일 |
| 턴 종료 | 영업일 종료 |
| 스테이지 | 분기 |
| 스테이지 목표 | 분기 목표 |

## 7. 확정된 큰 구조

시간 구조:

```text
영업일 < 분기 < 회계년도
```

회계년도 구성:

```text
1회계년도
- 1Q: 플레이
- 2Q: 플레이
- 3Q: 플레이
- 4Q: 휴가

2회계년도
- 1Q: 플레이
- 2Q: 플레이
- 3Q: 플레이
- 4Q: 휴가

3회계년도
- 1Q: 플레이
- 2Q: 플레이
- 3Q: 플레이
- 4Q: 플레이
```

총 플레이 분기 수는 10개이며, 총 플레이 영업일 수는 44일이다.

## 8. 구현 우선순위 원칙

```text
1. 데이터 구조와 상태 전환을 먼저 구현한다.
2. 기능이 규칙대로 동작하는 것을 우선한다.
3. UI 배치는 Unity 엔진 작업 단계에서 조정한다.
4. 연출은 최소 구현 후 단계적으로 보강한다.
5. 밸런스 수치는 테이블화하여 조정 가능하게 둔다.
```
