# 02. 코어 플레이 루프

## 1. 목적

이 문서는 주식 규칙 개편 이후의 반복 구조를 정의한다.

플레이어는 별도 액션 선택창을 사용하지 않고, 시장 8칸과 포트폴리오를 직접 조작한다.

---

## 2. 최상위 구조

시간 단위:

```text
영업일 < 분기 < 회계년도
```

전체 런:

```text
1회계년도 1Q~3Q 플레이
→ 1회계년도 4Q 휴가
→ 2회계년도 1Q~3Q 플레이
→ 2회계년도 4Q 휴가
→ 3회계년도 1Q~4Q 플레이
→ 최종 정산
```

월세 밀림이 10 이상이 되면 언제든 파산으로 루프를 중단한다.

---

## 3. 분기 루프

분기는 다음 흐름을 가진다.

```text
분기 시작
→ 시장 테이프 갱신
→ 1영업일 시작
→ 영업일 반복
→ 마지막 영업일 종료
→ 분기 마감
→ 다음 일정
```

분기 첫 영업일에는 시장 테이프 갱신만 하고 진행은 생략한다.

---

## 4. 영업일 루프

일반 영업일:

```text
영업일 시작
→ 시장 테이프 진행
→ 보유 주식 배당금 지급
→ 플레이어 입력 대기
→ 행동 확정
→ 영업일 종료
```

분기 첫 영업일:

```text
영업일 시작
→ 시장 테이프 갱신
→ 시장 테이프 진행 없음
→ 보유 주식 배당금 지급
→ 플레이어 입력 대기
```

영업일이 종료되면 남은 영업일을 확인한다.

```text
남은 영업일 > 0
→ 다음 영업일 시작

남은 영업일 == 0
→ 분기 마감
```

---

## 5. 행동 확정 원칙

```text
검토는 무료.
확정은 영업일 소비.
```

무료 행동:

```text
- 카드 호버 정보 확인
- 칩 배치
- 칩 회수
- 주식 매도
```

영업일 소비 행동:

```text
- 주식 매수
- 소모형 자원 카드 매수
- 주식 예약
- 다음 영업일
```

---

## 6. 시장 테이프 루프

시장은 1x8 구조이다.

```text
[1][2][3][4][5][6][7][8]
```

일반 진행:

```text
가장 왼쪽의 비예약 카드 제거
→ 예약된 주식은 고정
→ 비예약 카드가 왼쪽으로 이동
→ 오른쪽 비예약 빈칸에 새 카드 등장
```

빈칸 발생 시:

```text
시장 테이프 당김
```

시장 테이프 당김은 예약된 주식을 움직이지 않고 비예약 카드만 압축한다.

---

## 7. 카드 공급 루프

새 카드는 주식 덱 또는 소모형 자원 덱에서 뽑는다.

```text
75% → 주식 덱
25% → 소모형 자원 덱
```

선택한 덱이 비어 있으면 반대 덱에서 뽑는다.
소모형 자원 덱은 재순환 가능하다.
제거된 주식은 되돌리지 않는다.

---

## 8. 주식 매수 루프

```text
주식 선택
→ 결제 조건 검증
→ 투자 철학/딜 소비
→ 현금 차감
→ 포트폴리오 추가
→ 호일 조건 검사
→ 시장 테이프 당김
→ 추가 매수권 처리 또는 영업일 종료
```

포트폴리오가 8칸 가득 차 있으면 신규 매수는 실패한다.
단, 동일 주식 2장 보유 상태에서 3번째 동일 주식 매수는 호일 예외로 허용한다.

---

## 9. 소모형 자원 카드 루프

```text
소모형 자원 카드 선택
→ 현금 비용 검증
→ 현금 차감
→ 자원 획득 효과 발동
→ 카드 제거
→ 시장 테이프 당김
→ 영업일 종료
```

소모형 자원 카드로 얻은 현금은 수익에 포함하지 않는다.

---

## 10. 예약 루프

```text
주식 예약
→ 시장 슬롯에서 주식 잠금
→ 딜 +1
→ 월세 밀림 +1
→ 월세 밀림 한도 검사
→ 파산하지 않았다면 영업일 종료
```

예약 액션 직후 시장 테이프는 진행하지 않는다.
다음 영업일 시작 시 시장 테이프가 진행된다.

---

## 11. 주식 매도 루프

```text
보유 주식 Card Button 호버
→ Sell Button 표시
→ 매도
→ 현금 획득
→ 수익 기록
→ 포트폴리오에서 제거
→ 이번 게임에서 제거
```

주식 매도는 영업일을 소비하지 않으며 하루 횟수 제한이 없다.
Card Button과 Sell Button 사이로 커서를 옮기는 동안에는 Sell Button을 유지한다.
둘 다 벗어나면 Sell Button을 숨기며, Card Button 클릭은 표시 조건이 아니다.

---

## 12. 분기 마감 루프

```text
분기 마감
→ 분기 마감 정산 수익 반영
→ 분기 수익 확정
→ 목표 달성률 계산
→ 월세 밀림 증가량 계산
→ 월세 밀림 한도 검사
→ 파산하지 않았다면 분기 정산 UI
→ 다음 일정
```

분기 수익:

```text
배당금 + 주식 매도 수익 + 분기 마감 정산 수익
```

---

## 13. 파산 루프

```text
월세 밀림 증가
→ 10 이상 검사
→ 파산
→ 게임 오버 화면
```

파산하면 다음 영업일, 다음 분기, 휴가, 최종 정산으로 진행하지 않는다.

---

## 14. 최종 정산 루프

3회계년도 4Q 마감 후 파산하지 않았다면 최종 정산으로 이동한다.

표시 정보:

```text
- 최종 가치
- 최종 평가
- 총 수익
- 보유 주식 수
- 월세 밀림
- 최종 코멘트
```

최종 평가는 최종 가치 기준이다.

---

## 15. 구현 시 주의사항

```text
- 중앙 은행과 자원 확보 화면을 사용하지 않는다.
- 카드 상세보기 상태를 사용하지 않는다.
- 시장 영역 상태는 Market 하나로 유지한다.
- 시장 테이프 진행은 일반 영업일 시작 시 발생한다.
- 분기 첫 영업일에는 갱신만 하고 진행을 생략한다.
- 모든 시장 빈칸은 시장 테이프 당김으로 처리한다.
- 예약된 주식은 시장 테이프의 영향을 받지 않는다.
- 월세 밀림 한도 도달은 다른 진행보다 우선한다.
```
