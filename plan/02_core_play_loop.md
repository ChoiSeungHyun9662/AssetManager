# 02. 코어 플레이 루프

## 1. 목적

이 문서는 게임의 반복 구조를 정의한다.

본 게임은 액션 선택창을 통해 행동을 고르는 구조가 아니다.
플레이어는 보드 위의 시장 카드, 예약 카드, 중앙 은행, 다음 영업일 버튼을 직접 조작한다.
해당 조작이 특정 조건을 만족해 행동으로 확정되면, 그때 영업일을 소비한다.

이 문서는 다음 내용을 정의한다.

```text
- 전체 런 루프
- 회계년도 루프
- 분기 루프
- 영업일 루프
- 행동 확정 타이밍
- 영업일 소비 조건
- 분기 마감 진입 조건
- 런 종료 조건
```

---

## 2. 최상위 진행 구조

게임의 시간 단위는 다음 계층을 가진다.

```text
영업일 < 분기 < 회계년도
```

각 단위의 의미는 다음과 같다.

| 단위 | 의미 |
|---|---|
| 영업일 | 플레이어가 한 번의 주요 행동을 수행할 수 있는 최소 단위 |
| 분기 | 여러 영업일로 구성된 중간 목표 단위 |
| 회계년도 | 여러 분기로 구성된 상위 진행 단위 |
| 런 | 1회계년도부터 3회계년도까지 이어지는 전체 플레이 |

전체 런은 다음 구조를 따른다.

```text
1회계년도
→ 1Q
→ 2Q
→ 3Q
→ 4Q 휴가

2회계년도
→ 1Q
→ 2Q
→ 3Q
→ 4Q 휴가

3회계년도
→ 1Q
→ 2Q
→ 3Q
→ 4Q
→ 최종 정산
```

---

## 3. 전체 런 루프

전체 런은 다음 순서로 진행된다.

```text
런 시작
→ 1회계년도 시작
→ 1Q 시작
→ 영업일 반복
→ 분기 마감
→ 다음 분기 또는 회계년도 전환
→ 3회계년도 4Q 마감
→ 최종 정산
```

단, 진행 중 환매 압력이 한도에 도달하면 즉시 런 실패가 발생한다.

```text
환매 압력 >= 10
→ 대규모 환매 발생
→ 런 실패
```

환매 압력에 의한 실패는 다음 상황에서 발생할 수 있다.

```text
- 예약 액션 직후
- 분기 마감에서 목표 미달 환매 압력 증가 직후
```

---

## 4. 회계년도 루프

회계년도는 여러 분기로 구성된다.

### 4.1 1·2회계년도

1회계년도와 2회계년도는 각각 1Q, 2Q, 3Q만 플레이한다.
4Q는 휴가 화면으로 처리한다.

```text
1회계년도
- 1Q 플레이
- 2Q 플레이
- 3Q 플레이
- 4Q 휴가

2회계년도
- 1Q 플레이
- 2Q 플레이
- 3Q 플레이
- 4Q 휴가
```

4Q 휴가는 플레이 분기가 아니다.

```text
4Q 휴가에서는:
- 영업일 없음
- 자산 매수 없음
- 자원 확보 없음
- 예약 없음
- 분기 마감 시나리오 없음
- 보너스 없음
- 패널티 없음
```

4Q 휴가 화면은 해당 회계년도의 요약을 보여준 뒤 다음 회계년도로 넘어간다.

---

### 4.2 3회계년도

3회계년도는 1Q부터 4Q까지 모두 플레이한다.

```text
3회계년도
- 1Q 플레이
- 2Q 플레이
- 3Q 플레이
- 4Q 플레이
```

3회계년도 4Q 마감 후에는 휴가가 아니라 최종 정산으로 이동한다.

---

## 5. 분기 루프

각 분기는 다음 흐름을 가진다.

```text
분기 시작
→ 분기 설정
→ 영업일 시작
→ 영업일 반복
→ 마지막 영업일 종료
→ 분기 마감 정산
→ 다음 일정 진행
```

분기 시작 시 설정되는 요소는 다음과 같다.

```text
- 현재 회계년도
- 현재 분기
- 분기 목표 운용 수익
- 분기 남은 영업일 수
- 분기 마감 시나리오 또는 시장 국면
- 시장 테이프 처리
```

분기 안에서는 영업일을 하나씩 소비한다.

```text
영업일 시작
→ 운용 수익 발생
→ 플레이어 조작
→ 행동 확정
→ 영업일 종료
```

남은 영업일이 0이 되면 분기 마감으로 진입한다.

---

## 6. 영업일 루프

영업일은 본 게임의 최소 행동 단위이다.

영업일 시작 시 다음 처리를 한다.

```text
영업일 시작
→ 보유 자산의 영업일 시작 현금 발생
→ 시장 영역을 디폴트 시장 상태로 설정
→ 플레이어 조작 가능 상태 진입
```

플레이어는 액션 선택창 없이 보드 위 요소를 직접 조작한다.

가능한 주요 조작은 다음과 같다.

```text
- 시장 카드 클릭
- 예약 카드 클릭
- 중앙 은행 클릭
- 다음 영업일 클릭
```

각 조작은 즉시 영업일을 소비하지 않는다.
실제 영업일 소비는 행동이 확정되는 순간에 발생한다.

---

## 7. 행동 확정 원칙

핵심 원칙은 다음과 같다.

```text
행동을 검토하는 것은 영업일을 소비하지 않는다.
행동을 확정하는 순간 영업일을 소비한다.
```

예를 들어 카드 클릭은 무료이다.

```text
시장 카드 클릭
→ 카드 상세보기 진입
→ 영업일 소비 없음
```

하지만 매수 확정은 영업일을 소비한다.

```text
카드 상세보기
→ 매수 확정
→ 영업일 1일 소비
```

자원 확보도 마찬가지이다.

```text
중앙 은행 클릭
→ 자원 확보 화면 진입
→ 아직 영업일 소비 없음

자원 확보 화면에서 첫 자원 클릭
→ 자원 즉시 획득
→ 자원 확보 행동 확정
→ 이후 취소 불가
```

---

## 8. 행동별 영업일 소비 타이밍

### 8.1 자산 매수

```text
카드 클릭
→ 카드 상세보기
→ 칩 배치
→ 매수 확정
→ 영업일 소비
```

영업일을 소비하는 시점은 `매수 확정`이다.

```text
매수 확정 전 닫기
→ 영업일 소비 없음
```

---

### 8.2 카드 예약

예약 확인 상태는 존재하지 않는다.

```text
시장 카드 클릭
→ 카드 상세보기
→ 예약 버튼 클릭
→ 예약 즉시 실행
→ 영업일 소비
```

영업일을 소비하는 시점은 `예약 버튼 클릭`이다.

예약 버튼은 시장 카드 상세보기에서만 표시한다.
예약 카드 상세보기에서는 예약 버튼을 표시하지 않는다.

---

### 8.3 자원 확보

```text
중앙 은행 클릭
→ 자원 확보 화면 진입
→ 첫 자원 클릭
→ 해당 자원 즉시 획득
→ 자원 확보 행동 확정
```

첫 자원 획득 전에는 닫기가 가능하다.

```text
자원 확보 진입
→ 아직 자원 획득 없음
→ 닫기
→ 시장 디폴트 상태 복귀
→ 영업일 소비 없음
```

자원을 하나라도 획득하면 닫기는 불가능하다.

```text
자원 1개 이상 획득
→ 닫기 비활성화
→ 자원 확보가 완료될 때까지 진행
```

영업일은 자원 확보 조합이 완성되거나, 더 이상 유효 선택지가 없을 때 종료된다.

```text
같은 기본 자원 2개 획득
또는
서로 다른 기본 자원 3개 획득
또는
전문 자원 한도 때문에 더 이상 유효 선택지가 없음
→ 영업일 종료
```

---

### 8.4 다음 영업일

```text
다음 영업일 클릭
→ 현재 영업일 종료
→ 남은 영업일 1일 소비
```

다음 영업일 버튼은 시장 영역이 `시장` 상태일 때만 활성화된다.

```text
시장 상태
→ 다음 영업일 활성화

카드 상세보기 상태
→ 다음 영업일 비활성화

자원 확보 상태
→ 다음 영업일 비활성화
```

---

## 9. 시장 영역 상태와 루프 관계

시장 영역은 세 가지 상태를 가진다.

```text
1. 시장
2. 카드 상세보기
3. 자원 확보
```

### 9.1 시장 상태

기본 상태이다.
플레이어는 시장 카드, 예약 카드, 중앙 은행, 다음 영업일을 조작할 수 있다.

```text
시장 상태에서 가능한 주요 입력:
- 시장 카드 클릭
- 예약 카드 클릭
- 중앙 은행 클릭
- 다음 영업일 클릭
```

---

### 9.2 카드 상세보기 상태

시장 카드 또는 예약 카드를 클릭하면 진입한다.
카드 상세보기는 시장 영역을 대체한다.

```text
카드 클릭
→ 카드 상세보기 상태
```

닫기를 누르면 시장 상태로 복귀한다.

```text
카드 상세보기
→ 닫기
→ 시장 상태
→ 영업일 소비 없음
```

매수 또는 예약을 확정하면 영업일을 소비한다.

---

### 9.3 자원 확보 상태

중앙 은행을 클릭하면 진입한다.
자원 확보 화면은 시장 영역을 대체한다.

```text
중앙 은행 클릭
→ 자원 확보 상태
```

첫 자원 획득 전에는 닫기가 가능하다.
첫 자원 획득 후에는 닫기가 불가능하다.

---

## 10. 운용 수익 발생 타이밍

보유 자산의 운용 수익은 영업일 시작마다 발생한다.

```text
영업일 시작
→ 보유 자산의 영업일 시작 현금 발생
```

분기 첫 영업일에도 운용 수익은 정상 발생한다.

```text
분기 시작
→ 1영업일 시작
→ 운용 수익 발생
```

다음 영업일 버튼으로 영업일을 넘긴 경우에도, 다음 영업일이 존재한다면 운용 수익은 정상 발생한다.

```text
다음 영업일 클릭
→ 현재 영업일 종료
→ 다음 영업일 시작
→ 운용 수익 발생
```

단, 마지막 영업일을 종료해 다음 영업일이 없으면 운용 수익은 발생하지 않고 분기 마감으로 이동한다.

```text
마지막 영업일 종료
→ 다음 영업일 없음
→ 운용 수익 없음
→ 분기 마감
```

---

## 11. 영업일 종료 처리

영업일이 종료되면 다음 순서로 처리한다.

```text
영업일 종료
→ 남은 영업일 감소
→ 남은 영업일 확인
```

남은 영업일이 있으면 다음 영업일을 시작한다.

```text
남은 영업일 > 0
→ 다음 영업일 시작
→ 운용 수익 발생
→ 시장 상태 진입
```

남은 영업일이 없으면 분기 마감으로 이동한다.

```text
남은 영업일 == 0
→ 분기 마감 정산
```

---

## 12. 분기 마감 진입 조건

분기 마감은 해당 분기의 마지막 영업일이 종료되었을 때 발생한다.

```text
마지막 영업일 종료
→ 분기 마감
```

분기 마감에서는 다음 처리를 한다.

```text
태그별 정산 애니메이션
→ 정산 수익 반영
→ 분기 운용 수익 확정
→ 목표 달성률 계산
→ 환매 압력 증가량 계산
→ 환매 압력 한도 검사
→ 분기 정산 UI 표시
→ 다음 일정 진행
```

---

## 13. 분기 후 다음 일정

분기 마감 후에는 현재 회계년도와 분기에 따라 다음 일정으로 이동한다.

### 13.1 1·2회계년도 1Q 또는 2Q 마감

```text
다음 분기 시작
→ 시장 테이프 진행
→ 새 분기 시작
```

### 13.2 1·2회계년도 3Q 마감

```text
4Q 휴가 화면
→ 회계년도 요약 표시
→ 다음 회계년도 시작
→ 시장 테이프 갱신
```

### 13.3 3회계년도 1Q~3Q 마감

```text
다음 분기 시작
→ 시장 테이프 진행
→ 새 분기 시작
```

### 13.4 3회계년도 4Q 마감

```text
최종 정산
```

---

## 14. 시장 테이프 처리와 루프 관계

시장 테이프에는 두 가지 주요 처리가 있다.

```text
시장 테이프 진행
시장 테이프 갱신
```

### 14.1 시장 테이프 진행

시장 테이프 진행은 시장이 한 단계 흘러가는 처리이다.

```text
1. 매도 임박 카드 제거
2. 현재 시장 카드 → 매도 임박으로 이동
3. 예비 시장 카드 → 현재 시장으로 이동
4. 예비 시장 빈 슬롯 새 카드로 보충
```

발생 시점:

```text
- 같은 회계년도 내 다음 분기 시작
- 예약 액션 실행 후
```

---

### 14.2 시장 테이프 갱신

시장 테이프 갱신은 시장 전체를 새로 구성하는 처리이다.

```text
기존 시장 테이프 카드 전부 제거
→ 매도 임박 / 현재 시장 / 예비 시장 슬롯을 새 카드로 전부 구성
```

발생 시점:

```text
- 회계년도 시작
```

---

## 15. 추가 매수권과 루프

자산 매수 후 특정 규칙에 의해 추가 매수권이 발생할 수 있다.

추가 매수권은 다음 성격을 가진다.

```text
- 자산 매수 전용
- 다음 영업일로 이월되지 않음
- 중첩되지 않음
- 사용하지 않고 다음 영업일로 넘길 수 있음
```

추가 매수권이 발생하면 영업일을 즉시 종료하지 않는다.

```text
첫 번째 자산 매수
→ 추가 매수권 발생
→ 추가 매수 선택 상태
```

추가 매수 상태에서 가능한 행동은 다음뿐이다.

```text
- 자산 매수
- 다음 영업일
```

불가능한 행동:

```text
- 자원 확보
- 카드 예약
```

추가 매수로 두 번째 자산을 매수하면 즉시 영업일을 종료한다.

```text
추가 매수권 사용
→ 두 번째 자산 매수
→ 추가 매수권 소모
→ 영업일 종료
```

다음 영업일을 누르면 추가 매수권을 포기하고 영업일을 종료한다.

```text
추가 매수권 보유
→ 다음 영업일
→ 추가 매수권 폐기
→ 영업일 종료
```

---

## 16. 런 실패 루프

환매 압력이 10 이상이 되면 즉시 런 실패가 발생한다.

```text
환매 압력 증가
→ 한도 검사
→ 10 이상
→ 런 실패
```

환매 압력 증가는 다음 상황에서 발생한다.

```text
- 예약 액션
- 분기 목표 미달
```

런 실패가 발생하면 이후 영업일, 분기, 회계년도 진행은 중단한다.

```text
런 실패
→ 실패 화면 표시
→ 최종 정산으로 가지 않음
```

---

## 17. 최종 정산 루프

3회계년도 4Q 마감 후, 환매 압력으로 인한 실패가 발생하지 않았다면 최종 정산으로 이동한다.

```text
3회계년도 4Q 마감
→ 최종 정산
```

최종 정산에서 표시하는 정보는 다음과 같다.

```text
- 최종 운용가치
- 최종 평가
- 총 운용 수익
- 보유 자산 수
- 환매 압력
- 운용 코멘트
```

최종 평가는 최종 운용가치 기준으로 산정한다.

---

## 18. 루프 요약

전체 루프는 다음과 같이 요약할 수 있다.

```text
런 시작
→ 회계년도 시작
→ 시장 테이프 갱신
→ 분기 시작
→ 영업일 시작
→ 운용 수익 발생
→ 플레이어 직접 조작
→ 행동 확정
→ 영업일 종료
→ 남은 영업일 있으면 다음 영업일
→ 남은 영업일 없으면 분기 마감
→ 다음 분기 또는 회계년도 전환
→ 3회계년도 4Q 이후 최종 정산
```

환매 압력에 의한 실패는 언제든 루프를 중단할 수 있다.

```text
환매 압력 10 이상
→ 즉시 런 실패
```

```csharp
void StartBusinessDay()
```

### 19.3 영업일 종료 예시

```csharp
void EndBusinessDay()
{
    RemainingBusinessDays--;

    if (RemainingBusinessDays <= 0)
    {
        ResolveQuarterEnd();
        return;
    }

    StartBusinessDay();
}
```

```csharp
void ConfirmActionAndEndBusinessDay()

    ResolveCurrentAction();

    EndBusinessDay();
}
```

## 20. 구현 시 주의사항

```text
- 카드 클릭은 영업일을 소비하지 않는다.
- 카드 상세보기 닫기는 영업일을 소비하지 않는다.
- 자원 확보 화면 진입은 영업일을 소비하지 않는다.
- 자원 확보에서 첫 자원을 클릭하는 순간 행동이 확정된다.
- 매수 확정은 영업일을 소비한다.
- 예약 버튼 클릭은 즉시 예약 액션이며 영업일을 소비한다.
- 다음 영업일 버튼은 시장 상태에서만 활성화된다.
- 환매 압력 한도 도달은 다른 진행보다 우선한다.
- 추가 매수권이 발생하면 첫 번째 매수 직후 영업일을 종료하지 않는다.
```
