# 23. 재미 강화: 미션과 Mr.Market 제안

## 1. 목적

이 문서는 MVP 이후 재미 강화를 위한 큰 방향을 정리한다.

핵심 목표:

```text
- 카드 자체 효과를 늘리지 않고 빌드 재미를 강화한다.
- 산업 태그와 미션으로 이번 런의 투자 논문을 만든다.
- Mr.Market 제안으로 1x8 시장 테이프의 매수 타이밍 판단을 강화한다.
- 고점의 주역은 미션으로 두고, Mr.Market은 매입 품질을 높이는 보조 축으로 둔다.
```

이 문서는 매도 타이밍 설계 전까지의 합의 사항이다.
다음 논의는 별도 문서 또는 후속 섹션에서 `매도 타이밍 설계`로 이어간다.

---

## 2. 재미 구조

각 시스템의 역할:

```text
태그 = 산업 노출
미션 = 이번 런의 투자 논문
Mr.Market 제안 = 매수 타이밍과 매입 조건
호일 = 매입 흔적을 압축하는 고점 장치
예약 = 카드만 시장 슬롯에 고정하는 무료 보험
단일 1x8 시장 = 주식, 자원, 제안이 충돌하는 공간
```

큰 방향:

```text
- 미션은 정답 빌드가 아니라 위험한 투자 논문을 제안한다.
- 둘 다 가능하되, 고점은 집중 투자 쪽에 둔다.
- 분산형 미션은 안정적이고 빨리 켜지는 방향이다.
- 집중형/호일형 미션은 늦게 켜지지만 후반 고점이 높다.
```

---

## 3. 태그

주식 카드는 산업/테마 태그 1개를 가진다.

태그 5종:

```text
- 기술(Technology)
- 소비재(Consumer)
- 에너지(Energy)
- 금융(Financials)
- 공업(Industrials)
```

카드 자체 효과는 추가하지 않는다.
태그는 미션 조건과 정산 공식, 포트폴리오 방향성을 만드는 데 사용한다.

---

## 4. 미션 기본 구조

미션은 한 런에 하나만 확정된다.

```text
런 시작 직후 미션 후보 3장 생성
각 후보 슬롯은 언제든 1회 멀리건 가능
이미 멀리건을 쓴 슬롯은 수동 폐기 가능
수동 폐기한 후보 슬롯은 빈 슬롯으로 남고 보충하지 않음
클리어 조건을 만족한 순간 해당 미션으로 즉시 확정
확정 즉시 나머지 후보는 폐기
확정 후 후보 UI는 사라지고 확정 미션 1장만 표시
```

동시에 여러 후보가 조건을 만족하면 왼쪽 슬롯부터 우선한다.

미션은 다음 두 요소를 분리한다.

```text
클리어 조건 = 최초 확정 여부만 판단
정산 공식 = 확정 후 매 분기말 미션 수익 계산
```

확정된 미션도 매 분기말 현재 포트폴리오 기준으로 다시 정산한다.
미션이 확정된 같은 분기의 분기말부터 정산 대상이 된다.
정산 공식 조건을 만족하지 못하면 그 분기 미션 수익은 0이다.

---

## 5. 분기 평가와 미션 수익

용어:

```text
분기 평가 목표
= 월세 밀림을 피하기 위해 분기말까지 채워야 하는 평가 기준

현금 흐름
= 배당 + 매도 현금

미션 수익
= 확정 미션이 분기말 포트폴리오 기준으로 만든 평가 수익

월세 밀림 판정
= 현금 흐름 + 미션 수익
```

미션 수익은 현금과 아무 상관없는 평가 개념이다.
미션 수익은 내부적으로 별도 관리하지만, 유저에게는 최종 가치에 섞어서 표시한다.
분기말 정산 UI에서는 별도 행으로 `미션 수익 +N`을 보여준다.

미션 수익은 월세 밀림 판정에 항상 포함한다.

## 6. 영업일 수 변경

미션과 Mr.Market 제안을 넣으면 기존 4/5영업일은 지나치게 압축적이다.
계획과 타이밍 판단을 살리기 위해 분기당 영업일을 늘린다.

```text
1회계년도: 분기당 8영업일
2회계년도: 분기당 8영업일
3회계년도: 분기당 10영업일
```

영업일 증가에 따라 배당 기반 현금 흐름과 분기 평가 목표는 별도 밸런싱 대상이다.

## 7. 미션 카드 풀

미션은 1태그 중심이며, 2태그 미션은 일부만 둔다.

1차 풀 규모:

```text
1태그 미션
- 빠른 진입형: 5장
- 집중형: 5장
- 호일형: 5장
- 고가치형: 5장

2태그 안정형
- 기획적으로 어울리는 조합 5장

총 25장
```

미션 후보 3장 추첨 시 같은 태그 미션이 여러 장 나와도 허용한다.
특정 태그 런의 유혹을 만들 수 있기 때문이다.

난이도 표시는 자동 산정 초안을 만들 수 있지만, 최종적으로는 기획자가 직접 수정 가능한 디스플레이 전용 정보다.
난이도는 시스템 조건이나 보상 수치에 직접 영향을 주지 않는다.

미션 이름은 투자 논문형 톤으로 작성한다.

예:

```text
기술 독점론
에너지 슈퍼사이클
금융 재평가
공업 회복장
소비재 방어선
```

## 8. 미션 문법

미션의 클리어 조건 대상과 정산 공식 대상은 서로 달라도 된다.
정산 공식은 대상 태그 카드뿐 아니라 모든 보유 카드를 대상으로 할 수 있다.

빠른 진입형:

```text
클리어 조건: 대상 태그 3장 이상
정산 공식: 대상 태그 카드 1장당 +N
보상 체급: 낮음
역할: 일찍 켜서 여러 분기 누적
```

집중형:

```text
클리어 조건: 대상 태그 5장 이상
정산 공식: 대상 태그 카드 1장당 +N
보상 체급: 중간 이상
역할: 집중 투자 고점
```

호일형:

```text
클리어 조건: 대상 태그 호일 1장 이상
정산 공식: 대상 태그 모든 카드 +N, 호일은 추가 +M
보상 체급: 높음
역할: 적은 비중의 고점 미션
```

고가치형:

```text
클리어 조건: 대상 태그 최고 가치 카드 1장 기준
정산 공식: 대상 태그 카드 가치 합계 일부 반영
보상 체급: 중간 이상
역할: Mr.Market 성격 변환형과 결합하는 고점
```

2태그 안정형:

```text
클리어 조건: 두 태그 각각 2장 이상
정산 공식: 두 대상 태그 카드 1장당 +N
보상 체급: 낮음~중간
역할: 분산형 안정 전략
```

미션 정산과 클리어 조건은 Mr.Market 영구 가치 델타를 반영한다.

## 9. Mr.Market 제안 기본 구조

`시장 상황`이라는 별도 시스템은 만들지 않는다.
Mr.Market 제안이 곧 시장 상황이다.

분기 시작 처리:

```text
시장 테이프 갱신
→ Mr.Market 제안 3개 생성
→ 역할별 구간에 1개씩 배치
```

제안 역할과 배치:

```text
1~2번 슬롯: 결제 완화형
3~5번 슬롯: 성격 변환형
6~8번 슬롯: 템포형
```

제안 규칙:

```text
- 제안은 공개 정보다.
- 제안은 주식 카드에만 적용된다.
- 제안은 특정 태그나 특정 미션 조건을 요구하지 않는다.
- 소모형 자원 카드 위의 제안은 비활성으로 보이며 소진되지 않는다.
- 제안 슬롯 위 주식은 일반 매수할 수 없고, 반드시 제안 조건으로만 매수한다.
- 제안은 매수 성공 시에만 소진된다.
- 실패한 시도는 비용, 제안, 영업일, 추가 매수권을 바꾸지 않는다.
- 분기 종료 시 남은 제안은 폐기된다.
- 런 중 같은 이름의 제안이 반복 출현할 수 있다.
```

제안 3개는 매 분기 각 역할군에서 1개씩 랜덤 추첨한다.
후반일수록 고강도 제안이 나올 확률을 높인다.

## 10. 예약과 Mr.Market

예약은 무료 행동이다.
동시에 1장만 예약할 수 있다.

예약은 카드만 고정한다.
제안 자체를 예약하지 않는다.

```text
예약 카드가 제안 슬롯 위에 있으면
→ 그 분기 동안 해당 카드는 그 슬롯 제안 조건으로 매수된다.

다음 분기로 넘어가면
→ 예약 카드는 같은 슬롯에 남는다.
→ 기존 제안은 폐기된다.
→ 새 분기 제안이 슬롯에 다시 배치된다.
```

따라서 예약은 카드만 시장 지형 위에 고정하는 도구다.

예약된 카드가 제안 슬롯을 점유하면 해당 제안은 사실상 그 카드가 점유한다.
다른 카드는 그 제안을 사용할 수 없다.

---

## 11. Mr.Market 제안 타입

제안 타입은 세 가지다.

```text
결제 완화형
- 결제 순간에만 적용
- 영구 효과 없음

성격 변환형
- 비용 변화 가능
- 가치/배당 영구 델타 가능
- 영구 델타는 별도 매입 흔적으로 저장

템포형
- 추가 매수권 또는 특정 투자 철학 보너스 획득
- 비용 변화 없음
- 가치/배당 영구 델타 없음
```

영구 효과 제한:

```text
- 영구 제안 효과는 오직 가치 변화와 배당 변화만 사용한다.
- 최종 가치와 최종 배당은 0 미만으로 내려가지 않는다.
- 호일 합성 시 재료 3장의 영구 가치/배당 델타를 합산한다.
- 호일 배당 하한은 재료 효과 합산 후 최종 적용한다.
```

Mr.Market 영구 가치/배당 델타는 최종 정산 가치와 미션 정산에 반영한다.
매도 가격에는 반영하지 않는다.

---

## 12. 추가 매수권

템포형 제안은 추가 매수권을 줄 수 있다.

추가 매수권 상태에서 가능한 선택:

```text
- 주식 매수 → 추가 매수권 소모
- 소모형 자원 카드 매수 → 추가 매수권 소모
- 예약 → 가능, 추가 매수권 유지
- 다음 영업일 → 추가 매수권 폐기
```

추가 매수권으로 매수하는 주식은 시장 주식과 예약 주식을 모두 포함한다.

추가 매수권은 다음 영업일로 이월되지 않는다.
추가 매수로 산 주식도 매수한 영업일에는 배당금을 지급하지 않는다.

템포형 연쇄는 가능하다.

```text
추가 매수권으로 산 주식이 다시 템포형 제안을 밟으면
→ 새 추가 매수권을 얻을 수 있다.
```

시스템이 자동으로 연쇄를 종료하지 않는다.
플레이어가 주식 또는 소모형 자원 카드를 추가 매수하거나 다음 영업일을 눌러 수동 종료한다.

시장 당김으로 새로 들어온 카드도 즉시 추가 매수 후보가 될 수 있다.
한 영업일에 여러 제안이 소진될 수 있다.

밸런스는 연쇄 횟수 제한보다 제안 출현, 위치, 수치로 조정한다.

---

## 13. 표시 원칙

시장 카드에는 다음 정보를 표시한다.

```text
- 핵심 숫자
- 태그
- 제안 배지
- 모든 수정이 반영된 최종 정보
```

호버 및 호가인 모달에는 원본 정보와 변경 후 최종 정보를 표시한다.
어떤 사유로 변경되었는지는 표시하지 않는다.

```text
원본 가치 / 최종 가치
원본 배당 / 최종 배당
원본 비용 / 최종 비용
```

성격 변환형 영구 효과는 카드 배지로 표시한다.
제안 설명은 문장보다 수식 중심으로 표시한다.

예:

```text
현금 +1 / 가치 +2 / 배당 -1
독서 -1
추가 매수권 +1 / 명상 +1
```

제안 슬롯 위 주식 매수는 확인 모달을 보여준다.
확인 모달에서는 제안 조건과 적용 후 수치를 확인한다.

## 14. Mr.Market 제안 풀 초안

### 14.1 결제 완화형

결제 완화형은 4종만 사용하고 반복 출현을 허용한다.

| ID | 효과 |
|---|---|
| P01 | 현금 비용 -1 |
| P02 | 독서 비용 -1 |
| P03 | 명상 비용 -1 |
| P04 | 인내 비용 -1 |

비용 감소는 0 미만으로 내려가지 않는다.

### 14.2 성격 변환형

성격 변환형은 밸런스 검토용 초안 20종을 사용한다.
기획자가 수치와 유지 여부를 직접 수정한다.

| ID | 효과 |
|---|---|
| S01 | 현금 비용 +0 / 가치 +1 / 배당 -1 |
| S02 | 현금 비용 +1 / 가치 +2 / 배당 -1 |
| S03 | 현금 비용 +1 / 가치 +3 / 배당 -2 |
| S04 | 현금 비용 +2 / 가치 +4 / 배당 -2 |
| S05 | 현금 비용 +2 / 가치 +5 / 배당 -3 |
| S06 | 현금 비용 +0 / 가치 -1 / 배당 +1 |
| S07 | 현금 비용 +1 / 가치 -1 / 배당 +2 |
| S08 | 현금 비용 +1 / 가치 -2 / 배당 +2 |
| S09 | 현금 비용 +2 / 가치 -2 / 배당 +3 |
| S10 | 현금 비용 +2 / 가치 -3 / 배당 +3 |
| S11 | 현금 비용 +1 / 가치 +2 / 배당 +0 |
| S12 | 현금 비용 +2 / 가치 +3 / 배당 +0 |
| S13 | 현금 비용 +2 / 가치 +4 / 배당 +0 |
