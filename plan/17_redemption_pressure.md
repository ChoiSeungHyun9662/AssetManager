# 17. 월세 밀림

## 1. 목적

이 문서는 월세 밀림 시스템의 역할, 증가 요인, 한도, 파산 판정, 최종 코멘트와의 관계를 정의한다.

월세 밀림은 기존 환매 압력의 새 이름이다.
월세 밀림이 한도에 도달하면 파산으로 게임 오버가 발생한다.

```text
월세 밀림 >= 10
→ 파산
→ 게임 오버
```

---

## 2. 월세 밀림의 역할

월세 밀림은 다음 역할을 가진다.

```text
- 분기 평가 목표 미달에 대한 누적 페널티
- 파산 조건
- 최종 코멘트의 보조 지표
```

플레이어는 높은 가치를 노리면서도 월세 밀림이 10에 도달하지 않도록 관리해야 한다.

---

## 3. 한도

월세 밀림 한도는 10이다.

```text
월세 밀림 0~9
→ 계속 진행

월세 밀림 10 이상
→ 파산
```

구현 상수 예시:

```csharp
const int MaxRentArrears = 10;
```

---

## 4. 증가 요인

월세 밀림은 다음 상황에서 증가한다.

```text
1. 분기 평가 목표 미달
```

월세 밀림은 증가 즉시 한도 검사를 수행한다.

```text
월세 밀림 증가
→ 즉시 한도 검사
```

---

## 5. 예약과 월세 밀림

v3 기준 예약은 무료 행동이며 월세 밀림을 증가시키지 않는다.

```text
주식 예약
→ 영업일 소비 없음
→ 딜 획득 없음
→ 월세 밀림 증가 없음
```

---

## 6. 분기 목표 미달로 인한 증가

현금 흐름과 미션 수익의 합이 분기 평가 목표에 미달하면 목표 달성률에 따라 월세 밀림이 증가한다.

분기 평가 기준:

```text
현금 흐름 + 미션 수익
= 배당금
+ 주식 매도 현금
+ 미션 수익
```

제외:

```text
소모형 현금 카드로 얻은 현금
```

증가량:

```text
수익 >= 목표
→ +0

목표의 75% 이상 100% 미만
→ +1

목표의 50% 이상 75% 미만
→ +2

목표의 50% 미만
→ +3
```

표:

| 목표 달성률 | 월세 밀림 증가 |
|---:|---:|
| 100% 이상 | +0 |
| 75% 이상 100% 미만 | +1 |
| 50% 이상 75% 미만 | +2 |
| 50% 미만 | +3 |

---

## 7. 파산 처리

월세 밀림이 10 이상이면 즉시 파산한다.

```text
월세 밀림 증가
→ 10 이상 검사
→ 파산
```

파산이 발생하면 다음 흐름으로 이동하지 않는다.

```text
- 다음 영업일 시작 없음
- 다음 분기 시작 없음
- 4Q 휴가 없음
- 최종 정산 없음
```

파산 화면의 실패 사유:

```text
월세 밀림
```

또는 더 코믹한 문구:

```text
월세가 밀려 파산했습니다.
```

---

## 8. 분기 정산 UI

분기 마감에서 월세 밀림이 증가했지만 파산하지 않았다면, 분기 정산 UI에 증가량과 현재값을 표시한다.

성공 예시:

```text
현금 흐름: 21
미션 수익: +3
분기 평가 목표: 20
달성률: 120%

월세 밀림 증가: 없음
현재 월세 밀림: 4 / 10
```

실패 예시:

```text
현금 흐름: 11
미션 수익: +3
분기 평가 목표: 20
달성률: 70%

월세 밀림 증가: +2
현재 월세 밀림: 6 / 10
```

## 9. 예약 UI

예약 입력에는 슬롯 고정과 예약 한도를 표시한다.

```text
예약
카드 고정
```

MVP에서는 버튼 또는 툴팁의 짧은 표기만으로 충분하다.

## 10. 최종 코멘트

최종 평가 등급은 최종 가치 기준으로 산정한다.
월세 밀림은 최종 등급을 낮추지 않는다.

월세 밀림은 최종 코멘트에 사용할 수 있다.

```text
최종 평가 등급 × 월세 밀림 단계
```

월세 밀림 단계 초기값:

```text
낮음: 0~2
중간: 3~5
높음: 6+
```

기준은 데이터 테이블로 조정 가능하게 둔다.

## 12. 증가 함수 예시

```csharp
void AddRentArrears(int amount)
{
    if (amount <= 0)
        return;

    CurrentRentArrears += amount;

    if (CurrentRentArrears >= MaxRentArrears)
    {
        TriggerBankruptcy();
        return;
    }
}
```

## 13. 구현 시 주의사항

```text
- 환매 압력이라는 용어를 사용하지 않는다.
- 월세 밀림 한도는 10이다.
- 월세 밀림 10 이상이면 즉시 파산이다.
- 주식 예약은 월세 밀림을 증가시키지 않는다.
- 분기 평가 목표 미달은 달성률에 따라 월세 밀림 +1/+2/+3을 발생시킨다.
- 월세 밀림 증가 후 즉시 한도 검사를 한다.
- 파산 시 이후 영업일/분기/회계년도 진행을 중단한다.
- 최종 평가 등급은 월세 밀림으로 낮아지지 않는다.
- 월세 밀림은 최종 코멘트에 사용할 수 있다.
```
