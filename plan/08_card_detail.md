# 08. 카드 호버 확대

## 1. 목적

이 문서는 기존 카드 상세보기 화면을 대체하는 카드 호버 확대 규칙을 정의한다.

카드 상세보기 화면은 제거한다.
카드를 클릭하거나 선택해 시장 영역을 대체하는 별도 상태로 들어가지 않는다.

---

## 2. 핵심 규칙

```text
시장 카드에 마우스 호버
→ 확대 카드 표시
→ 호버 종료
→ 확대 카드 숨김
```

호버 확대는 영업일을 소비하지 않는다.
시장 테이프에도 영향을 주지 않는다.

---

## 3. 제공 정보

시장에 표시된 카드와 호버 확대 카드가 제공하는 정보는 동일하다.
스케일만 다르다.

### 3.1 주식 카드

```text
- 이미지
- 이름
- 등급
- 산업 태그
- 가치
- 코스트
- 배당금
- Mr.Market 제안 배지
- 원본 수치와 최종 수치
```

### 3.2 소모형 자원 카드

```text
- 이미지
- 등급
- 제공 자원
- 코스트
```

소모형 자원 카드에는 이름을 표시하지 않는다.

---

## 4. 입력과 행동 확정

호버는 정보 확인이다.
행동 확정은 별도 매수/예약 입력에서 발생한다.

주식 카드 가능 입력:

```text
- 매수
- 예약
```

소모형 자원 카드 가능 입력:

```text
- 매수
```

예약은 주식 카드에만 가능하다.

---

## 5. 결제 조작

주식 매수의 투자 철학 비용은 칩 슬롯으로 표현한다.
딜은 칩 슬롯에 배치하지 않는다.
이미 획득한 투자 철학 마스터리는 같은 종류 투자 철학 비용을 자동으로 낮춘다.

카드 상세보기 화면이 없더라도, 비용 슬롯은 다음 중 하나의 방식으로 제공할 수 있다.

```text
- 카드 주변 확장 UI
- 호버 확대 카드 위의 결제 영역
- 매수 입력 후 나타나는 인라인 결제 영역
```

구체적인 UI 배치는 Unity 구현 단계에서 정한다.
기획상 중요한 점은 다음이다.

```text
매수 확정 전 칩 배치와 회수는 영업일을 소비하지 않는다.
매수 확정 시에만 자원을 소비하고 영업일을 소비한다.
```

---

## 6. 예약 표시

예약된 주식은 시장 슬롯 안에 그대로 남는다.
호버 확대 카드에서도 예약 상태를 표시한다.

표시 예:

```text
예약됨
```

이미 예약된 주식에는 예약 입력을 표시하지 않는다.
다른 주식이 이미 예약되어 있으면 새 예약 입력은 비활성화한다.

Mr.Market 제안 슬롯 위의 주식에는 제안 조건과 적용 후 최종 수치를 표시한다.
제안 슬롯 위 주식은 일반 매수가 아니라 제안 조건으로만 매수한다.
소모형 자원 카드 위의 제안은 비활성으로 표시하고 소진되지 않았음을 분명히 한다.

## 8. 구현 시 주의사항

```text
- CardDetail 상태를 사용하지 않는다.
- 카드 상세보기 닫기 흐름을 사용하지 않는다.
- 호버 확대는 정보 표시일 뿐이다.
- 호버 확대는 영업일을 소비하지 않는다.
- 시장 카드와 호버 확대 카드의 표시 정보는 동일하다.
- 매수 확정만 영업일을 소비한다.
- 예약은 영업일을 소비하지 않는 무료 행동이다.
- 소모형 자원 카드는 예약할 수 없다.
```
